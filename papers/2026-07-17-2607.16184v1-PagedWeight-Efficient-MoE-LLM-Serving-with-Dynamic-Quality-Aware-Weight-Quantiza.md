# PagedWeight: Efficient MoE LLM Serving with Dynamic Quality-Aware Weight Quantization

**Authors:** Yuchen Yang, Yifan Zhao, Anisha Dasgupta, Sasa Misailovic
**Published:** 2026-07-17 | **arXiv:** [2607.16184v1](http://arxiv.org/abs/2607.16184v1)

## Summary
This paper tackles a key bottleneck in serving large mixture-of-experts (MoE) language models: the competition between storing model weights and storing key-value caches during inference. PagedWeight dynamically adjusts the precision of model weights at runtime to free up GPU memory for the KV cache, enabling faster serving without sacrificing accuracy. This matters because MoE models are increasingly popular for their efficiency, but serving them at scale requires careful memory management.

## Theoretical Background
Mixture-of-Experts (MoE) models route different input tokens to different subsets of parameters (experts), reducing computation compared to dense models while maintaining quality. During LLM serving, two memory demands compete: the model weights themselves, and the key-value cache — stored activations from prior tokens needed for fast decoding. Quantization — reducing numerical precision from float32 or float16 to lower bitwidths — shrinks weight storage. Prior work uses static quantization (fixed precision throughout), but this ignores the dynamic nature of serving workloads and doesn't account for which weights matter most at different moments.

## Core Contributions
1. **Dynamic quality-aware quantization**: Unlike static methods, PagedWeight adjusts per-expert weight precision during runtime based on current KV cache pressure and accuracy needs. The key insight is that not all experts need the same precision at the same time.
2. **Tradeoff navigation**: The paper formalizes and empirically explores the three-way tension between task accuracy, GPU memory, and throughput/latency — and provides a method to navigate it automatically.
3. **Practical gains**: Achieves FP16-equivalent accuracy with 72% memory savings and 1.94× throughput improvement, and outperforms existing quantization baselines by up to 39.3% in quality at matched memory budgets.

## Connections to Other Work
PagedWeight extends prior quantization work (e.g., weight quantization, post-training quantization) by introducing temporal and spatial awareness — treating quantization as a **dynamic allocation problem** rather than a one-time compression. It relates to [[KV cache optimization]] and [[inference optimization]] literature. The work is complementary to [[speculative decoding]] and [[batching strategies]] — all aimed at reducing serving costs. It differs from [[pruning]] (removing weights entirely) and [[distillation]] (training smaller models), as it preserves model capacity while trading precision.

## Practical Applications
- **Cost reduction for large-scale LLM APIs** (e.g., inference services like OpenAI, Together AI): Serve more concurrent users per GPU, reducing infrastructure spend.
- **Mobile and edge deployment**: Tight memory budgets on phones or embedded devices can now accommodate larger MoE models.
- **Real-time chatbots and agents**: Faster latency enables better user experience in latency-sensitive applications.
- **Multi-tenant serving**: Pack more user requests per hardware unit by dynamically tuning memory allocation.

## Technical Limitations
- **Assumes MoE architecture**: Benefits are specific to mixture-of-experts; may not apply equally to dense models or other sparse architectures.
- **Accuracy-latency tradeoff**: At extreme memory pressure, some accuracy loss is inevitable; the paper doesn't quantify failure modes or provide hard guarantees.
- **Hardware specificity**: Optimal quantization decisions likely depend on GPU type, batch size, and token length; generalization across diverse setups is unclear.
- **Per-expert precision granularity**: Managing precision per expert may create load imbalance (some experts quantized more aggressively) or underutilize sparsity benefits.
- **KV cache growth**: The method assumes KV cache is the bottleneck; in very short sequences or high-throughput settings, this assumption may not hold.
- **Overhead of dynamic quantization**: The cost of runtime quantization decisions and dequantization is not fully characterized.

## Open Questions
- How does PagedWeight perform with dynamic scaling (variable batch size, token length) in production? Is the quantization policy stable or does it oscillate?
- Can we predict optimal quantization policies *offline* (before serving) to avoid runtime overhead?
- Does routing-aware quantization (quantizing experts differently based on expert load) outperform uniform expert quantization?
- How does PagedWeight interact with other serving optimizations like speculative decoding, batching strategies, or tensor parallelism?
- What happens when quantization combines with other compression techniques (pruning, distillation, low-rank approximation)?
- Can we extend this to attention precision (not just feed-forward expert weights)?

## Key Takeaway
Dynamic weight quantization balances GPU memory between model weights and KV cache, making MoE serving faster and cheaper without sacrificing accuracy.
