# Similarity search generalisation in contrastive learning with InfoNCE loss

**Authors:** Nick Whiteley
**Published:** 2026-07-10 | **arXiv:** [2607.09405v1](http://arxiv.org/abs/2607.09405v1)

## Summary
This paper provides theoretical guarantees for contrastive learning with InfoNCE loss, showing how similarity search performance generalizes from training to unseen data. The key insight is that as you increase the number of negative samples (k), the learned embeddings approximate an idealized similarity search up to an error of O(1/k), and the authors introduce a new continuity bound that explains why using more negatives stabilizes learning.

## Theoretical Background
Contrastive learning trains embedding models by pushing similar examples together and dissimilar ones apart. InfoNCE (Information Noise-Contrastive Estimation) is a widely-used loss function that compares a positive example against k negative samples using a softmax over similarities. Prior work has analyzed InfoNCE through mutual information maximization and through alignment-uniformity tradeoffs in embedding space, but mostly in limiting cases (k→∞). This paper complements those results by providing finite-k analysis using tools from functional analysis — specifically Gâteaux differentiation, which measures how a function changes when you perturb its input in a particular direction.

## Core Contributions
1. **Finite-k population risk bound**: Proves the population risk with k negatives is O(1/k) close to a cross-entropy loss comparing learned embeddings to an "oracle" similarity from the positive sample generator — giving concrete meaning to what InfoNCE learns at finite sample sizes.

2. **New continuity bound**: Introduces a generalization bound via Gâteaux differentiation that preserves the averaging structure of the InfoNCE loss (multiple negatives) and includes an interpretable inverse-temperature parameter tying to algorithmic choices.

3. **Generalization-stabilization result**: Demonstrates formally that the averaging over k negatives translates to lower generalization error as k grows, under Lipschitz assumptions on embeddings — explaining empirically observed stabilization.

## Connections to Other Work
This work sits between [[self-supervised learning]] theory and [[contrastive learning]] practice. It bridges the gap from mutual information interpretations (e.g., DIM, BYOL theory) and alignment-uniformity frameworks to practical generalization. The use of Gâteaux differentiation is novel for this setting; related work uses Lipschitz arguments or Rademacher complexity, but this approach directly captures the negative-sampling averaging effect that makes InfoNCE distinctive.

## Practical Applications
- **Embedding validation**: Teams building recommendation or search systems can estimate how many negative samples are truly needed for convergence without trial-and-error.
- **Model selection**: Helps practitioners choose k and inverse-temperature hyperparameters with principled bounds rather than grid search.
- **Transfer learning**: Understanding how learned embeddings generalize to unseen data is critical for pre-training foundation models for downstream tasks (e.g., vision or NLP retrieval).
- **Federated learning**: Bounds can guide how many negatives each client in a distributed setting should use, balancing compute and accuracy.

## Product & Startup Ideas
1. **AutoML for embedding models**: A tool that automatically tunes the number of negatives, batch size, and temperature based on dataset size and desired generalization margin — package as a scikit-learn-like module.
2. **Embedding quality audit**: SaaS that analyzes trained embeddings using the continuity bound to certify whether a model is generalizing well and recommend data/sample size fixes before deployment.
3. **Contrastive learning curriculum**: Adaptive training framework that starts with few negatives and gradually increases k based on the O(1/k) bound, matching compute budgets.
4. **Neural search indexing service**: Cloud API for similarity search that uses these bounds to optimize negative sampling per dataset, undercutting vector DB costs.
5. **Benchmark suite for contrastive models**: Open-source library that runs standardized tests with varying k on embeddings, reports bound-based confidence intervals, helps compare models fairly.

## Technical Limitations
- **Lipschitz assumption**: The Lipschitz-in-parameter assumption may not hold for modern neural networks in all regimes; the bound could be loose if this condition is violated.
- **Population risk vs. empirical**: The analysis is population-level; finite-sample empirical bounds would be needed for small datasets.
- **Oracle knowledge**: The O(1/k) result assumes you can meaningfully define an "oracle" similarity from the positive generator, which may be ill-defined for some data distributions.
- **Negative distribution**: The bound doesn't account for how negatives are sampled (e.g., hard negatives vs. uniform); it assumes a fixed negative distribution.
- **Scalability of bounds**: For high-dimensional embeddings or non-Euclidean geometries, the Gâteaux-based bound might be loose in practice.

## Open Questions
- Can you derive empirical finite-sample versions of these bounds (e.g., via concentration inequalities) that are tight enough to be useful on real datasets?
- How do the bounds change if negatives are adaptively sampled (e.g., hard-negative mining) rather than fixed?
- Does the theory extend to other contrastive losses (SimCLR, NT-Xent with different temperatures)?
- Can you characterize when the Lipschitz assumption holds (or doesn't) for common architectures like ResNets or ViTs?
- How do these bounds interact with data augmentation and the structure of positive pairs in contrastive learning?

## Key Takeaway
Increasing the number of negatives in InfoNCE stabilizes generalization to unseen similarity searches with diminishing returns (O(1/k)), and new continuity bounds explain and quantify this effect.
