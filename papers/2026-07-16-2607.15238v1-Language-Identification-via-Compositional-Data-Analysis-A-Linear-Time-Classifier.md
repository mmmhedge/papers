# Language Identification via Compositional Data Analysis: A Linear-Time Classifier Based on Log-Ratio Geometry

**Authors:** Paul-Andrei Pogăcean, Sanda-Maria Avram
**Published:** 2026-07-16 | **arXiv:** [2607.15238v1](http://arxiv.org/abs/2607.15238v1)

## Summary
This paper presents a lightweight, linear-time method for identifying which language a text is written in by treating character and bigram frequencies as compositional data — mathematical objects constrained to sum to 1, like probability distributions. Instead of using neural networks or standard distance metrics, the authors map these frequency vectors into a special geometric space (via centered log-ratio transformation) where standard Euclidean distance becomes statistically appropriate, then classify with simple nearest-neighbor or linear methods. The approach is fast, interpretable, and works well even on short texts.

## Theoretical Background
The paper builds on two distinct traditions. First, it inherits from classical language identification methods that rely on n-gram frequency analysis — counting how often character pairs appear in text. Second, and more technically novel, it applies **compositional data analysis (CoDA)**, a branch of statistics that treats data living on the simplex (vectors that sum to 1, like probability distributions) differently from ordinary vectors. Standard Euclidean geometry is inappropriate for such data because the simplex is curved; the **centered log-ratio (CLR) transformation** bijectively maps compositions onto a flat zero-sum subspace where Euclidean distance becomes meaningful — it corresponds to Aitchison distance, a proper metric for compositional data. The method also employs **Laplace smoothing**, a classical technique for handling sparse frequency counts (assigning small pseudocounts to unseen n-grams).

## Core Contributions
1. **Formal application of compositional geometry to language ID**: Reframes n-gram frequency distributions as compositional vectors and justifies using CLR-transformed space + Euclidean distance, rather than ad-hoc frequency-based metrics.
2. **Practical pipeline**: Combines CLR-transformed unigram and bigram features with Laplace smoothing to handle sparsity robustly, particularly on short texts.
3. **Empirical validation**: Demonstrates that this approach achieves strong accuracy across variable text lengths on a six-language benchmark, with particularly robust performance on longer sequences.
4. **Deterministic efficiency**: Provides an interpretable, resource-light alternative to neural methods — no learned parameters, O(n) time complexity.

## Connections to Other Work
This work sits at the intersection of classical statistical NLP and modern compositional statistics. It contrasts with [[Transformers]] and [[Large Language Models|LLMs]], which solve language ID (among other tasks) as a byproduct but require orders of magnitude more compute. It relates to classical n-gram language models and multinomial-based text classifiers, but applies proper statistical geometry rather than treating frequencies as ordinary vectors. The compositional data framework is standard in geology, genomics, and microbiome analysis but underutilized in NLP — this paper argues for its relevance. The method is deterministic and interpretable, aligning with recent interest in [[mechanistic-interpretability]] of text representations, though it does not claim to explain learned features.

## Practical Applications
1. **Low-resource language detection**: Embedded in edge devices (phones, IoT sensors) for identifying language in user input without cloud compute.
2. **Content moderation pipelines**: Initial coarse-grained language classification before routing to language-specific moderation rules or models.
3. **Document routing in multilingual systems**: Fast preprocessing in translation, search, or document management systems to route text to appropriate language-specific downstream processing.
4. **Linguistic analysis in resource-constrained settings**: Field researchers or NGOs working in regions with unreliable electricity or connectivity who need language ID without downloading large models.
5. **Anomaly detection in logs**: Identify unexpected language switches in server logs or user submissions that might indicate injection attacks or malformed input.

## Technical Limitations
1. **Limited language coverage**: Evaluated on only six languages; performance on distant language pairs (e.g., Chinese vs. English vs. Arabic) is unclear, as is scaling to 100+ language scenarios.
2. **Simplex assumption may not hold at very short sequences**: On texts shorter than a few dozen characters, frequency estimates become unreliable; CLR transformation can amplify noise from pseudocounts.
3. **No handling of code-switching**: Text mixing two languages (common in multilingual communities) is not addressed; the model assumes one dominant language.
4. **Script/orthography dependence**: Method relies on character and bigram statistics; languages sharing alphabets (e.g., English, German, Dutch) or heavily transliterated data may be harder to distinguish.
5. **No learning or adaptation**: Model is fixed once n-gram statistics are computed; cannot adapt to new language varieties, slang, or domain shifts without recomputation.
6. **Baseline comparison weakness**: Paper compares mainly to distance-based methods, not to modern lightweight neural approaches (e.g., fastText, DistilBERT); computational advantage may be marginal in practice.

## Open Questions
1. How does performance degrade on heavily code-switched text or creole languages? Can compositional geometry naturally model language mixture?
2. Can the CLR-transformed space be visualized or interpreted to show which bigrams are most discriminative between language pairs? This could strengthen the interpretability claim.
3. How sensitive is the method to the choice of Laplace smoothing parameter? Is there a principled way to tune it?
4. Does this approach generalize to other text classification tasks (e.g., dialect, genre, or sentiment), or is it specifically suited to language ID?
5. How does it perform on very long documents (e.g., novels) where bigram statistics are extremely stable? Does accuracy saturate?
6. Can you combine this method with learned embeddings (e.g., pre-trained subword tokenizers) to achieve better performance while retaining interpretability?

## Key Takeaway
Fast, interpretable language identification using compositional statistics instead of neural networks—simple, efficient, and mathematically principled.
