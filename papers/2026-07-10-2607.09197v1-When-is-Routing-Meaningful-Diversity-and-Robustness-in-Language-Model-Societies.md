# When is Routing Meaningful? Diversity and Robustness in Language Model Societies

**Authors:** Fantine Huot, Michael Kaisers, Mirella Lapata
**Published:** 2026-07-10 | **arXiv:** [2607.09197v1](http://arxiv.org/abs/2607.09197v1)

## Summary
This paper argues that routing policies in multi-model LLM systems are typically evaluated only on accuracy and cost, missing two critical properties: whether the models actually behave differently (behavioral differentiation) and whether the routing is stable under input variations (robustness). The authors show that high accuracy can mask meaningless routing—where either all models respond identically or the same query gets routed inconsistently—and propose metrics to diagnose these failure modes.

## Theoretical Background
The work builds on two main foundations. First, it extends **Hierarchic Social Entropy (HSE)**, a measure from organizational theory and multi-agent systems that quantifies diversity within a group of actors. Second, it adopts the framework of **routing policies** from mixture-of-experts systems and retrieval-augmented generation, where a router selects which model or component should handle a given query. The paper treats language models as agents in a society, borrowing concepts from multi-agent systems (coordination, specialization, redundancy). The perturbation-based robustness framework is inspired by adversarial robustness in machine learning—testing whether small surface-form changes (e.g., rephrasing) alter the router's decisions.

## Core Contributions
1. **Conceptual reframing**: Identifies behavioral differentiation and routing stability as orthogonal and necessary properties for meaningful routing, independent of task accuracy.
2. **HSE adaptation**: Applies Hierarchic Social Entropy to quantify diversity in LLM societies, showing strong diminishing returns (suggesting most diversity is captured by <10 agents from larger pools).
3. **Robustness metric**: Introduces a perturbation-based robustness evaluation that tests whether routing decisions are stable under query paraphrasing.
4. **Empirical findings**: Demonstrates that KNN-based routers gain accuracy from specialist societies but collapse under perturbation, while prompted routing maintains stability—showing accuracy and meaningfulness can sharply diverge.

## Connections to Other Work
This relates to several research areas:
- **[[Mixture of Experts]]** and conditional computation: routing in MoE systems also selects specialized components, but typically only optimizes for efficiency and accuracy.
- **[[Multi-agent reinforcement learning]]**: concerns about coordination, specialization, and redundancy in agent societies.
- **Robustness and adversarial robustness**: the perturbation framework echoes work on input paraphrasing and model stability.
- **Model composition and ensembling**: the paper contrasts with approaches that combine models additively without explicit routing.
- **[[RLHF]] and preference learning**: related to how models are optimized, though this paper focuses on system-level properties rather than individual model training.

## Practical Applications
1. **Enterprise AI systems**: Designing cost-effective multi-LLM deployments where you verify not just accuracy but actual delegation—ensuring you're not overpaying for redundant models.
2. **Retrieval-augmented generation (RAG) systems**: Diagnosing whether specialized retrievers (e.g., for law, medicine, finance) are genuinely handling distinct query types or whether routing is unstable.
3. **API marketplace selection**: Helping organizations choose complementary third-party LLM APIs by testing behavioral differentiation before committing to a multi-model contract.
4. **Quality assurance in chatbots**: Ensuring conversational routing (intent classification → specialized responder) actually routes consistently on paraphrased user inputs.

## Product & Startup Ideas
1. **Router Diagnostics SaaS**: Automated tool that audits multi-LLM systems for behavioral differentiation and robustness; flags redundant models and unstable routing before deployment.
2. **LLM Society Designer**: Interactive platform that recommends optimal subsets of models (coreset heuristics) for a given task portfolio, using HSE-like metrics to minimize cost while preserving diversity.
3. **Robustness Testing for LLM APIs**: Marketplace for stress-testing custom routing policies across paraphrases, domain shifts, and adversarial inputs—certification for production multi-model systems.
4. **Specialization Analyzer**: Tool that profiles LLM behavior across domains (legal, medical, technical) and visualizes which model pairs are truly complementary vs. redundant, enabling better router design.
5. **Prompt Optimization for Routers**: Automated system that tunes router prompts (vs. KNN embedding-based routing) to maximize both accuracy and perturbation robustness.

## Technical Limitations
1. **Limited router architectures tested**: Only KNN and prompted routers are evaluated; other routing approaches (learned classifiers, graph-based) are not explored.
2. **Perturbation scope**: The robustness metric relies on paraphrasing as the main perturbation type; doesn't test domain shift, adversarial noise, or other real-world variations.
3. **HSE diminishing returns claim**: The observation that <10 agents capture most diversity is empirical and may not generalize across all task distributions or model pools.
4. **Behavioral differentiation metric**: The paper uses HSE but doesn't deeply explore *why* models diverge (training data, architecture, scale) or how to design societies to ensure differentiation.
5. **Scalability unclear**: Experiments on RouterBench and EmbedLLM may not reflect performance on much larger or more heterogeneous model pools (100s of models).
6. **Causality**: High accuracy despite routing collapse might indicate the task is easy enough that any model works; the paper doesn't fully disentangle task difficulty from routing meaningfulness.

## Open Questions
1. **Optimal society design**: Given a task and budget, what algorithm provably selects the smallest set of models maximizing both accuracy and HSE diversity?
2. **Causal analysis of divergence**: Which factors (model size, training data, fine-tuning) causally drive behavioral differentiation? Can you predict it without extensive sampling?
3. **Robustness under distribution shift**: How do routing stability metrics (perturbation-based) correlate with out-of-distribution robustness in real deployments?
4. **Router-model co-design**: Can you jointly optimize router training and model specialization to guarantee both accuracy and meaningful routing?
5. **Theoretical bounds**: Is there a fundamental tradeoff between routing stability and accuracy? Under what conditions can you have both?
6. **Generalization of coreset heuristic**: Does the "<10 agents" rule hold for language understanding, code generation, reasoning tasks, etc., or is it task-specific?

## Key Takeaway
A multi-model routing system can achieve high accuracy while remaining meaningless—routing identically behaving models or inconsistently assigning queries—and you need explicit metrics for behavioral differentiation and routing stability to diagnose this failure mode.
