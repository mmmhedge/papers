# Persian Pixel: A large-scale synthetic OCR dataset for Persian language

**Authors:** Pouria Mahdi, Haq Nawaz Malik
**Published:** 2026-07-22 | **arXiv:** [2607.20385v1](http://arxiv.org/abs/2607.20385v1)

## Summary
This paper introduces Persian Pixel, a large synthetic dataset of 343,000 text images designed to train optical character recognition (OCR) systems for Persian language. The work solves a critical bottleneck: Persian OCR lags far behind Latin-script systems because Persian's complex cursive script and lack of annotated training data make it hard to build accurate recognition models. By programmatically generating realistic training images with authentic Persian typography and document degradation, the authors provide a cost-effective path to robust Persian OCR without expensive manual labeling.

## Theoretical Background
OCR systems learn to map document images to text. Traditional approaches require large annotated datasets (image-text pairs), which are expensive to create manually. Recent advances use synthetic data generation — computationally creating realistic training images from text — to bypass this cost. Persian script introduces unique challenges absent in Latin systems: characters join obligatorily in cursive form, glyphs change shape based on position (context-dependent rendering), diacritics stack above/below base letters, and typographic styles vary widely (Naskh, Nastaliq, etc.). Prior work in OCR has succeeded with synthetic data for simpler scripts, but Persian's typographic complexity demands faithful modeling. Modern OCR architectures like [[TrOCR]] (transformer-based) and Donut can leverage large labeled datasets to learn robust recognition.

## Core Contributions
1. **SynthOCR-Gen rendering framework**: A pipeline that generates Persian text images with accurate typographic behavior — context-dependent glyph joining, positional variants, diacritic placement — rather than naive character-by-character rendering.
2. **Large-scale synthetic dataset**: 343,000 image-text pairs spanning sentence, paragraph, and full-page layouts, sourced from a curated 7-million-word Persian corpus.
3. **Domain-gap bridging via degradation**: Over 25 stochastic degradation models (ink bleed, paper aging, blur, illumination variation, scanner noise, compression artifacts) that make synthetic images resemble real scanned documents.
4. **Demonstrated utility**: Validation that the dataset can train transformer-based OCR models (TrOCR, Donut) effectively, with open-source release to accelerate Persian document digitization research.

## Connections to Other Work
This work fits within the broader trend of synthetic data generation for vision tasks. It parallels efforts in [[generative-models]] for data augmentation and addresses challenges similar to those in low-resource NLP — when real labeled data is scarce, programmatic generation becomes viable. The use of domain randomization (degradation models) echoes sim-to-real transfer techniques in robotics and computer vision. The paper implicitly builds on knowledge of [[transformers]] applied to OCR (TrOCR, Donut) and acknowledges the data bottleneck common in non-Latin script NLP, a recurring theme in multilingual language models.

## Practical Applications
1. **Persian document digitization**: Government archives, libraries, and universities can convert historical Persian manuscripts and legal records to searchable digital text at scale.
2. **Multilingual document processing**: Companies processing contracts, invoices, or forms in Persian-speaking regions (Iran, Afghanistan, Tajikistan, parts of Iraq) can integrate Persian OCR into automated document workflows.
3. **Historical manuscript preservation**: Museums and heritage organizations can digitize rare Persian literary and scientific texts (poetry, scientific treatises) for scholarly access.
4. **E-commerce and fintech**: Businesses operating in Persian-speaking markets can automate identity document verification (visas, passports, ID cards) and receipt scanning.
5. **Accessibility**: Converting Persian text in images (screenshots, signs, books) to machine-readable form for text-to-speech and screen readers.

## Product & Startup Ideas

2. **Multilingual Invoice Processing**: An OCR-powered accounts payable automation tool that handles Persian (via this dataset) alongside Arabic, Urdu, and English invoices, using the trained models to extract vendor, amount, and date fields for accounting software.

## Technical Limitations
1. **Synthetic-to-real gap**: Despite degradation models, a gap likely remains between the distribution of synthetic images and real, historically variable documents. Performance may degrade on very old, damaged, or unusual typography.
2. **Typeface coverage**: The dataset uses "representative" Persian typefaces; handwritten or very decorative scripts are not addressed.
3. **Layout generalization**: The dataset covers sentence, paragraph, and full-page layouts, but may not handle unusual document structures (tables, mixed scripts, images with text overlays).
4. **Evaluation scope**: The paper does not provide detailed benchmark comparisons against existing Persian OCR systems or competing datasets (if they exist), making it hard to quantify the improvement.
5. **Diacritic correctness**: While diacritics are modeled, the linguistic correctness of diacritic placement in the synthetic corpus is not validated — errors could propagate to trained models.
6. **Language model decoupling**: The dataset pairs images with text, but downstream OCR models may need language modeling to correct errors; the dataset does not address integration with Persian language models.

## Open Questions
1. **Quantitative benchmarking**: What is the relative performance of OCR models trained on Persian Pixel versus real annotated Persian data (if available) or other low-resource script datasets? How much synthetic data is needed to match real-data performance?
2. **Handwriting and cursive variation**: Can the framework extend to synthesize handwritten Persian text with sufficient realism? How does handwriting fidelity affect OCR accuracy?
3. **Historical script rendering**: Can the degradation and typeface models faithfully represent historical Persian scripts (classical Nastaliq, Kufic elements) for manuscript digitization tasks?
4. **Cross-script robustness**: How well do models trained on Persian Pixel handle mixed-script documents (Persian + Arabic, Persian + English) common in real documents?
5. **Fine-grained error analysis**: Which Persian OCR failure modes (diacritics, ligatures, rare glyphs) does this dataset mitigate most effectively? Are there remaining hard cases?
6. **Downstream applications**: How much does improved Persian OCR accuracy impact downstream tasks like named entity recognition, machine translation, or information extraction for Persian text?

## Key Takeaway
Synthetic data generation with realistic typography and degradation unlocks OCR for complex non-Latin scripts, scaling Persian text recognition without costly manual annotation.
