# ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning

**Authors:** Yanjun Zhao, Ruizhong Qiu, Tianxin Wei, Yuanchen Bei, Zhining Liu et al.
**Published:** 2026-07-02 | **arXiv:** [2607.02509v1](http://arxiv.org/abs/2607.02509v1)

## Summary
This paper introduces ReContext, a training-free method that improves how large language models use information in very long documents (up to 128K tokens). The key insight: LLMs can access long contexts but often fail to actually use the relevant information when answering questions. ReContext solves this by recursively identifying and replaying relevant evidence before generating answers, without requiring model retraining or external memory systems.

## Theoretical Background
The paper builds on two foundational ideas: (1) the observation that modern LLMs like Llama and Qwen have expanded context windows but exhibit a "lost in the middle" phenomenon — they struggle to attend to relevant passages buried in long documents; (2) associative memory theory from cognitive science, which frames context as a memory store, questions as retrieval cues, attention mechanisms as cue-trace associations, and the replay process as trace reactivation. This theoretical lens reframes the long-context problem as a memory retrieval problem rather than purely an attention problem.

Prior work has tackled long-context reasoning through context compression, external retrieval systems, or fine-tuning. ReContext differs by using the model's own internal attention signals (already computed during inference) to organize evidence without modifying the model weights or architecture.

## Core Contributions
1. **ReContext mechanism**: A recursive, training-free algorithm that extracts model-internal relevance signals to construct a query-conditioned evidence pool, then replays selected evidence before final generation while keeping the original context intact.

2. **Theoretical grounding**: Formal analysis connecting long-context reasoning to associative memory principles, providing interpretability for why recursive replay works.

3. **Empirical validation**: Consistent improvements across three model sizes (Qwen3-4B, Qwen3-8B, Llama3-8B) on eight long-context benchmarks, achieving best average rank across all backbones — demonstrating generalization.

4. **Simplicity and practicality**: Inference-only method with no training overhead, no external dependencies, and no context pruning (preserves full input).

## Connections to Other Work

## Practical Applications
- **Legal document review**: Analyze contracts, case law, or depositions spanning hundreds of pages to extract relevant clauses and precedents.
- **Scientific literature synthesis**: Extract relevant findings and methodologies from long research papers or patent documents for summarization or prior-art analysis.
- **Customer support**: Process long conversation histories or product documentation to generate contextually accurate responses.
- **Financial analysis**: Extract relevant information from long earnings reports, regulatory filings (10-Ks), or analyst notes for question-answering.
- **Code understanding**: Analyze large codebases or API documentation to answer developer questions about specific functionality.

## Technical Limitations
- **Computational overhead during inference**: Recursive evidence selection requires additional forward passes through the model to extract relevance signals, increasing latency compared to single-pass generation.
- **Relevance signal quality**: The method depends on the model's internal attention patterns being reliable proxies for relevance; adversarial or out-of-distribution queries may produce poor evidence pools.
- **Scalability to extreme lengths**: While tested on 128K contexts, behavior at 200K+ tokens is unexplored; memory and compute costs may become prohibitive.
- **Domain generalization**: Tested on specific benchmarks; performance on highly specialized domains (medical, legal jargon) or low-resource languages is unclear.
- **No explicit ranking**: Recursive replay reorders evidence but doesn't provide confidence scores or explicit ranking for downstream verification.

## Open Questions
- **How many recursion steps are optimal?** Does performance plateau, and does it vary by model size or document type?
- **Does ReContext help on reasoning-heavy vs. retrieval-heavy tasks differently?** Some tasks need evidence; others need synthesis.
- **Can ReContext be combined with external retrieval?** Does mixing in retrieved documents (from RAG) help or hurt?
- **How sensitive is the method to the order of replayed evidence?** What if evidence is re-sorted by confidence or diversity?
- **Does this work for multilingual contexts or cross-lingual retrieval?** Can evidence in one language help answer questions in another?
- **What's the theoretical connection between associative memory and transformer attention?** Can this theory predict when ReContext will fail?

## Key Takeaway
ReContext demonstrates that LLMs' long-context failures often stem from poor evidence organization at inference time, not architectural limits — and this can be fixed by recursively replaying model-discovered relevant passages without retraining.
