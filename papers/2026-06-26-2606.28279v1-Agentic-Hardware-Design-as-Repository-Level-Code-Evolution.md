# Agentic Hardware Design as Repository-Level Code Evolution

**Authors:** Cunxi Yu, Chenhui Deng, Nathaniel Pinckney, Brucek Khailany
**Published:** 2026-06-26 | **arXiv:** [2606.28279v1](http://arxiv.org/abs/2606.28279v1)

## Summary
HORIZON is an agentic framework that applies self-evolving AI agents to hardware design, treating chip design artifacts as code repositories that can be iteratively improved through automated loops. The system achieves 100% completion on multiple hardware design benchmarks (ChipBench, RTLLM, Verilog-Eval) using a hands-free agent approach, but the authors acknowledge these benchmarks are controlled proxies for the vastly more complex real-world chip design problem.

## Theoretical Background
Hardware design traditionally involves writing Register Transfer Language (RTL) code—low-level descriptions of digital circuits—which is time-consuming and error-prone. Recent work has explored using large language models to generate hardware code, but most treat design as a single-pass generation problem. This paper builds on the concept of repository-scale code evolution: the idea that code systems can be iteratively improved by AI agents that understand version control (git), project structure, and feedback mechanisms. The Markdown harness is a domain-specific language wrapper that packages design problems with evaluation criteria and acceptance predicates (conditions that mark success).

## Core Contributions
1. **Repository-as-state**: Extends repository-scale self-evolution (previously applied to EDA tools) to hardware design artifacts themselves, using git worktrees for isolated state management and replay.
2. **Domain-agnostic harness**: A Markdown compilation system that encodes domain knowledge, executable evaluators, acceptance predicates, and runtime policies—making it possible to apply the same agentic loop to diverse hardware design tasks.
3. **Empirical completeness**: Demonstrates 100% benchmark completion across four distinct benchmark suites with no human intervention, including structured categories (CVDP) typically seen as particularly challenging.

## Connections to Other Work
This work bridges [[large-language-models]] (code generation) with [[agents]] (autonomous iteration), following the paradigm of self-improvement loops seen in [[RLHF]] and similar agent-based code synthesis. It extends prior repository-scale code evolution work from EDA (Electronic Design Automation) software to the hardware designs those tools produce. Related to broader work on AI-driven chip design (RTLLM, Verilog-Eval benchmarks) but differs by treating design as an iterative refinement problem rather than one-shot generation.

## Practical Applications
1. **Automated RTL generation**: Accelerate early-stage hardware design by auto-generating and refining Verilog/SystemVerilog code for standard digital blocks (memory controllers, interconnects, arithmetic units).
2. **Design bug fixing**: Automatically patch failing RTL designs by iterating on code based on simulation feedback.
3. **Design space exploration**: Systematically evolve hardware designs toward Pareto-optimal points (power, area, timing trade-offs) by encoding those objectives as acceptance predicates.
4. **EDA tool integration**: Embed HORIZON-like loops into existing CAD flows to enable hands-free design refinement stages.
5. **Junior engineer augmentation**: Use agentic loops to handle routine design patterns, freeing human engineers for architecture and verification decisions.

## Product & Startup Ideas
1. **AI-native HDL IDE**: A code editor that runs HORIZON loops in the background, continuously refining and suggesting improvements to hardware designs with human review gates.
2. **Design-as-a-service**: Cloud platform accepting RTL specs and acceptance predicates, returning optimized designs via agentic evolution (pay per compute/iteration).
3. **Hardware bug bounty network**: Automated service that finds and fixes RTL bugs and performance issues in open-source hardware projects using agentic loops.
4. **Parametric IP library generator**: Tool that takes a hardware IP specification and auto-generates optimized implementations for different process nodes, power budgets, or latency constraints.
5. **Chip design assistant SaaS**: Real-time VSCode/IDE plugin that suggests RTL improvements, test coverage, and design patterns as engineers write, powered by agentic loops on each edit.

## Technical Limitations
1. **Benchmark vs. reality gap**: The four benchmarks are controlled, isolated problems; real chip design involves cross-functional constraints (physical design, power delivery, routing), team dependencies, and multi-month project timelines that these proxies don't capture.
2. **Scalability unknown**: No data on how the approach scales to designs with millions of lines of RTL or complex microarchitectures; benchmarks likely involve smaller, self-contained units.
3. **Acceptance predicate design**: Requires humans to specify clear success criteria (acceptance predicates) upfront; for open-ended design goals this is non-trivial.
4. **Simulation overhead**: Hardware design evaluation is expensive (functional sim, formal verification); the paper doesn't detail how evaluation cost scales or how many iterations are needed in practice.
5. **Generalization**: Unclear whether agents trained or tuned on one design domain generalize to unfamiliar hardware types (e.g., analog, mixed-signal, custom memory).

## Open Questions
1. How does iteration count and total wall-clock time compare to human-engineer timelines for equivalent design quality?
2. Can the approach handle designs with implicit cross-module constraints or architectural dependencies not captured in a single git worktree?
3. What happens when acceptance predicates conflict (e.g., minimize area vs. maximize throughput)? How sensitive is the loop to predicate specification?
4. Does the framework generalize beyond digital RTL (e.g., analog circuits, layout, physical design)?
5. How much human oversight or validation is actually needed before taping out a design evolved by HORIZON in production?
6. Can multiple agent instances collaborate (multi-agent) on a single large design?

## Key Takeaway
HORIZON demonstrates that AI agents can autonomously evolve hardware designs via repository-level iteration loops, but benchmark success does not yet prove the approach solves the broader, messier engineering challenge of real chip design.
