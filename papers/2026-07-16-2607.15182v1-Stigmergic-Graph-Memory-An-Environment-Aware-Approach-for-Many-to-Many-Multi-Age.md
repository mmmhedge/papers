# Stigmergic Graph Memory: An Environment-Aware Approach for Many-to-Many Multi-Agent Pickup and Delivery

**Authors:** Aditya Dutta, Joon-Seok Kim
**Published:** 2026-07-16 | **arXiv:** [2607.15182v1](http://arxiv.org/abs/2607.15182v1)

## Summary

This paper addresses a key logistics problem: how to automatically assign pickup-and-delivery tasks across multiple agents in crowded warehouses. The authors propose Stigmergic Graph Memory (SGM), a memory system that tracks recent activity (congestion, collisions, delays) at warehouse locations and uses that information to smartly choose which pickup and delivery points to assign to each agent—before path planning happens. On simulated warehouse layouts, SGM achieves 20–37% throughput gains over existing methods by letting past execution patterns inform task allocation, not just routing.

## Theoretical Background

Multi-agent pickup-and-delivery (MAPD) extends classical vehicle routing: instead of fixing source and destination endpoints upfront, the system must select *which agent*, *which source location*, and *which destination location* to pair for each incoming request. Prior work on graph-guidance methods typically optimize routing *after* endpoints are chosen, leaving the critical allocation decision uninformed by live warehouse state.

Stigmergy is a principle from swarm intelligence—agents leave signals in their environment (like ant pheromones) that guide future agents without explicit communication. SGM adapts this idea by storing a decaying memory of recent execution signals (collisions, congestion, delays) on warehouse nodes and directed edges. This memory layer sits upstream of the path planner, influencing which feasible endpoint pairs are ranked highest and thus more likely to enter the solver—without changing the planner's collision-avoidance or validity guarantees.

## Core Contributions

1. **SGM architecture**: A bounded, decaying memory layer that records execution signals (congestion, collisions, traversal times) at graph nodes and edges in real time.
2. **Allocation-first design**: Unlike prior graph-guidance work, SGM shapes *endpoint selection* (which source–destination pair to assign), not only routing after goals are fixed.
3. **Empirical validation**: Systematic evaluation across 5 warehouse layouts, 3 load levels, and 25 random seeds per condition, showing consistent 20.5–36.7% throughput gains over reconstructed baselines.
4. **Planner-agnostic integration**: SGM works with any underlying path planner and collision resolver, maintaining their validity guarantees.

## Connections to Other Work

This work sits at the intersection of:
- **Classical multi-agent path finding (MAPF)** and task allocation: most MAPF research assumes fixed goal locations; SGM extends this to the many-to-many case where goals must be instantiated.
- **Swarm intelligence and stigmergy**: borrows the conceptual principle of environment-mediated coordination without explicit agent-to-agent messaging.
- **Graph-based routing optimization**: extends prior [[graph-guidance]] methods by operating earlier in the decision pipeline (allocation vs. routing).
- **Decentralized warehouse control**: relates to work on congestion-aware task assignment and real-time load balancing in fulfillment systems.

## Practical Applications

1. **Amazon/Walmart fulfillment centers**: Real-time task assignment to robotic fleets during peak order surges, reducing congestion and cycle times.
2. **Autonomous port terminals**: Assignment of multiple AGVs to pick containers from different stacks and deliver them to different destinations, avoiding gridlock in congested areas.
3. **Hospital logistics robots**: Assigning courier robots to fetch items from multiple supply rooms and deliver to different departments, with memory of recent bottlenecks.
4. **Manufacturing shop floors**: Coordinating multiple autonomous carts to move work-in-progress between job stations while respecting congestion hot spots.

## Technical Limitations

1. **Bounded memory**: SGM's fixed-size memory may lose signal in very long transients or rapidly changing demand patterns; no principled decay-rate tuning provided.
2. **Myopic allocation**: Ranking endpoints only on recent execution signals ignores longer-term demand forecasts or future request streams; may suboptimize for sustained high load.
3. **Planner dependency**: SGM relies on the downstream planner's collision-resolution quality; poor or slow planners will erode gains.
4. **Real-world noise**: Assumes accurate sensing of congestion signals; sensor failures, occlusions, or multi-robot entanglement may corrupt memory.
5. **Limited layout generalization**: Evaluation on 5 simulated layouts; unclear how SGM generalizes to novel warehouse geometries or significantly different topologies (outdoor, multi-floor).
6. **No formal guarantees**: No provable throughput bounds, convergence rates, or optimality certificates.

## Open Questions

1. **Optimal decay schedules**: How should memory weights decay with time? Is exponential optimal, or should decay adapt to demand intensity or agent load?
2. **Multi-timescale memory**: Could SGM benefit from separate fast (congestion) and slow (preference) memory channels, tuned at different cadences?
3. **Interaction with learning**: Could SGM memory feed into a learned policy (e.g., neural network) that selects endpoints, rather than simple ranking?
4. **Comparative bounds**: How does SGM's throughput compare to the theoretical optimum (e.g., LP relaxation of the MAPD problem)?
5. **Real robot deployment**: What happens when SGM is deployed on actual physical robots with noisy sensors, actuators, and communication delays?
6. **Scalability**: How does SGM performance scale to 100+ agents and 1000+ requests per hour? Does memory overhead become a bottleneck?
7. **Adversarial requests**: How robust is SGM to adversarial or adversarially-timed request streams designed to exploit its memory decay?

## Key Takeaway

Recent warehouse activity memory improves task allocation throughput by steering agent-endpoint pairs toward less-congested options before routing begins.
