# Selective Disclosure Watermarking for Large Language Models

**Authors:** Xuyang Chen, Xiang Li, Yangxinyu Xie, Qi Long
**Published:** 2026-07-06 | **arXiv:** [2607.05353v1](http://arxiv.org/abs/2607.05353v1)

## Summary
This paper introduces HeRo (Hierarchical Vocabulary Routing), a watermarking method for LLM-generated text that lets different parties verify only the portions of an embedded message they're authorized to see. Unlike existing watermarking approaches that require full message exposure to verify any part, HeRo enables fine-grained access control over watermark disclosure—solving a real privacy problem in authentication and IP protection scenarios.

## Theoretical Background
Watermarking embeds hidden, verifiable signals into data—in this case, text generated by LLMs. Prior work divides watermarking into two categories: zero-bit schemes (yes/no detection: "is this text synthetic?") and multi-bit schemes (embedding metadata like model identity or generation timestamp). The field builds on classical information embedding theory and cryptographic access control. HeRo extends multi-bit watermarking by introducing *hierarchical disclosure*—borrowed from cryptographic concepts like threshold schemes and hierarchical encryption, where different keys unlock different layers of information. The core challenge is embedding information at multiple granularities without degrading text quality, which requires careful sampling strategies that don't bias the underlying language model's token distributions.

## Core Contributions
1. **Selective disclosure framework**: First watermarking method allowing partial verification without revealing the entire payload—verifiers with different access levels decode only authorized portions.
2. **Hierarchical vocabulary routing**: A recursive vocabulary partitioning scheme that distributes watermark bits across tree-like access levels, enabling fine-grained control.
3. **Unbiased sampling proof**: Demonstrates the method preserves the unbiasedness of the LLM's token sampling process, maintaining text quality (perplexity and human evaluation metrics).
4. **Empirical validation**: Shows high detection accuracy and low latency across access levels while maintaining generation quality.

## Practical Applications
1. **Content attribution in generative AI platforms**: News organizations or academic publishers embed watermarks allowing editors (high access) to see full metadata (timestamp, author, revision history) while public readers see only "this is LLM-generated."
2. **Model licensing & IP protection**: Companies can embed usage restrictions in generated content—cloud providers see full license terms, end-users see only verification of legitimate generation.
3. **Legal/regulatory compliance**: Enterprises embed compliance metadata (data provenance, PII handling logs) verifiable only by auditors, not by general users.
4. **Multi-tenant SaaS systems**: Different customers of an LLM API see only watermark portions relevant to their deployment (cost allocation, version, security tier).
5. **Deepfake detection with privacy**: Detect synthetic speech/video while revealing only necessary metadata (e.g., "synthetic" yes/no) to the public, but full provenance to law enforcement.

## Technical Limitations
1. **Vocabulary partitioning overhead**: Recursive partitioning adds computational cost to token generation; scales with access control depth (number of authorization levels).
2. **Watermark robustness not addressed**: No evaluation against adversarial attacks (paraphrasing, token substitution, distillation) that could erase the watermark.
3. **Access control model assumed static**: Doesn't handle dynamic access revocation or policy updates during/after generation.
4. **Limited to English/single-language**: Experiments likely focus on English; behavior on multilingual models unclear.
5. **Scalability to very large payloads**: Unclear how many bits can be reliably embedded without degrading quality; tradeoff between message size and detection accuracy not fully explored.
6. **No threat model for collusion**: If multiple verifiers at different levels collaborate, could they reconstruct the full message?

## Open Questions
1. **Robustness evaluation**: How does HeRo withstand paraphrasing, back-translation, or distillation attacks?
2. **Collusion resistance**: Can colluding verifiers at different access levels reconstruct the full payload? What cryptographic guarantees exist?
3. **Adaptive access control**: Can the framework support dynamic access changes post-generation without re-watermarking?
4. **Multilingual & multimodal extension**: Does hierarchical routing generalize to non-English text or image/audio generation?
5. **Information-theoretic limits**: What are fundamental bounds on payload size vs. detectability vs. access levels?
6. **User study on disclosure privacy**: Do end-users actually value selective disclosure? What are realistic access control scenarios?

## Key Takeaway
HeRo brings selective disclosure to LLM watermarking, enabling fine-grained access control over embedded metadata—a missing capability that bridges cryptography and content authentication for real-world deployment scenarios.
