# Requential Coding: Pushing the Limits of Model Compression with Self-Generated Training Data

**Authors:** Shikai Qiu, Marc Finzi, Yujia Zheng, Kun Zhang, Andrew Gordon Wilson
**Published:** 2026-07-13 | **arXiv:** [2607.11883v1](http://arxiv.org/abs/2607.11883v1)

## Summary
This paper introduces "requential coding," a new compression method that measures how well a student neural network can reproduce samples selected by a teacher model trained on the same data. Unlike prior compression techniques that struggle with model size or data entropy, requential coding produces code lengths that depend only on disagreement between teacher and student—enabling dramatically shorter codes and tighter generalization bounds for billion-parameter language models.

## Theoretical Background
Model compression builds on information theory: a short code for training data implies the model has discovered regularities. Prior work distinguishes two approaches. Quantization (parameter-based methods) compresses weights directly, but code length scales with parameter count regardless of how much information those parameters actually encode. Prequential coding compresses the training trajectory—recording how the model's predictions evolve—but must encode the full data sequence, yielding long codes when data has high entropy (randomness). This paper builds on [[PAC-Bayes]] bounds, which relate model complexity to generalization via compression length, treating compression as a measure of implicit regularization. The key insight is that a teacher-student framework can separate learnable structure from noise in the data.

## Core Contributions
1. **Requential coding method**: A new compression scheme where a teacher model selects samples and the student's code records only *which* samples were selected (via teacher-student disagreement), not the samples themselves. This decouples code length from both parameter count and data entropy.
2. **Scale-independent compression**: Code length grows with model disagreement, not model size, and improves with scale—larger ensembles compress to shorter codes despite more parameters.
3. **Tight PAC-Bayes bounds**: Plugging requential codes into PAC-Bayes bounds yields state-of-the-art generalization guarantees for billion-parameter LLMs, outperforming bounds based on quantization even when quantization has zero error.
4. **Empirical insights**: The code length predicts multi-epoch overfitting and reveals that lower-entropy text contains far more learnable structure than higher-entropy image data, isolating signal from noise.

## Connections to Other Work
This extends [[PAC-Bayes bounds]], which have struggled to provide tight generalization guarantees for large neural networks. It relates to [[Model compression]] and [[Quantization]], which it outperforms. The teacher-student framework echoes [[Knowledge distillation]] but uses the student's own distribution, not synthetic data. The connection between compression and generalization originates in [[Minimum description length (MDL)]] principles. The work also informs debates about [[Scaling laws]]—showing compression improves with scale in compute-optimal regimes—and touches on [[Implicit regularization]] by measuring what structure models actually learn.

## Practical Applications
1. **Model deployment**: Compressing large language models to minimal codes enables efficient storage and transmission without losing theoretical guarantees on generalization.
2. **Generalization auditing**: Practitioners can compute requential codes to predict how well a model will generalize before deployment, replacing empirical validation.
3. **Dataset quality assessment**: Code length isolates learnable vs. random content, helping identify noisy or mislabeled data and informing data collection strategies.
4. **Efficient fine-tuning**: Using requential compression to select which samples to fine-tune on, reducing computational cost for adaptation.
5. **Model selection**: Comparing codes across architectures or hyperparameters without expensive validation sets.

## Product & Startup Ideas
1. **Model Compression-as-a-Service**: An API that compresses and audits large model generalization bounds in hours, sold to ML teams deploying billion-parameter models; price scales with model size.
2. **Dataset Quality Analytics**: A tool for data teams that computes requential codes to flag noisy, redundant, or low-signal examples in image, text, and multimodal datasets before training.
3. **Efficient Model Marketplace**: A platform where practitioners upload models; requential coding automatically ranks them by generalization efficiency, enabling "Pareto-optimal" model recommendations.
4. **AutoML for Compression**: A system that optimizes teacher-student pairs to minimize code length, then uses the code to recommend data subsets and model architectures for a target domain.
5. **LLM Audit Suite**: Enterprise software that computes requential codes and PAC-Bayes bounds for proprietary LLMs, providing formal generalization guarantees for regulatory compliance (e.g., financial or healthcare AI).

## Technical Limitations
1. **Teacher-student dependency**: Requential coding requires training a teacher model, adding computational overhead; the method is only as good as the teacher's quality, and optimal teacher selection is unclear.
2. **Scalability to very large models**: The paper uses models up to billions of parameters; practical limits for trillion-parameter models (mixture-of-experts, etc.) are unexplored.
3. **Cross-domain generalization**: The code is specific to a training dataset and teacher; how it transfers to new domains or data distributions is unclear.
4. **Information-theoretic assumptions**: The analysis assumes the teacher can effectively capture learnable structure; if the teacher itself has fundamental limitations (e.g., cannot parse certain linguistic phenomena), the code may be loose.
5. **Computational cost**: Computing codes for large datasets and models may be expensive; the paper does not thoroughly characterize runtime vs. dataset/model size trade-offs.
6. **Data-dependent bounds**: PAC-Bayes bounds are data-dependent and loose in practice; tightness for real-world deployment remains unclear.

## Open Questions
1. How does requential coding perform with fundamentally different teacher architectures (e.g., sparse, recurrent, or symbolic models) vs. the same dense architecture?
2. Can the method be extended to online or streaming settings where data arrives sequentially and the teacher must adapt?
3. Does requential coding reveal interpretable structure about which parts of a dataset are learnable vs. noise (e.g., via visualization)?
4. How does the method scale to models trained on heterogeneous or federated data, where a single teacher cannot see all training samples?
5. Can requential codes be used as a training signal (e.g., in curriculum learning) to improve sample efficiency?
6. What is the relationship between requential compression and recent work on test-time adaptation and in-context learning in LLMs?

## Key Takeaway
Requential coding compresses models by measuring teacher-student disagreement, yielding codes independent of model size and achieving tight generalization bounds for billion-parameter networks.
