# The Logic of Data Access and Data Exchanges

**Authors:** Alexandru Baltag, Sonja Smets
**Published:** 2026-06-30 | **arXiv:** [2606.31858v1](http://arxiv.org/abs/2606.31858v1)

## Summary
This paper develops a new logical framework that extends Dynamic Epistemic Logic (DEL) — a formal system for reasoning about what multiple agents know and how knowledge changes — to handle data access and exchange scenarios. The key innovation is adding operators that express not just *whether* an agent knows something, but also how much they can narrow down unknowns (e.g., "I can restrict this variable to 5 possible values") and what happens when data sources become accessible. The work matters because it provides a rigorous mathematical foundation for reasoning about privacy, data sharing, and information leakage in multi-agent systems.

## Theoretical Background
Dynamic Epistemic Logic (DEL) is a formal framework from mathematical logic for modeling knowledge in multi-agent systems and how knowledge states change following events (like public announcements). Traditional DEL focuses on propositional knowledge — facts that are true or false. This paper builds on DEL's machinery of epistemic models (which represent agents' beliefs about the world) and update mechanisms, but extends it to handle *numerical knowledge* — cases where an agent knows the exact value of a number, or can at least constrain it to a finite set of possibilities. The work also incorporates formal semantics for data sources as structured repositories of information, and defines operations on these sources (access, modification, revelation) that change the global information state.

## Core Contributions
1. **Non-propositional knowledge operators**: Formal operators expressing knowledge of specific variable values and conditional narrowing ("given this extra info, I can reduce possibilities from 10 to 3").
2. **Cardinality-bounded knowledge**: Formulas expressing that an agent can narrow a variable to *at most N* possibilities, generalizing beyond exact-value knowledge.
3. **Minimization-based definite descriptions**: Operators that let the logic name and compare hypothetical values by selecting the minimum among possibilities (useful for ordering and reasoning about data).
4. **Data-exchange dynamics**: DEL-style modalities for events beyond simple announcements — including database hacking, open-source data sharing, and access-controlled information flow.
5. **Complete axiomatizations and decidability**: Formal proofs that the resulting logics are axiomatically complete and algorithmically decidable, with co-expressivity results.

## Connections to Other Work
This extends [[Dynamic Epistemic Logic]] by incorporating techniques from formal epistemology and database theory. It relates to:
- **Epistemic logic with quantitative operators**: Prior work on logics for measuring knowledge (e.g., degrees of belief, information-theoretic bounds).
- **Privacy and information flow**: Formal approaches to reasoning about data leakage in protocols (connecting to work on secure multi-party computation and differential privacy, though those are computational rather than purely logical).
- **Temporal and dynamic logic**: Similar update mechanisms appear in temporal logics and transition systems, but DEL's epistemic focus is distinctive.
- **Logic of knowledge and action**: Related to logics combining knowledge and capability (agents that can do things as well as know things).

## Practical Applications
1. **Privacy auditing frameworks**: Formal verification that a data-sharing protocol doesn't inadvertently leak sensitive information even to sophisticated adversaries who can infer values from limited access.
2. **Access control policy specification**: Rigorous specification and verification of policies that govern when and what data becomes accessible to which agents or groups.
3. **Cyber-security threat modeling**: Formal models of scenarios where attackers gain partial or complete access to databases, enabling reasoning about what they can deduce.
4. **Data governance in collaborative systems**: Specification of rules for multi-stakeholder systems (e.g., healthcare, finance) where different parties have different data rights.
5. **Blockchain and distributed ledger auditing**: Formal analysis of information visibility and state revelation in consensus protocols where agents progressively learn about transactions or states.

## Product & Startup Ideas
1. **Formal privacy verification tool**: A software product that takes a data-sharing protocol (e.g., an API gateway, federated learning system, or microservice architecture) and automatically checks whether it satisfies formal privacy guarantees, reporting specific inference attacks that violate policies.
2. **Epistemic risk dashboard**: A monitoring/analytics platform for enterprises that logs data-access events and computes formal measures of "what can be inferred by whom" in real time, alerting on anomalies.
3. **Compliance-as-code framework**: A domain-specific language (DSL) compiler targeting this logic, allowing lawyers and compliance officers to write GDPR/CCPA/sectoral rules as executable formal specifications, with automatic audit trails.
4. **AI alignment verification**: A tool for reasoning about what learned models in a multi-agent system can infer about unobserved training data or other agents' capabilities, relevant to adversarial robustness and privacy-preserving ML.
5. **Secure protocol synthesis**: An automated tool that, given a formal specification of which agents may learn which variables and under what conditions, synthesizes communication protocols guaranteed to achieve those constraints.

## Technical Limitations
1. **Scalability of decidability**: While the logic is decidable, the computational complexity (likely high — possibly non-elementary) means that practical decidability algorithms may be infeasible for large agent counts or complex formulas; the paper doesn't discuss complexity bounds.
2. **Abstraction gap**: The logic reasons about propositional and numerical knowledge but not *probabilistic* uncertainty — it's classical, so agents either can or cannot narrow a set of values, not how likely each is. Real adversaries use Bayesian inference.
3. **Infinite domains**: The framework assumes finite domains for variables and agent sets; extending to infinite-valued or uncountable variables is not addressed.
4. **Data semantics**: The paper treats data sources abstractly; it doesn't model the *syntax* or *structure* of databases, so reasoning about queries, joins, or derived inference is limited.
5. **Temporal and resource constraints**: No modeling of computational cost (queries take time, storage is limited) or real-time dynamics; all access is instantaneous.
6. **Non-logical side channels**: The logic models logical inference but not timing attacks, power analysis, or other physical information leaks.

## Open Questions
1. **Complexity analysis**: What is the exact computational complexity of satisfiability and model checking for the proposed logics? Are there fragments with better complexity?
2. **Probabilistic extension**: Can the framework be extended with probabilistic operators to reason about inference attacks by computationally bounded or Bayesian-rational agents?
3. **Infinite-valued variables and continuous data**: How do these logics handle real-valued data or infinite domains? Can the decidability results be preserved?
4. **Compositionality**: If two data-exchange protocols are formally verified in isolation, can their sequential or parallel composition be verified without re-checking from scratch?
5. **Implementation and case studies**: Can the axiomatizations be implemented in a practical theorem prover or SAT solver? Can real protocols (e.g., federated learning, multi-party computation) be formally modeled and verified?
6. **Connection to information theory**: How do the formal constraints (e.g., "narrowable to N possibilities") relate to entropy and information-theoretic bounds on leakage?

## Key Takeaway
Reasoning rigorously about who knows what values and what can be inferred from selective data access requires extending epistemic logic with non-propositional knowledge operators and formal semantics for data exchange events.
