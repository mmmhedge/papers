# Scalable Behaviour Cloning on Browser Using via Skill Distillation

**Authors:** Kaisen Yang, Zheng Jiang, Yuzhao Peng, Houde Qian, Boshi Zhang et al.
**Published:** 2026-06-30 | **arXiv:** [2606.32014v1](http://arxiv.org/abs/2606.32014v1)

## Summary
This paper proposes a method for training browser automation agents by distilling human web interactions into reusable natural-language skills. Rather than training agents from scratch on individual tasks, the researchers extract high-level behavioral patterns from real user browsing data, compress them into skills an agent can understand and reuse, and organize these skills into a structured graph. The key insight is that internet users already demonstrate vast amounts of skilled behavior online—the challenge is not collecting training data but teaching agents to reason abstractly about incomplete information.

## Theoretical Background
The paper builds on [[behavior cloning]] — learning agent policies by imitating human demonstrations — and skill learning frameworks that decompose complex behaviors into reusable primitives. It assumes agents struggle most with decision-making under partial observability (not knowing the full state), and that this knowledge gap can be bridged by extracting implicit task structure from human interaction traces. The approach leverages language models as a medium for skill representation, treating natural language as a way to compress and communicate behavioral patterns that agents can later retrieve and compose. This connects to work in imitation learning, hierarchical reinforcement learning, and prompt-based agent control.

## Core Contributions
1. **Skill distillation pipeline**: A method to automatically convert raw user interaction trajectories into compact, natural-language task descriptions that agents can execute and reason about.
2. **Skill graph organization**: A structured representation that prevents unbounded skill accumulation by organizing distilled skills hierarchically, allowing new capabilities to emerge through composition rather than memorization.
3. **Scalability via implicit priors**: Demonstration that real-world human browsing traces contain sufficient structure to train capable agents without manually curated task sets.
4. **Empirical validation**: Evidence that language-based skill representations improve agent generalization and compositionality compared to low-level imitation baselines.

## Connections to Other Work
This work extends classical [[behavior cloning]] by adding an abstraction layer — instead of cloning raw actions, it clones higher-level skills encoded as language. It relates to [[prompt-based agents]] and in-context learning approaches that condition agents on task descriptions. The skill graph concept echoes hierarchical RL and modular policy learning, but grounds it in language rather than learned latent codes. It also connects to recent work on [[web agents]] and automated task completion, which typically relies on manual task specification or reinforcement learning; this paper argues for the sufficiency of human interaction data alone.

## Practical Applications
1. **Software RPA (Robotic Process Automation)**: Automating enterprise workflows (data entry, report generation, email management) by learning from existing employee interactions.
2. **Personal productivity assistants**: Browser plugins that watch your web habits and offer to automate repetitive tasks (form filling, recurring searches, document workflows).
3. **Accessibility tools**: Automating complex web interactions for users with motor or cognitive disabilities, learning customized shortcuts from their typical patterns.
4. **Customer support automation**: Training agents on support staff's browsing patterns to handle routine customer requests (order lookup, account updates, basic troubleshooting).
5. **Web data collection and curation**: Automated crawling and information extraction tailored to organization-specific workflows without manual scripting.

## Product & Startup Ideas
1. **"SkillShift"** — A browser extension that watches your work, distills your repetitive tasks into reusable skills, and offers 1-click automation suggestions with explainable reasoning.
2. **"TaskGenerator for Enterprise"** — SaaS for large organizations to extract automation opportunities from employee interaction logs, automatically generating and deploying robotic process automation workflows without IT engineering.
3. **"BrowserMind"** — A personal AI agent that learns your unique web habits and proactively suggests new automations; sells the aggregated (anonymized) skill graph as benchmarks to enterprise AI teams.
4. **"AccessBot"** — Specialized tool for generating accessibility scripts from real user browsing sessions, enabling disabled users to share their interaction patterns to create adaptive automation.
5. **"Skill Marketplace"** — Community platform where users monetize distilled skills (e.g., "fill tax forms," "manage AWS billing") that others can download, adapt, and run in their own browser environments.

## Technical Limitations
1. **Skill extraction quality**: The distillation process depends on LLM summarization quality; noisy or ambiguous human traces may yield vague or incorrect skill descriptions.
2. **Incomplete information assumption**: The paper assumes partial observability is the main bottleneck, but agents may also struggle with long-horizon reasoning, novel situations, and common-sense reasoning not present in training data.
3. **Skill graph scalability**: Organizing skills into a graph assumes meaningful hierarchical structure exists; some domains may have flat or tangled skill dependencies that resist clean composition.
4. **Generalization bounds unclear**: No formal analysis of when skills trained on one user's patterns transfer to another user's workflows, or how distribution shift between training traces and deployment affects performance.
5. **Evaluation scope**: Likely evaluated on relatively well-structured domains (forms, document editing). Unclear how method scales to creative, goal-directed browsing or adversarial environments.

## Open Questions
1. How does skill transfer perform across domains and users with different interaction styles? Is there a "skill kernel" that works universally, or are most skills idiosyncratic?
2. Can the skill graph be learned end-to-end rather than hand-organized, discovering hierarchical decompositions from interaction data directly?
3. How do agents handle skills with conflicting preconditions or side effects? Is there a principled way to reason about skill composition safety?
4. What is the minimum interaction data needed to distill a robust skill? Is there a sample-efficiency frontier?
5. Can this approach extend beyond web automation to other structured environments (APIs, CLI, desktop GUIs) where human traces are plentiful?

## Key Takeaway
Scalable browser agents may be built not by collecting more task labels but by automatically extracting and reusing the behavioral skills already embedded in billions of hours of human web browsing.
