# OpenAI to acquire Ona

**Source:** OpenAI
**Published:** 2026-06-11 | **Link:** [https://openai.com/index/openai-to-acquire-ona](https://openai.com/index/openai-to-acquire-ona)

## Summary
OpenAI is acquiring Ona to integrate secure, persistent cloud environments into Codex, its code generation model. This acquisition enables long-running AI agents to operate reliably within enterprise workflows, extending beyond single-session code execution.

## Key Points
- OpenAI plans to acquire Ona (a cloud infrastructure/environment provider)
- Goal is to enhance Codex with persistent, secure cloud compute capabilities
- Use case: deploying long-running AI agents in enterprise settings
- Focus on enabling agents to maintain state and execution across extended periods rather than isolated code generation sessions

## Why It Matters
Persistent cloud environments are critical infrastructure for practical AI agent deployment. By acquiring Ona, OpenAI removes a technical barrier to embedding autonomous agents into real enterprise workflows—where agents must maintain context, handle failures, and execute tasks over hours or days rather than seconds. This positions OpenAI to compete more directly in the emerging AI-agents-as-service market.

## Connections to Other Work
This acquisition reflects broader industry momentum toward [[agents]] and [[multi-agent-systems]] as the next frontier beyond static LLM APIs. It's analogous to how infrastructure acquisitions (containerization, orchestration) were critical for scaling distributed systems in the 2010s. Related to concepts like [[mechanistic-interpretability]] for understanding agent behavior and [[alignment]] for ensuring safe long-running autonomous systems.

## Key Takeaway
OpenAI is investing in infrastructure to operationalize AI agents as long-running enterprise tools, not just one-off code generation.
