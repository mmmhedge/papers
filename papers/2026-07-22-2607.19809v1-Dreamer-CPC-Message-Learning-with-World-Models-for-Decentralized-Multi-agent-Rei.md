# Dreamer-CPC: Message Learning with World Models for Decentralized Multi-agent Reinforcement Learning

**Authors:** Taisuke Takayama, Naoto Yoshida, Tadahiro Taniguchi
**Published:** 2026-07-22 | **arXiv:** [2607.19809v1](http://arxiv.org/abs/2607.19809v1)

## Summary
This paper proposes Dreamer-CPC, a decentralized multi-agent reinforcement learning method where agents learn to communicate by exchanging messages derived from their internal world models rather than raw observations. The key insight is that messages grounded in latent world-model states (which encode history) enable better coordination than messages based only on current observations, especially when some agents have missing or partial information.

## Theoretical Background
The paper builds on three foundational ideas: (1) **world models** — neural networks that learn compact representations of environment dynamics by predicting future observations from past experience; (2) **Collective Predictive Coding (CPC)** — a representation learning framework where agents learn shared message representations by predicting each other's latent states; and (3) **decentralized multi-agent RL** — settings where agents must coordinate through local communication rather than a central controller. DreamerV3 is the base architecture—a recent state-of-the-art model-based RL agent. The paper extends CPC from working on raw observations (as in prior work IPPO-CPC) to working on latent world-model states, which carry temporal context.

## Core Contributions
1. **Integration of CPC into world models**: First to embed message learning into the latent state space of a world model (DreamerV3), allowing messages to encode temporal history rather than just instantaneous information.
2. **Decentralized message inference**: Each agent independently maintains both a world model and a message module, with no central coordinator—practical for truly distributed settings.
3. **Empirical validation on partial observability**: Demonstrates 4–5× improvement over IPPO-CPC on the new CatchApple task, where temporary observation loss makes history-aware communication critical.
4. **New benchmark task**: Introduces CatchApple, a structured environment designed to test coordination under missing observations.

## Connections to Other Work
Dreamer-CPC sits at the intersection of several research areas:
- **World model literature** (e.g., [[Dreamer]], [[PlaNet]]): Extends model-based RL to multi-agent settings with learned communication.
- **MARL communication** (e.g., [[QMIX]], [[RIAL/DIAL]]): Differs by grounding communication in latent world dynamics rather than policy networks or value functions.
- **Representation learning for coordination**: Related to [[Contrastive Predictive Coding]] and self-supervised pretraining for multi-agent systems.
- **Partial observability**: Connects to POMDP and partially observable multi-agent game literature; world models naturally address non-stationarity from other agents' hidden states.

## Practical Applications
1. **Robotic swarms in GPS-denied environments**: Multiple robots with intermittent sensor failures (e.g., underwater drones, satellite occlusion) can coordinate by sharing learned state representations.
2. **Autonomous vehicle platoons**: Vehicles share learned models of road/traffic dynamics to maintain formation when one vehicle's sensors are temporarily obscured.
3. **Distributed sensor networks**: Sensor agents with partial coverage learn and exchange compressed models of environmental state rather than raw data, reducing bandwidth.
4. **Multi-robot exploration**: Teams exploring unknown environments where line-of-sight is intermittent; shared latent world models help maintain coherent task progress.
5. **Game-playing agents**: Multi-agent game AI where agents must infer opponent intentions from noisy observations—world models capture opponent behavior patterns over time.

## Product & Startup Ideas
1. **Decentralized coordination software for IoT**: Platform enabling smart home/industrial IoT devices to learn and share world models for resilient automation without a central server (privacy + robustness angle).
2. **Swarm robotics middleware**: Plug-and-play library for multi-robot systems that automatically learns inter-robot communication protocols, targetable to logistics/warehouse automation companies.
3. **Autonomous fleet management SaaS**: Real-time coordination system for delivery drones or autonomous vehicles that learns communication strategies for dynamic routing under sensor unreliability.
4. **Simulation-as-a-service for MARL**: Cloud platform pre-training decentralized agents in synthetic worlds (e.g., traffic, warehouse), fine-tuned on customer hardware with learned communication protocols.
5. **Edge AI for smart manufacturing**: Distributed control system for factory robots that learns to self-organize communication to maintain production under equipment failures or sensor occlusion.

## Technical Limitations
1. **Scalability unclear**: No results with >2 agents shown; how does message size/computation scale with team size?
2. **World model assumptions**: Assumes agents can learn accurate latent dynamics models; fails when environment is highly stochastic, rapidly changing, or requires long-horizon prediction.
3. **Communication bottleneck not addressed**: No analysis of message bandwidth, latency, or dropout; real networks have constraints not modeled here.
4. **Limited baselines**: Only compared to IPPO-CPC and no-communication; missing comparisons to other MARL communication methods (QMIX, CommNet, TarMAC).
5. **Task specificity**: Both evaluation environments are relatively simple and synthetic; transfer to realistic, high-dimensional domains (vision-based robotics) is unvalidated.
6. **Partial observability design**: The CatchApple task is somewhat artificial (deterministic observation loss); doesn't test robustness to realistic sensor noise or adversarial occlusion.

## Open Questions
1. **Scalability**: How does Dreamer-CPC perform with 5+ agents? Does message size grow, and what is the latency for communication rounds?
2. **Heterogeneous agents**: Can agents with different observation spaces or action spaces learn compatible messages? What happens with asymmetric information?
3. **Communication efficiency**: Can we quantify the bits-per-episode needed? Can we compress messages learned by CPC without degrading performance?
4. **Transfer across tasks**: Do learned world models and communication strategies transfer to different environments or reward structures?
5. **Adversarial robustness**: How robust are learned messages to noise, delays, or adversarial message corruption?
6. **Comparison to alternatives**: How does Dreamer-CPC compare to parameter sharing, attention-based communication, or graph neural networks for coordination?
7. **Convergence guarantees**: Any theoretical analysis of when/why CPC-based communication in world models leads to convergent multi-agent learning?

## Key Takeaway
Agents learn richer communication by grounding messages in world-model history rather than raw observations, enabling coordination despite missing sensor data.
