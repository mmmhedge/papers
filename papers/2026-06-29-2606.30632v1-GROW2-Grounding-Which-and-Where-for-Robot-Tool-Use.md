# GROW$^2$: Grounding Which and Where for Robot Tool Use

**Authors:** Yuhong Deng, Yuyao Liu, David Hsu
**Published:** 2026-06-29 | **arXiv:** [2606.30632v1](http://arxiv.org/abs/2606.30632v1)

## Summary
GROW² enables robots to creatively repurpose objects as tools (e.g., using a plate to cut cake) by grounding task instructions into both semantic object understanding and geometric 3D localization. The system leverages vision-language models for commonsense reasoning about which object to use and where to apply force, avoiding the need for large labeled datasets and achieving zero-shot generalization to novel objects.

## Theoretical Background
This work builds on several foundations:

**Affordance grounding** — the problem of understanding which object regions support which actions, traditionally studied in robotics and embodied AI. Earlier approaches relied on task-specific training or predefined tool categories.

**Vision-Language Models (VLMs)** — multimodal neural networks (like CLIP or GPT-4V) that reason jointly over images and text, enabling transfer of commonsense knowledge without task-specific training. VLMs have recently become powerful for zero-shot visual understanding.

**Foundation models** — large pretrained neural networks that generalize across domains. The paper uses both VLMs (for semantic reasoning) and vision foundation models (for 3D spatial grounding from RGB-D images).

**Part-based representations** — decomposing objects into semantic parts (e.g., handle, blade) rather than treating objects holistically. Parts serve as a natural middle ground between high-level semantics and low-level geometry.

The key insight is hierarchical decomposition: split the hard problem of "which object and where to act" into a semantic level (handled by language models) and a geometric level (handled by vision models), allowing modular reuse of existing foundation models.

## Core Contributions
1. **Hierarchical affordance grounding framework** — formalizing open-world tool use as a two-stage problem: semantic selection (which object/part) followed by geometric localization (where in 3D space).

2. **VLM-based tool selection pipeline** — using commonsense reasoning from pretrained language models to parse natural-language task instructions, select suitable objects from open categories, and identify task-relevant parts without fine-tuning.

3. **Zero-shot generalization** — demonstrating that the approach generalizes to novel objects and tool use patterns without additional training, a significant advance over prior work requiring task-specific data.

4. **Empirical validation across benchmarks** — outperforming state-of-the-art on established affordance prediction tasks and showing success in both simulated and real-world robotic manipulation.

## Connections to Other Work
This paper addresses limitations of prior approaches:

- **End-to-end imitation learning** for robot manipulation typically requires large, task-specific datasets; GROW² avoids this via foundation models.
- **Affordance prediction** (e.g., [[Predicting Grasps and Manipulation Actions from RGB-D Images]]) traditionally used closed-world object categories; this work extends to open-vocabulary settings.
- **Language-grounded robotics** (e.g., [[Language Models as Zero-Shot Planners]], [[SayCan]]) have used language models for task planning but relied on predefined skill primitives; GROW² grounds language directly to 3D geometry.
- **Part-based models** in computer vision are well-studied, but their integration with VLMs for robot control is novel.

Related to broader trends in **embodied AI** and the emerging paradigm of using [[Transformers]] and large vision-language models as components in robotic systems.

## Practical Applications
1. **Kitchen robotics** — collaborative robots that perform novel food preparation tasks by repurposing available utensils when standard tools are unavailable or during learning phases.

2. **Disaster response** — field robots that adapt tool use to novel damage scenarios (e.g., using a rigid pipe as a lever when standard pry bars are unavailable).

3. **Manufacturing assembly** — industrial arms that flexibly use object parts identified from design specifications without requiring task-specific training for each variant.

4. **Household assistance for elderly/disabled** — robots that understand and execute creative task workarounds when precise tools aren't in the environment.

5. **Warehouse automation** — sorting and handling systems that generalize to novel object categories and improvise with available implements.

## Product & Startup Ideas
1. **"RoboMechanic" — a robotic maintenance platform** that uses GROW² to inspect and manipulate industrial equipment by zero-shot reasoning about which parts of available tools to use for each repair task.

2. **"AdaptiveArm" — a plug-and-play software layer** for existing robot arms that adds creative tool-use capabilities to commercial manipulators, monetized as a SaaS subscription with cloud-based VLM inference.

3. **"Kitchen Companion" — a home robot system** for meal prep that queries a language model to solve "what can I use instead?" queries, reducing reliance on specialized tooling and increasing user autonomy.

4. **"ConstructBot" — construction site assistance** leveraging GROW² to identify and use available materials creatively for tasks (e.g., scaffolding setup), trained on job-site safety constraints.

5. **"Disassembly-as-a-Service"** — a robotics platform for electronic waste and product recycling that uses semantic grounding to identify which part of a tool to use on novel product designs, scaling circular economy operations.

## Technical Limitations
1. **RGB-D dependency** — the geometric grounding stage requires depth information; performance may degrade with sensor noise, transparent objects, or reflective surfaces common in real-world scenes.

2. **VLM hallucination** — commonsense reasoning from language models can fail on domain-specific or safety-critical tasks; the paper doesn't deeply address when semantic selection is unreliable.

3. **Part representation bottleneck** — assumes that task-relevant affordances decompose neatly into object parts. Some tools (e.g., a rope) have continuous structure not well-captured by discrete parts.

4. **Limited physical validation** — while real-world experiments are included, the scope appears small; scaling to high-force operations (cutting, pressing) with complex contact dynamics remains unexplored.

5. **No explicit safety modeling** — the system selects tools and grasping regions but doesn't reason about force distribution, stability, or risk of tool breakage—critical for deployment.

6. **Computational cost** — requires multiple inference passes (VLM for semantics, vision foundation model for localization); latency and computational requirements for deployment are not characterized.

## Open Questions
1. **How does the approach handle ambiguous affordances?** If multiple objects could serve a function (two types of spatula), does the system reason about tool effectiveness ranking?

2. **Can the method scale to multi-step tool use?** Real tasks often chain operations (drill hole, insert screw); does GROW² plan across sequential tool changes?

3. **What failure modes emerge with real force dynamics?** The paper shows grasping and localization, but does a plate actually cut cake effectively? When does semantic correctness fail at physics?

4. **How does performance degrade with part ambiguity?** Some objects have ill-defined parts; does VLM part identification remain reliable for unusual objects?

5. **Can this integrate with force/torque feedback?** Real manipulation requires closed-loop control; how would the system adapt when predicted regions don't produce expected forces?

6. **Does the approach work for adversarial or safety-critical domains?** How reliable is zero-shot affordance prediction when mistakes risk damage or injury?

## Key Takeaway
By splitting open-world tool use into semantic reasoning (which object, which parts) via VLMs and geometric grounding (where in 3D) via vision foundation models, GROW² achieves zero-shot generalization to novel objects and tasks without task-specific training, fundamentally shifting robot manipulation from data-heavy end-to-end learning toward compositional reasoning.
