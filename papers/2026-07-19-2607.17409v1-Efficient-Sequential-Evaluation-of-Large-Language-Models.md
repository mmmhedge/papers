# Efficient Sequential Evaluation of Large Language Models

**Authors:** Chia-Yu Hsu, Shubhanshu Shekhar
**Published:** 2026-07-19 | **arXiv:** [2607.17409v1](http://arxiv.org/abs/2607.17409v1)

## Summary
This paper tackles the problem of efficiently evaluating a new large language model (LLM) on a fixed set of questions by reusing historical performance data from previous LLMs. The authors develop statistical methods to build tight confidence intervals around the model's true capability and design smart querying strategies that select which questions to ask next, minimizing the number of evaluations needed. This matters because LLM evaluation is expensive, and more efficient methods could accelerate model development cycles.

## Theoretical Background
The paper builds on sequential analysis and online learning, specifically the theory of confidence sequences (CSs) — statistical intervals that remain valid as you collect more data, rather than requiring a fixed sample size upfront. The authors focus on two classical approaches: reverse information projection (RIPr), which uses optimal transport and information geometry to construct tight sequences, and testing-by-betting, a martingale-based framework where bets accumulate to create confidence guarantees. The core idea is inverting test supermartingales (sequences that maintain probabilistic guarantees under any stopping rule) to obtain confidence intervals. They also leverage active learning principles — the observation that intelligently selecting which data points to label can yield faster learning than passive sampling.

## Core Contributions
1. **Oracle-optimal CS construction**: Proves that the RIPr-based approach is optimal in an oracle setting (where prediction errors are known), establishing a theoretical benchmark.
2. **Growth-oriented querying rule**: Proposes a principled active-learning strategy that maximizes worst-case expected log-increment at the endpoints of the current confidence interval, balancing exploration and exploitation.
3. **Diagnosis of shrinkage bottlenecks**: Identifies two concrete factors that slow convergence — accumulated prediction mismatch (errors in the learned predictive model) and spikiness in the query distribution — providing actionable insights for practitioners.
4. **Mixture querying strategies**: Introduces hybrid rules combining growth-oriented queries, prediction refinement, and uniform exploration, designed to mitigate identified slowdowns.
5. **Empirical finding**: Surprisingly, simple uniform sampling often matches or beats adaptive methods, suggesting the problem structure may not always reward complexity.

## Practical Applications
1. **LLM benchmarking platforms**: Reduce evaluation cost by 30–50% when assessing new models on established benchmark sets by reusing patterns from historical models.
2. **A/B testing for model updates**: Rapidly compare two LLM variants on a question set without exhaustively evaluating both on all questions.
3. **Budget-constrained model selection**: When you have a fixed evaluation budget, adaptively allocate questions to narrow down which of several candidate models is best with high confidence.
4. **Continuous monitoring**: Track production LLM performance over time using minimal labeling, updating confidence bounds incrementally as new data arrives.
5. **Multi-model comparison**: Simultaneously evaluate 5–10 LLM candidates and identify the top performers with fewer total evaluations than a naive all-pairs approach.

## Technical Limitations
1. **Assumption of stable question distribution**: The method assumes question-level performance patterns are stationary over time — if new question types emerge or the benchmark evolves, the learned predictors become stale.
2. **Dependence on high-quality historical data**: Predictions of correctness are learned from prior models; if historical data are sparse, noisy, or from very different model families, accumulated prediction mismatch balloons and the CS shrinkage slows dramatically.
3. **Limited to fixed question sets**: The framework cannot handle open-ended or continuously growing question pools; it's designed for one-shot evaluation on a static benchmark.
4. **Scalability to very large benchmarks**: While more efficient than exhaustive evaluation, the method still requires learning individual question-level predictors, which becomes costly for benchmarks with millions of items.
5. **Active learning regret**: The paper doesn't provide regret bounds showing the method converges quickly — only empirical evidence; in worst-case adversarial settings, adaptive strategies could underperform.
6. **Uniform sampling paradox unexplained**: The surprising finding that uniform sampling sometimes beats adaptive rules suggests the theoretical understanding is incomplete; the conditions under which this holds are unclear.

## Open Questions
1. **When does adaptivity help?** Under what precise conditions (question diversity, model similarity, budget regime) do adaptive querying rules beat uniform sampling? Can you characterize the problem structure?
2. **Transfer across diverse models**: How well do predictors learned from GPT-3 and Llama generalize to radically different architectures (e.g., mixture-of-experts models)? Is a domain-adaptation pre-step necessary?
3. **Confidence-sequence tightness in practice**: How much is the RIPr-based CS over-conservative in realistic settings due to model mismatch? Can you design less pessimistic variants?
4. **Computational cost of querying**: The paper assumes querying cost is homogeneous; what if different questions have vastly different evaluation times (image generation vs. text classification)? How should cost-aware CS construction change?
5. **Human-in-the-loop refinement**: Can you incorporate human feedback on prediction mismatch to dynamically retrain the correctness predictor and speed up shrinkage?
6. **Multi-task evaluation**: Extend the framework to simultaneously evaluate multiple related LLM families or capabilities (reasoning, factuality, bias) with shared confidence regions.

## Key Takeaway
Sequential evaluation using historical LLM data can shrink uncertainty about a new model's capability using 30–50% fewer questions, but uniform sampling sometimes beats adaptive strategies.
