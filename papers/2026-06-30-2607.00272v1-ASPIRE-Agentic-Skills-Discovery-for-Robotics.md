# ASPIRE: Agentic /Skills Discovery for Robotics

**Authors:** Runyu Lu, Yubo Wu, Ethan Kou, Letian Fu, Wenli Xiao et al.
**Published:** 2026-06-30 | **arXiv:** [2607.00272v1](http://arxiv.org/abs/2607.00272v1)

## Summary
ASPIRE is a system that automatically discovers, writes, and refines robot control programs through continuous learning and exploration. Instead of manually programming each robot behavior, ASPIRE autonomously diagnoses failures, generates fixes, and builds a reusable library of skills that work across different tasks, simulated and real robots, and robot types—achieving dramatic improvements (up to 77%) over prior methods on complex manipulation tasks.

## Theoretical Background
This work builds on several foundational ideas: the code-as-policy paradigm (where control logic is expressed as executable programs rather than just neural network weights), continual learning systems (which accumulate knowledge over time), and evolutionary search (which explores diverse solutions rather than optimizing a single trajectory). Prior robot learning systems typically train task-specific policies or rely on hand-coded skill libraries. ASPIRE extends these by automating the discovery and refinement of skills through closed-loop execution traces—detailed multimodal records (vision, proprioception, force feedback) of what happened during robot execution. The system leverages recent advances in large language models (LLMs) for code generation and reasoning about failure modes, though the specifics of how LLMs are integrated aren't detailed in the abstract.

## Core Contributions
1. **Autonomous failure diagnosis and repair synthesis**: The system closes the loop—it observes robot execution failures, analyzes multimodal traces to understand what went wrong, and automatically generates fixes to the control code.
2. **Continual skill library building**: Rather than one-off fixes, ASPIRE distills validated repairs into reusable skills that transfer across tasks and embodiments.
3. **Evolutionary exploration strategy**: Goes beyond single-trajectory refinement by generating diverse task sequences and control programs, enabling broader exploration of the behavior space.
4. **Sim-to-real transfer evidence**: Skills discovered in simulation transfer to real robots with different APIs and embodiments, reducing real-world programming effort.
5. **Strong empirical results on long-horizon tasks**: Achieves 31% success on unseen long-horizon tasks in LIBERO-Pro Long versus 4% for baselines, even without test-time reasoning.

## Connections to Other Work
ASPIRE relates to several research threads:
- **Code generation for robotics**: Extends recent work using LLMs to generate robot policies by adding the critical closed-loop refinement loop.
- **Skill discovery and hierarchical RL**: Complements [[Hierarchical Reinforcement Learning]] by automatically structuring learned skills into libraries.
- **Sim-to-real transfer**: Addresses a longstanding challenge in robotics where simulation-learned policies often fail on real hardware.
- **Evolutionary robotics and neuroevolution**: Uses evolutionary search, but applied to program synthesis rather than neural network weights.
- **Failure recovery and robust control**: Builds on control theory's emphasis on failure modes, now applied to learned policies.

## Practical Applications
1. **Manufacturing and assembly lines**: Robots discovering skills for complex multi-step assembly tasks (grasping, insertion, fastening) with automatic recovery from part variations.
2. **Household robotics**: Long-horizon task execution (tidying, organizing, food prep) where manual programming per environment is infeasible.
3. **Medical robotics**: Autonomous discovery of manipulation skills for delicate surgical or rehabilitation tasks where failures are costly.
4. **Space robotics**: Robots on Mars or orbital platforms automatically developing skills for sample collection and equipment maintenance without real-time human teleoperation.
5. **Warehouse automation**: Bin picking and depalletization with adaptive handling for diverse object shapes and weights.

## Product & Startup Ideas
1. **Robot skill marketplace platform**: A cloud service where robots upload discovered skills (e.g., "robust grasping for flexible objects," "compliant insertion") that other users can license and adapt. Revenue via subscription or per-skill licensing.
2. **Autonomous robot commissioning service**: Deploy ASPIRE to factories as a service—send the system a new robot and task description, get a validated control library back in days instead of weeks of engineering. Charge per deployment or per robot-hour of autonomously discovered capability.
3. **Sim-to-real validation-as-a-service**: Offer enterprises a platform to safely explore skill discovery in simulation, with guaranteed transfer to their specific real robot hardware. Reduce real-robot trial-and-error costs.
4. **Robot skill OS (operating system)**: A unified skill library and runtime layer that lets any robot access discovered skills—think "Git for robot behaviors." Monetize via enterprise licensing and premium curated skill packs.
5. **Failure diagnosis and repair toolkit for roboticists**: Standalone tool that integrates ASPIRE's failure analysis and code-generation capabilities into existing robot development workflows. Sell as developer toolkit or SaaS for small robotics companies.

## Technical Limitations
1. **Scalability of code synthesis**: Generating and validating diverse control programs could become computationally expensive at scale; no analysis of wall-clock time or sample efficiency is mentioned.
2. **Multimodal trace quality**: Depends on having rich, aligned sensor data (vision, force, proprioception). Robots with poor or missing sensors will struggle.
3. **Simulation fidelity**: Sim-to-real transfer assumes simulation captures relevant physics (contact dynamics, friction, deformation). Sufficiently different simulators or robots may fail.
4. **Long-horizon credit assignment**: For failure diagnosis in long tasks, pinpointing which action caused which failure becomes harder; the abstract doesn't clarify how this is handled.
5. **Generalization beyond training distribution**: While tested on diverse tasks, there's no clear boundary on what kinds of novel tasks the accumulated library can solve—likely fails on truly out-of-distribution scenarios.
6. **Embodiment diversity**: Works across embodiments per the abstract, but practical limits (e.g., very different DOF, actuation types) aren't specified.

## Open Questions
1. **How does ASPIRE scale with skill library size?** Does retrieval and composition of skills remain efficient as the library grows to thousands of primitives?
2. **What is the sample complexity?** How many real robot trials are needed to bootstrap ASPIRE? Can it learn from simulation alone, or does real-world interaction remain essential?
3. **How are skills composed for novel tasks?** Can ASPIRE reliably chain learned skills, or does it reinvent the wheel each time?
4. **What are the failure modes of failure diagnosis?** When does the system misdiagnose the cause of failure and synthesize ineffective fixes?
5. **How much domain knowledge is embedded in the evolutionary search?** Is the task sequence generation truly open-ended, or does it require hand-crafted curricula?
6. **Can this approach scale to manipulation tasks requiring object learning?** E.g., adapting to new object categories not seen during skill discovery.

## Key Takeaway
ASPIRE automates the discovery and refinement of robot skills through closed-loop learning and code generation, building persistent, reusable libraries that transfer across tasks and robots—a major step toward reducing manual robot programming effort.
