# SGA: Plug&Play Geometric Verification for Educational Video Synthesis

**Authors:** Lopez Jhon, Hinojosa Carlos, Ghanem Bernard
**Published:** 2026-07-20 | **arXiv:** [2607.18116v1](http://arxiv.org/abs/2607.18116v1)

## Summary
This paper addresses a real problem in AI-generated educational videos: when LLMs write code to create animated math lessons (using Manim), the resulting animations often have objects overlapping or blocking each other, making them hard to understand. The authors propose SGA, a "plug-and-play" quality-control module that catches these geometric mistakes before rendering and fixes them automatically. They also introduce MVQS, a fast way to score animation quality without actually rendering the video.

## Theoretical Background
Recent approaches use LLMs to generate executable animation code for pedagogy — this is a natural extension of code generation capabilities of modern language models. Manim is a Python library for creating mathematical animations programmatically. The core challenge is that LLMs generate code that is syntactically correct but spatially incoherent — two objects end up in the same location, or one hides another. Prior work on pedagogical animation has focused on content accuracy but largely ignored geometric correctness. Scene graphs (symbolic representations of what objects exist and where) are a classical computer graphics concept. The paper bridges these domains: using LLM code generation + scene graph extraction + spatial conflict detection.

## Core Contributions
1. **SGA module**: A trainable agent that intercepts LLM-generated animation code, partially executes it to build symbolic scene graphs (without full rendering), detects spatial conflicts, and refines problematic code — all without retraining the underlying LLM.
2. **MVQS metric**: A deterministic, rendering-free proxy for animation spatial quality that correlates with visual legibility, enabling fast iteration.
3. **Empirical validation**: Systematic evaluation across 4 LLM backbones and 2 agentic pipelines on the MMMC-Code benchmark, showing consistent 16.1% relative MVQS improvement.

## Connections to Other Work
This relates to [[RLHF]] and broader work on post-hoc correction of LLM outputs — instead of retraining, SGA acts as a verifier/refiner layer. It echoes ideas from [[mechanistic-interpretability]] in that it inspects intermediate model states (extracted scene graphs). The geometric verification step is reminiscent of formal verification pipelines in code generation (e.g., [[Program Synthesis]]). The agent architecture (intercept → analyze → refine) parallels recent work on agentic code generation and self-correction loops in multi-turn reasoning.

## Practical Applications
- **Online education platforms** (Khan Academy, Coursera): auto-generate cleaner pedagogical animations from course content without manual animation labor.
- **Textbook publishing**: automatically convert mathematical proofs/derivations into animated explanations with guaranteed visual clarity.
- **Accessibility in STEM**: generate high-quality animated explanations for students with different learning preferences.
- **Real-time tutoring systems**: produce on-the-fly animated walkthroughs of student problem solutions.

2. **AnimationQA SaaS**: Upload a textbook chapter or lecture script; get back a set of correctly-rendered pedagogical animations without hiring animators.

## Technical Limitations
- **Limited to Manim domain**: The approach is tightly integrated with Manim's API; generalization to other animation libraries (Three.js, P5.js, etc.) is unclear.
- **Scene graph complexity**: As animations grow more complex (many objects, transformations), scene graph extraction may become brittle or computationally expensive.
- **Proxy metric weakness**: MVQS is rendering-free but may not capture all visual quality issues (e.g., temporal pacing, readability of labels, color contrast). Correlation to human preference is not measured.
- **Refinement cost**: The paper doesn't quantify wall-clock time for SGA's partial execution and refinement loop; could be slow for very long animations.
- **Assumes LLM generates parseable code**: If the base LLM's code is syntactically broken or deeply incoherent, SGA cannot recover.

## Open Questions
- How does SGA scale to animations with 50+ objects or recursive structures?
- Can the refinement strategy be learned end-to-end (e.g., via RL) rather than rule-based?
- Does MVQS predict human pedagogical effectiveness (do students actually learn better from SGA-corrected animations)?
- Can SGA generalize to other animation libraries (Blender, Three.js, etc.) with minimal retraining?
- How sensitive is the approach to different LLM architectures and instruction-tuning methods?

## Key Takeaway
LLM-generated educational animations have geometry bugs; this paper's agent module catches and fixes them automatically, improving quality 16% without retraining the LLM.
