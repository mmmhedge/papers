# Program Synthesis for Simulation-Based Inference: Joint Model Selection and Parameter Estimation

**Authors:** Siddharth Mishra-Sharma
**Published:** 2026-07-20 | **arXiv:** [2607.17540v1](http://arxiv.org/abs/2607.17540v1)

## Summary
This paper combines large language models with neural simulation-based inference to automatically discover which model structure best fits data, not just estimate parameters within a fixed model. Instead of requiring users to hand-code a simulator, an LLM writes candidate simulator programs that are iteratively refined and evaluated against observations. The work matters because real-world inference often requires choosing between competing model families (like different epidemic models or physics theories), and automating this choice could accelerate scientific discovery.

## Theoretical Background
Simulation-based inference uses neural density estimation to infer parameters when the likelihood function is intractable — common in fields like epidemiology and cosmology where simulators are expensive but observations are available. Traditional approaches fix a simulator upfront and only estimate its parameters. Large language models have recently shown the ability to generate executable code from natural language descriptions. This paper bridges these: LLMs propose simulator implementations, and neural inference methods evaluate which ones match observed data. Neural density estimation networks (trained with techniques like normalizing flows or score-based models) learn to estimate probability distributions from simulated data, enabling amortized inference across many parameter combinations at once.

## Core Contributions
1. A framework that jointly performs model selection (choosing simulator structure) and parameter estimation in a unified neural inference pipeline — not a sequential two-step process.
2. A feedback-driven mutation strategy to iteratively refine LLM-generated simulator code based on how well candidate models explain data.
3. Empirical validation across three diverse domains (deterministic dynamics, stochastic epidemic models, dark matter lensing) showing the method can identify correct model families from open-ended natural language prompts.
4. Demonstration that inference accuracy naturally reflects both the information content of observations and the identifiability of competing models.

## Connections to Other Work
This work sits at the intersection of [[neural density estimation]] for simulation-based inference and recent advances in [[code generation with LLMs]]. It relates to broader program synthesis literature, which has explored using neural feedback to guide code generation (e.g., reinforcement learning from execution). The paper implicitly addresses model comparison problems traditionally tackled via Bayesian model evidence calculations, but here uses amortized neural estimation instead. It connects to work on automated scientific discovery and symbolic regression, though here the search space is simulator structures rather than symbolic equations.

## Practical Applications
- **Drug discovery pipelines**: Automatically select pharmacokinetic/pharmacodynamic model structures when fitting clinical trial data, eliminating tedious model-comparison workflows.
- **Climate modeling**: Discover plausible subgrid parameterization schemes from observational data without manual equation engineering.
- **Financial risk modeling**: Automatically propose stochastic volatility model variants and select the best-fit family for a given asset class or market regime.
- **Epidemiology**: During disease outbreak response, rapidly identify whether transmission follows simple SIR dynamics, super-spreading, or other variants from early case data.
- **Cosmology/physics**: Infer dark matter distribution or particle interaction models from imaging data without hand-coding multiple simulators.

## Product & Startup Ideas
1. **AutoInference SaaS**: A platform where domain experts upload data and natural language model descriptions; the system auto-generates simulators, runs inference, and returns ranked candidate models with uncertainty quantification — target: pharmaceutical companies, materials science labs.
2. **Scientific Model Studio**: IDE for scientists combining an LLM-powered simulator code generator, an interactive visualization dashboard for model diagnostics, and automatic hyperparameter tuning for the neural inference backend.
3. **Automated Calibration Engine**: Embedded inference tool for engineering/manufacturing companies to automatically calibrate and select simulation models (fluid dynamics, structural, thermal) from real sensor data on production lines.
4. **Hypothesis Testing API**: REST endpoint that takes a system description, data, and candidate model sketches (in natural language), returns Bayesian model weights and parameter posteriors — enables data-driven experimental design for researchers.
5. **Finance Model Discovery**: Tool for quant teams that auto-generates and backtests alternative market microstructure or asset pricing model variants against historical tick data, surfacing overlooked model classes.

## Technical Limitations
- **LLM reliability**: The approach depends on an LLM producing syntactically correct, semantically sensible simulators; malformed code or physically implausible proposals could waste compute. Robustness to prompt variation is unclear.
- **Scalability of search**: No analysis of how search over model space scales with problem dimensionality. Mutation strategies may not scale to hundreds of candidate models or high-dimensional parameter spaces.
- **Simulator cost**: The method requires evaluating many simulator variants on many parameter draws; intractable for extremely expensive simulations (e.g., large-scale climate runs).
- **Identifiability assumptions**: Accuracy claims rely on models being identifiable from data; the paper shows this works empirically but doesn't guarantee identifiability in principle.
- **Limited to simulation-based models**: Requires a user to provide a domain description amenable to LLM code generation; doesn't handle purely black-box observational systems where no simulator exists.
- **Evaluation bias**: Benchmarks span relatively well-understood domains; unclear how well the method generalizes to truly novel or adversarial model spaces.

## Open Questions
- How does performance degrade as the number of candidate model families grows? Is there a computational scaling wall?
- Can the framework detect when *no* model in the candidate set explains the data, versus settling on a poor-fit model?
- How sensitive is the method to the quality and specificity of natural language prompts? Can it recover from vague or misleading problem descriptions?
- Does the mutation strategy learn to explore the model space efficiently, or does it require careful tuning for each domain?
- How does this compare (in speed and accuracy) to traditional Bayesian model comparison on the same benchmarks?
- Can the framework propose *novel* model structures not explicitly suggested in the initial prompt, or does it remain constrained to LLM variations on seed ideas?

## Key Takeaway
LLMs can auto-generate simulator code, and neural inference can pick the best model family from data without hand-coding each candidate.
