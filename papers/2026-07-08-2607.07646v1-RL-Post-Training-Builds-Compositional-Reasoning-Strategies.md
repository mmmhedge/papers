# RL Post-Training Builds Compositional Reasoning Strategies

**Authors:** Azwar Abdulsalam, Nishil Patel, Andrew Saxe
**Published:** 2026-07-08 | **arXiv:** [2607.07646v1](http://arxiv.org/abs/2607.07646v1)

## Summary
This paper investigates whether reinforcement learning post-training can teach language models to discover genuinely new strategies by composing simpler skills, rather than just amplifying existing abilities. Using a controlled environment where all model outputs can be inspected, the authors show that RL discovers and reuses complex multi-step procedures that the base model rarely finds, while rejection fine-tuning (a simpler alternative) produces mostly invalid shortcuts.

## Theoretical Background
The paper builds on two key ideas: (1) the hypothesis that neural networks contain latent compositional structure that training can unlock, related to work on [[compositionality in neural networks]], and (2) reinforcement learning for language models, which typically uses sparse reward signals to guide exploration — in this case, a single binary "correct/incorrect" judgment rather than dense per-token feedback. The authors use a rewrite-grammar environment, a formal system where problems are solved by applying symbol-rewriting rules in sequence, which allows perfect observability and auditability — unlike open-ended language tasks. This setup connects to program synthesis and mechanistic interpretability (the study of what internal algorithms neural networks learn).

## Core Contributions
1. **Empirical demonstration that RL builds compositional strategies**: RL post-training solves held-out problems that remain rare even with much larger sampling from the base model, proving the emergence of new capability, not just amplification.
2. **Mechanistic characterization of the learning process**: By inspecting every rewrite trace, the authors show RL follows a phased mechanism: first strengthening primitive reductions, then discovering and consolidating composed procedures (both sequential and parallel).
3. **Key insight on selectivity vs. exploration volume**: The difference between RL and rejection fine-tuning is not how much the model explores, but *which outputs it reinforces* — RL concentrates on valid, reusable structure while RFT produces scattered invalid shortcuts.
4. **Gating by procedural organization, not skill exposure**: Pretraining ablations reveal that merely exposing the model to primitives is insufficient; pretraining must organize those primitives into procedural form for RL to compress them into higher-level strategies.

## Connections to Other Work
This work relates to [[program synthesis]] and [[neural program induction]], which also study how networks learn compositional algorithms. It connects to [[mechanistic interpretability]] research (e.g., work on circuits and causal tracing) by using manual trace analysis to reverse-engineer learned strategies. The finding that RL's strength lies in *selectivity* complements recent work on [[rejection sampling]] vs. active learning trade-offs. The phased learning (primitives → compositions) echoes hierarchical RL and curriculum learning. The work also speaks to debates about whether [[scaling]] (larger models/data) or better learning algorithms (like RL) unlock capability — here, RL finds structures that scale alone does not.

## Practical Applications
1. **Improving code generation and synthesis tools**: RL post-training could teach code-generating models to discover and reuse multi-line algorithms or library functions, boosting success on unseen problem classes.
2. **Mathematical reasoning**: Post-training reasoning models (like those used in competition mathematics) with RL could help them compose lemmas and proof techniques into novel high-level strategies.
3. **Query optimization**: Database systems could use compositional RL to discover valid rewrite rules that fold multiple query transformations into efficient single steps.
4. **Optimization and constraint satisfaction**: RL could train solvers to compose primitive constraint checks and reduction steps into domain-specific heuristics.

## Product & Startup Ideas
1. **RL-guided program synthesis platform**: A tool that takes user-provided primitive operations/APIs and uses RL to auto-discover and rank higher-level abstraction layers for code generation, reducing manual library design.
2. **Compositional theorem prover**: An IDE for formal math/proof writing that learns from user-proved lemmas via RL to automatically compose and suggest multi-step proof strategies on new problems.
3. **Automated optimization kernel library**: A service that analyzes domain-specific algorithms, uses RL to discover compositional rewrite rules, and synthesizes optimized code for custom hardware or constraints.
4. **RL-enhanced query optimizer**: A database layer that post-trains on customer-specific query logs to learn compositional rewrites, dynamically discovering domain-tailored optimization strategies.
5. **Workflow automation engine**: An RPA (robotic process automation) system that learns to compose primitive UI actions into higher-level multi-step procedures via RL, reducing manual scripting.

## Technical Limitations
1. **Requires fully observable, auditable environments**: The core insight depends on inspecting every trace — this is infeasible for open-ended language tasks, multimodal problems, or real-world environments with latent state. Generalizing to black-box reward functions is unclear.
2. **Sparse reward signal**: The paper uses only binary final-answer feedback. Performance on tasks requiring denser intermediate rewards, or where the final answer is ambiguous, is unexplored.
3. **Grammar structure is hand-designed**: The rewrite grammar is provided; the paper does not address discovering grammar rules or decompositions *de novo*. Real compositional structure in language is not pre-provided.
4. **Small-scale evaluation**: Tested on symbol rewriting in a controlled domain. Scaling to realistic language tasks, multi-agent environments, or tasks with continuous state spaces is unaddressed.
5. **Comparison baseline limitations**: Rejection fine-tuning is the only baseline; comparisons to curriculum learning, hierarchical RL, or other compositional learning methods are absent.
6. **Pretraining dependence**: The paper shows composition depends on how pretraining organizes primitives, but does not provide a recipe for designing pretrain distributions that set up RL success.

## Open Questions
1. **Can this scale to natural language?** How much of the insight transfers when grammar rules, reduction steps, and compositionality are implicit (not auditable) in language?
2. **What pretraining curriculum maximizes RL compositionality?** Can we derive principles for designing pretraining to set up RL for compositional discovery in target domains?
3. **Do composed procedures generalize beyond held-out problems?** Do RL-learned strategies transfer to longer chains, out-of-distribution problem structures, or new domains?
4. **Can RL discover compositional structure in environments with partial observability or noisy rewards?** How does trace visibility and reward density affect the learning phases?
5. **How does model scale affect composition emergence?** Do larger models discover different or more complex compositional strategies under the same RL regime?
6. **Can we predict compositionality from pretraining traces?** Is there a metric on the base model's behavior that predicts whether RL will successfully compose?

## Key Takeaway
RL post-training does not merely amplify base model skills—it actively reorganizes primitive competence into novel, stable, high-level strategies through a learnable phased mechanism, but only when pretraining has structured those primitives as reusable procedures rather than isolated examples.
