# Sound Probabilistic Safety Bounds for Large Language Models

**Authors:** Mahdi Nazeri, Anne-Kathrin Schmuck, Sadegh Soudjani, Alessandro Abate
**Published:** 2026-07-22 | **arXiv:** [2607.20286v1](http://arxiv.org/abs/2607.20286v1)

## Summary
This paper develops a mathematical framework to rigorously measure how often large language models (LLMs) generate harmful outputs. The core innovation is using Clopper-Pearson confidence intervals — a classical statistical tool — combined with a smart search algorithm to efficiently compute provably-correct lower bounds on harm probability, even when the true probability is extremely small. This matters because it enables formal safety certification of LLMs, moving beyond ad-hoc testing.

## Theoretical Background
The work builds on two foundational ideas: **Clopper-Pearson confidence intervals** (a frequentist statistical method from 1934 that gives rigorous bounds on unknown probabilities from samples) and **probably approximately correct (PAC) learning** (a framework from computational learning theory that provides formal guarantees about generalization). The authors apply these classical tools to a new problem: estimating the probability an autoregressive LLM — which generates text token-by-token, forming a tree of possible outputs — produces harmful content. The challenge is that this tree is exponentially large, so naive sampling would require infeasible numbers of forward passes.

## Core Contributions
1. **Novel PAC framework for LLM harm bounds**: First rigorous application of Clopper-Pearson intervals to formally certify LLM safety (lower bounds are provably sound — guaranteed not to exceed true harm probability).
2. **Latent-space-guided branching algorithm**: An algorithm that prioritizes exploring high-harm branches in the autoregressive generation tree using learned features in the model's latent space, dramatically improving sample efficiency.
3. **Formal guarantees for extreme probabilities**: Demonstrates non-trivial lower bounds even when true harm probability is vanishingly small — a regime where naive approaches fail.
4. **Empirical validation**: Tests on state-of-the-art LLMs show the method produces useful certified bounds in practice.

## Practical Applications
- **AI safety auditing**: Regulatory and internal teams could formally certify LLM deployments meet safety thresholds before release.
- **Model comparison**: Enables quantitative safety ranking of different LLMs or fine-tuned variants.
- **Targeted harm reduction**: Identifies which harmful branches the model is most likely to follow, enabling focused mitigation.
- **Insurance and liability**: Provides defensible probabilistic guarantees for high-stakes applications (healthcare, finance, legal).
- **Iterative safety improvement**: Measure whether safety-focused fine-tuning actually reduces harm probability with formal rigor.

## Product & Startup Ideas
1. **LLM Safety Certification Service**: A SaaS platform that runs this algorithm on customer LLMs and issues formal safety reports with numerical bounds (analogous to code security scanning).
2. **Safety Dashboard for Model Developers**: Real-time tool tracking how fine-tuning or RLHF updates change harm probability bounds, integrated into model development pipelines.
3. **Regulatory Compliance Toolkit**: Package for enterprises to automatically audit internal LLM deployments and generate audit trails for compliance (GDPR, AI Act, etc.).
4. **Comparative Safety Benchmarking**: Public leaderboard showing certified harm probabilities for popular open-source LLMs (e.g., Llama, Mistral, etc.).
5. **Safety-Gated API Wrapper**: Middleware service that intercepts LLM API calls, estimates harm risk using this method, and blocks high-risk outputs with formal justification.

## Technical Limitations
- **Computational cost**: Even with smart branching, exploring enough paths to get tight bounds on very safe models may require many forward passes; scalability to extremely large models is unclear.
- **Latent space dependency**: The algorithm relies on learned features in latent space being good predictors of harm; this assumes the model's internal representations correlate with harm — may fail for adversarial or deceptive models.
- **Harm definition**: Requires a clear, consistent harm classifier/oracle; defining "harm" is itself contested and context-dependent (harm differs by use case, culture, etc.).
- **Lower bounds only**: The method computes lower bounds on harm probability, not the true value; actual harm may be higher, limiting precision.
- **Autoregressive generation assumption**: Designed for left-to-right token generation; doesn't directly apply to other architectures (e.g., diffusion models, VAEs).
- **No adaptation to new prompts**: Bounds are prompt-specific; no transfer across different input distributions.

## Open Questions
- How tight can lower bounds become with more samples? Can the method achieve high precision for real-world safety thresholds?
- How sensitive is the method to the choice of harm oracle/classifier? What happens if the classifier itself is noisy or biased?
- Can this framework extend to **conditional harm** (e.g., harm only when a certain jailbreak is used)?
- Does the latent-space guidance scale to multimodal models or very large models (>100B parameters)?
- How do bounds change after safety interventions (RLHF, fine-tuning, prompting)? Can you attribute changes to specific training steps?
- Can you combine this with other uncertainty quantification methods (e.g., Bayesian approaches) to get both lower *and* upper bounds?

## Key Takeaway
Formal probabilistic guarantees for LLM safety enable rigorous certification where red-teaming alone cannot.
