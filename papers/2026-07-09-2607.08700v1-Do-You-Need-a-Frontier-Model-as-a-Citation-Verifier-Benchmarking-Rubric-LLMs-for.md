# Do You Need a Frontier Model as a Citation Verifier? Benchmarking Rubric LLMs for Deep-Research Source Attribution

**Authors:** Ethan Leung, Elias Lumer, Corey Feld, Austin Huber, Vamse Kumar Subbiah et al.
**Published:** 2026-07-09 | **arXiv:** [2607.08700v1](http://arxiv.org/abs/2607.08700v1)

## Summary
This paper investigates whether expensive frontier LLMs are necessary to judge citation quality in AI research systems—or whether cheaper models work just as well. The authors benchmark 8 LLMs as "judges" that evaluate whether citations support claims, finding that smaller models like GPT-4o-mini perform competitively, but all models show hidden biases (false positive/negative rates) that scalar metrics like F1 scores hide. This matters because these judges are used as reward signals in reinforcement learning training, so their hidden biases get baked into the AI system itself.

## Theoretical Background
This work sits at the intersection of three areas: (1) **reward modeling** in reinforcement learning — the idea that an external judge (reward model) scores agent outputs to guide training, (2) **rubric-based evaluation** — breaking complex judgments into structured dimensions that can be scored objectively, and (3) **long-form language generation** in grounded systems, where LLMs must cite sources for their claims. The paper builds on prior work showing that LLM-as-judge systems can introduce subtle biases, and extends this to understand how those biases propagate through RL loops. The citation quality task itself is structured: two independent dimensions (source relevance and factual support) are judged on a binary rubric.

## Core Contributions
1. **Calibration analysis of LLM judges for citation rubrics**: Systematically compares 8 LLM judges on citation quality evaluation, with human-annotated gold labels and 378 adversarial "hard cases" where human judges disagreed.
2. **Exposes hidden bias in cheap judges**: Shows that while cheaper models (GPT-4o-mini, Claude-3.5-Haiku) match frontier models on F1 score, they exhibit substantially different false positive rates, false negative rates, and pass-rate drift—directional biases that aggregate metrics mask.
3. **Demonstrates that no single model dominates**: On factual support dimension, judges are statistically indistinguishable (overlapping confidence intervals), contradicting the assumption that "bigger = better."
4. **Establishes calibration as a prerequisite for RL reward signals**: Argues that using rubric-based judges as reward models requires explicit bias accounting, not just F1 reporting.

## Connections to Other Work
This extends the [[RLHF]] (reinforcement learning from human feedback) literature by raising a practical calibration question: when humans are replaced by LLM judges, what gets lost? It relates to broader work on **reward model misalignment** and **specification gaming** — cheaper judges might optimize for proxy metrics rather than true quality. The adversarial evaluation approach echoes recent work on red-teaming frontier models and measuring robustness to distribution shift. It also connects to [[Alignment]] concerns: if reward models encode subtle biases, those biases become embedded in trained policies.

## Practical Applications
1. **Search-grounded AI research assistants** (Perplexity, Claude's research features, etc.): Need to decide which LLM to use for fact-checking claims. This work shows cheaper options work, reducing operational costs by 10–100× without quality loss on surface metrics.
2. **Content verification platforms**: Services that audit AI-generated research or journalism could use cheaper judges to scale audits, provided they account for discovered biases.
3. **Synthetic data quality control**: Organizations generating training data via LLMs can use cheaper judges for attribution verification during data pipeline steps.
4. **Knowledge base construction**: Systems that automatically cite sources for knowledge graph entries can calibrate their citation judges to avoid systematic false positives (spurious citations) or false negatives (missed citations).

## Product & Startup Ideas
1. **Citation Quality Auditing SaaS**: A service that analyzes an existing LLM system's citation judge, measures its directional biases, and recommends which judge to use or how to post-hoc calibrate outputs. Charge per audit or per 1M citations checked.
2. **Bias-Aware Judge Ensemble**: Package that combines multiple cheaper LLM judges with learned confidence weights, beating any single model while minimizing cost. Sell as an API or library.
3. **Rubric Calibration Toolkit**: Developer tool that, given a new rubric task, automatically benchmarks candidate judges and generates a report on hidden biases. Target research teams and fact-check platforms.
4. **RL Training Sanity Checker**: Pre-training service that validates a proposed reward model before it's used in RL loops. Detects and flags systematic biases that would otherwise corrupt policy learning.
5. **Cost-Optimized Judge Router**: Intelligent system that routes different citations to different judges (cheap for easy cases, expensive for ambiguous ones), maximizing calibration while minimizing cost.

## Technical Limitations
1. **Limited rubric scope**: Only evaluates citation quality on a long-form research benchmark. Results may not generalize to other rubric tasks (code review, safety red-teaming, etc.).
2. **Gold label quality**: Human reviewers themselves can be biased; the "gold standard" is only as good as the adjudication process. No inter-annotator agreement stats provided for the gold labels themselves.
3. **No analysis of why biases exist**: The paper identifies directional bias (false positive vs. negative rates) but doesn't diagnose *why* cheaper models drift differently—is it prompt sensitivity, training data, or architectural differences?
4. **Static evaluation**: Judges are benchmarked once; doesn't test whether biases shift with prompt rephrasing, system message changes, or few-shot examples.
5. **Downstream RL outcome missing**: Paper doesn't train an actual RL agent using biased judges and measure whether the predicted bias *actually* corrupts the learned policy in practice.
6. **Assumes citation dimension independence**: Treats source relevance and factual support as independent, but they likely interact (irrelevant sources can't provide factual support).

## Open Questions
1. **Why do cheap models exhibit different bias profiles?** Are there simple prompt edits or few-shot examples that can align cheaper judges to frontier model behavior?
2. **Can bias be corrected post-hoc?** If you identify a judge's false positive rate, can you apply a learned calibration curve to outputs without retraining?
3. **How do these biases propagate through RL?** Run actual RL experiments: train agents with biased vs. calibrated judges and measure downstream task performance. Does F1 gap translate to policy quality gap?
4. **Do biases generalize across domains?** Test whether a judge's bias profile learned on research citations holds for other citation-heavy tasks (legal briefs, medical records, etc.).
5. **Multi-judge aggregation**: What's the optimal way to combine multiple cheaper judges to get both calibration and cost savings? Simple voting? Learned ensemble weights?
6. **Temporal stability**: Do judge biases drift over time or with model updates? Can you predict when recalibration is needed?

## Key Takeaway
Before using an LLM judge as a reward signal in reinforcement learning, measure not just accuracy (F1) but also directional bias (false positive/negative rates)—because metrics like F1 hide the biases that RL will amplify.
