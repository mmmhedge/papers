# PalmClaw: A Native On-Device Agent Framework for Mobile Phones

**Authors:** Hongru Cai, Yongqi Li, Ran Wei, Wenjie Li
**Published:** 2026-07-14 | **arXiv:** [2607.13027v1](http://arxiv.org/abs/2607.13027v1)

## Summary
PalmClaw is a framework that runs AI agents directly on mobile phones, letting them control device features and complete multi-step tasks without sending data to servers. Unlike existing mobile agents that click buttons and swipe screens, PalmClaw exposes device capabilities (like contacts, calendar, sensors) as explicit tools with clear inputs and outputs, achieving 11.5% better task success and 95% faster execution.

## Theoretical Background
This work builds on recent advances in [[LLM agents]] — systems where large language models don't just generate text but act as task executors by calling external tools, observing results, and deciding next steps. Prior mobile agent work relied on GUI automation (mimicking human touches and swipes), which requires learning long sequences of interface-dependent actions. PalmClaw shifts the paradigm by treating device capabilities as first-class tools with structured APIs, similar to how desktop agent frameworks (like ReAct or tool-using agents) expose external APIs. The core insight is that mobile devices have rich native APIs that agents can call directly, bypassing the need to interpret visual interfaces.

## Core Contributions
- **Native on-device execution**: The agent loop (planning, tool calls, observation, re-planning) runs entirely on the phone, not on a server, reducing latency and privacy exposure.
- **Device tools abstraction**: Exposes mobile capabilities (contacts, calendar, camera, sensors, installed apps) as explicit tools with defined arguments and structured results, rather than raw GUI actions.
- **Execution boundaries**: Each tool call has clear pre/post-conditions and success/failure signals, making agent behavior predictable and debuggeable.
- **Significant performance gains**: 11.5% relative improvement in task success rate and 94.9% reduction in task completion time over the strongest baseline (GUI-automation approaches).

## Connections to Other Work
PalmClaw relates to the broader [[agents]] landscape: it shares the multi-step reasoning and tool-use pattern of [[ReAct]] and similar frameworks, but applies them to an under-explored domain (on-device mobile execution). It contrasts with mobile GUI agents (e.g., those using computer vision to parse screens) by moving away from pixel-level reasoning. The work also connects to edge AI and privacy-preserving ML — keeping the agent loop on-device avoids centralizing user data. It complements [[RLHF]] and alignment research by focusing on task execution rather than language generation safety.

## Practical Applications
- **Mobile automation**: Users ask their phone to "reschedule all meetings next Tuesday" or "send a summary email of today's messages" — the agent reads calendar/messages and executes directly.
- **Accessibility**: Voice-controlled agents help users with motor or visual impairments navigate apps and settings by interpreting voice commands into device tool calls.
- **Smart home integration**: Agents on phones can orchestrate connected devices (lights, thermostats, locks) by calling device APIs instead of tapping apps repeatedly.
- **Offline task completion**: On a flight or in areas without connectivity, agents can still complete calendar, note-taking, and local-data tasks without server calls.
- **Privacy-sensitive use**: Medical or financial data stays on-device; the agent processes it locally and only shares results the user approves.

## Product & Startup Ideas
1. **Personal assistant OS layer**: A lightweight agent runtime that manufacturers pre-install; users voice-control everyday tasks (scheduling, messaging, payments) with on-device execution and optional cloud offload for heavy LLM reasoning.
2. **Mobile automation marketplace**: Curated, open-source skill library for PalmClaw; users discover and install agent skills for domain-specific tasks (fitness tracking, expense splitting, HR workflows) without writing code.
3. **Privacy-first enterprise mobile MDM**: Device-management platform for companies using on-device agents to automate HR, IT, and compliance tasks while ensuring sensitive data never leaves the phone.
4. **Accessibility-first voice AI**: Mobile app that runs PalmClaw locally to provide hyper-responsive, offline voice control for people with disabilities, without latency or privacy concerns of cloud speech services.
5. **Cross-device agent sync**: Platform that runs complementary PalmClaw instances on phone, tablet, and laptop, sharing state and memory so agents coordinate tasks across user's ecosystem (e.g., compose draft on phone, finalize on desktop).

## Technical Limitations
- **Model size constraints**: Running LLM inference on-device requires small, quantized models; very large or reasoning-heavy LLMs (e.g., GPT-4) may not fit or run fast enough, limiting agent sophistication.
- **Tool coverage**: The framework is only as good as its device tool library; unsupported or third-party app integrations require custom tool definitions, which is friction for users.
- **Memory and context**: Mobile devices have limited RAM; long task histories or complex multi-turn reasoning may exhaust context windows or cause slowdowns.
- **Firmware and permission fragmentation**: Android and iOS have different APIs, permission models, and OS versions; maintaining a unified abstraction across devices is non-trivial.
- **Evaluation scope**: The paper tests on a specific set of mobile tasks; generalization to novel devices, custom ROMs, or enterprise-managed phones is unclear.
- **Error recovery**: When a tool call fails (e.g., due to permission denial or network hiccup), the paper doesn't detail how agents recover or escalate to the user.

## Open Questions
- How does PalmClaw scale to agents that need to reason over very long task histories or coordinate across many devices?
- Can the framework automatically generate tool definitions for third-party apps, or does it require manual SDK integration?
- What are the thermal and battery impacts of running continuous agent loops on-device for hours?
- How well do smaller, on-device LLMs perform compared to larger cloud models for complex reasoning tasks?
- Can PalmClaw agents learn and adapt from user feedback or task traces without retraining the base model?
- How does the framework handle adversarial or malicious tool calls (e.g., an agent trying to access sensitive data without user consent)?

## Key Takeaway
On-device mobile agents with native API access beat GUI automation by 95× faster execution and cleaner control.
