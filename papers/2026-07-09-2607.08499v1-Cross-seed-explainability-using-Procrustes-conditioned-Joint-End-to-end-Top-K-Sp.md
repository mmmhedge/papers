# Cross-seed explainability using Procrustes-conditioned Joint End-to-end Top-K Sparse Autoencoders

**Authors:** Bendegúz Váradi, Zoltán Kmetty
**Published:** 2026-07-09 | **arXiv:** [2607.08499v1](http://arxiv.org/abs/2607.08499v1)

## Summary
This paper solves a fundamental problem in neural network interpretability: when you train the same model architecture multiple times from different random starting points, the internal features it learns end up in different coordinate systems, making it impossible to compare what they've "learned." The authors develop a method using sparse autoencoders (SAEs) and geometric alignment to extract universal, interpretable features that stay consistent across independently trained BERT models, validated on three text classification tasks.

## Theoretical Background
Mechanistic interpretability aims to reverse-engineer what individual neurons or features in neural networks represent. Sparse autoencoders (SAEs) are a dictionary learning technique that decomposes neural activations into interpretable sparse components. A key obstacle is the alignment problem: dictionary learning optimizes a non-convex objective, so two independently trained SAEs produce feature spaces rotated relative to each other—mathematically, they differ by an orthogonal transformation. The Procrustes problem solves exactly this: finding the best rotation matrix to align two point clouds in high-dimensional space. BERT is a transformer-based language model trained on masked language modeling; transformers use self-attention mechanisms to process sequential text.

## Core Contributions
1. **Procrustes-conditioned SAE pipeline**: The paper combines orthogonal Procrustes alignment with SAE training to enforce cross-seed feature alignment before joint optimization—not as post-hoc correction but as a conditioning step.
2. **End-to-end downstream objective**: Rather than training SAEs on raw activations alone, they jointly optimize for downstream task performance (classification) while maintaining sparsity.
3. **Dead-feature revival mechanism**: An auxiliary loss that reactivates collapsed sparse dimensions, improving feature coverage and universality.
4. **Systematic validation**: Demonstrates universality (r ≥ 0.70 correlation across seeds) on ten BERT models across three datasets, with qualitative confirmation that aligned features encode sociolinguistic patterns.

## Practical Applications
1. **Model auditing and compliance**: Extract universal safety-relevant features from independently trained language models to verify consistent learned concepts (e.g., detecting bias patterns).
2. **Knowledge distillation**: Identify which features in a large model are "essential" across training runs, then transfer only those to smaller models.
3. **Transfer learning quality assessment**: Compare feature universality across pre-trained models to predict downstream task performance or transferability.
4. **Neuroscience analogy**: Help validate that learned features correspond to replicable linguistic concepts, not artifacts of initialization.

## Product & Startup Ideas
1. **Model Transparency-as-a-Service**: SaaS platform where organizations upload fine-tuned BERT/transformer variants and get a unified feature dashboard showing which interpretable patterns are learned consistently—used for regulatory compliance and trustworthiness certification.
2. **Feature Marketplace**: Curated library of universal, cross-validated features extracted from public models; teams license pre-certified feature dictionaries to skip SAE training.
3. **Ensemble Consistency Checker**: Tool for ML teams to audit whether their multiple model checkpoints learn the same high-level concepts; surfaces misalignment risks before production deployment.
4. **Educational Interpretability Platform**: Interactive visualization tool for students/researchers showing how Procrustes alignment works and what universal features look like across model seeds.
5. **LLM Behavior Prediction**: Predict how a new independently trained model will behave on downstream tasks by comparing its universal feature profile to a reference database.

## Technical Limitations
1. **Computational overhead**: Procrustes alignment + SAE training + end-to-end optimization significantly increases training cost vs. standard SAE pipelines; scalability to larger models (GPT-scale) unclear.
2. **Seed selection bias**: Only five seed pairs tested; results may not generalize to pathological random initializations or very different training hyperparameters.
3. **Linear alignment assumption**: Procrustes assumes an orthogonal transformation is sufficient; non-linear misalignments (e.g., from different optimization dynamics) may persist.
4. **Sparsity-universality tradeoff**: Enforcing sparsity while aligning features may force compromise where neither is optimal; no ablation on sparsity level vs. universality.
5. **Task-specific features**: Features trained on SST-2, Politeness, and TweetEval may not be universal across truly different domains (e.g., scientific or code understanding).
6. **Interpretability validation**: "Sociolinguistic patterns" validated only qualitatively; no systematic human evaluation of whether extracted features are actually interpretable to practitioners.

## Open Questions
1. Does cross-seed universality predict downstream transfer performance? Can you use r ≥ 0.70 as a proxy for feature importance?
2. How does Procrustes-SAE scale to 70B+ parameter models and does alignment remain stable?
3. Can you align features across *different* architectures (e.g., BERT vs. RoBERTa), or is universality only meaningful within-architecture?
4. What happens if you apply this to models trained with different objectives (e.g., MLM vs. contrastive learning)? Is there a fundamental universality limit?
5. Can you reconstruct or predict model outputs *from* universal features alone, validating that alignment preserves task-relevant information?
6. How sensitive is the method to the hyperparameter choices (sparsity coefficient, Procrustes weighting, dead-feature revival strength)?

## Key Takeaway
Orthogonal Procrustes alignment can resolve the feature misalignment problem in mechanistic interpretability, allowing extraction of genuinely universal, interpretable features that replicate across independent model runs.
