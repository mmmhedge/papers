# AgentsCAD: Automated Design for Manufacturing of FDM Parts via Multi-Agent LLM Reasoning and Geometric Feature Recognition

**Authors:** Emmanuel George, Christopher Keefe, Peter Pak, Amir Barati Farimani
**Published:** 2026-07-02 | **arXiv:** [2607.02448v1](http://arxiv.org/abs/2607.02448v1)

## Summary
AgentsCAD is a multi-agent system that uses large language models (LLMs) to automatically fix 3D CAD designs for 3D printing (FDM). The system reads a design file, detects manufacturing problems like steep overhangs, and proposes geometry modifications—reorienting parts, adding fillets, or chamfers—then verifies the fixes visually. This matters because current 3D printing software can identify defects but cannot fix them; automating this repair process saves engineers significant manual work and improves print success rates.

## Theoretical Background
The paper builds on several established areas:

**Design for Additive Manufacturing (DFAM)** — the engineering discipline of modifying designs to be printable by 3D printers; current tools detect failures (overhangs over 45°) but don't propose fixes.

**Boundary-representation (B-Rep) geometry** — a standard CAD format representing 3D shapes as surfaces; STEP files are the industry standard for exchanging B-Rep models.

**Graph neural networks (specifically GraphSAGE)** — neural networks that learn from topology; here trained on MFCAD++ (59,665 CAD parts) to recognize semantic features like holes, pockets, and bosses.

**Multi-agent LLM systems** — architectures where specialized language models (Claude Sonnet, GPT-4o) reason about different subtasks (geometry reasoning, visual verification) and coordinate; this leverages recent advances in treating LLMs as planning and reasoning agents rather than just text generators.

The system bridges a key gap: LLMs understand language and high-level design reasoning, but CAD geometry is numeric and topological; parsing STEP files and constructing face-adjacency graphs lets LLMs "see" the geometry structure.

## Core Contributions
1. **Geometry-to-language translation pipeline**: Converts B-Rep geometry (STEP files) into structured representations (face-adjacency graphs, semantic feature labels) that LLMs can reason about.

2. **Semantic feature recognition at scale**: Fine-tunes GraphSAGE on 59,665 CAD parts (MFCAD++) to automatically label geometric features (fillets, holes, pockets); this enables context-aware repair recommendations, not just generic rules.

3. **Multi-stage agentic workflow**: Coordinates overhang detection, topology parsing, feature labeling, design reasoning (Claude), and visual verification (GPT-4o vision); demonstrates how to chain LLM agents for a complex CAD task.

4. **Visual verification loop**: Uses GPT-4o to inspect rendered views of the proposed modifications, catching cases where text-based reasoning produces invalid geometry—a form of self-correction.

5. **Concrete end-to-end demonstration**: Shows the system working on a real test case (birdhouse), proposing valid modifications and producing both a corrected STEP file and human-readable report.

## Connections to Other Work
This work relates to:

- **[[Large Language Models for Code and Structured Data]]**: Similar to using LLMs for program synthesis, AgentsCAD treats CAD modifications as structured reasoning tasks; the key novelty is handling geometry rather than discrete symbols.

- **[[Vision-Language Models]]**: Uses GPT-4o vision to verify geometry; part of a broader trend of grounding LLMs with visual feedback.

- **[[Graph Neural Networks]]**: The GraphSAGE feature labeling is one approach to semantic understanding; alternative architectures (PointNet, PointNet++) might extract features from point clouds instead.

- **[[Design Automation and Optimization]]**: Related to parametric CAD and generative design, but differs by targeting printability constraints rather than aesthetic or performance optimization.

- **[[Multi-Agent Reinforcement Learning]]**: AgentsCAD doesn't use RL, but the coordination pattern (dispatch work to specialized agents) mirrors multi-agent RL; future work might learn repair policies end-to-end.

The paper does *not* claim to replace human designers, only to automate the tedious and rule-based DFAM step—a pragmatic scope that increases adoption likelihood.

## Practical Applications
1. **In-house 3D printing pipelines for manufacturers**: Automatically pre-processes CAD files before sending to FDM printers, reducing failed prints and post-processing labor (sanding, support removal).

2. **On-demand manufacturing services** (e.g., Shapeways, Formlabs): Automatically correct customer-uploaded designs to meet printability standards, reducing design rejection rates and support tickets.

3. **Supply chain integration**: Embedded in CAD workflows at contract manufacturers; engineers upload designs, the system auto-fixes them, and a report is emailed back for approval.

4. **Educational and research 3D printing labs**: Undergraduates submit designs; the system auto-corrects and explains what was wrong, accelerating learning of DFAM principles.

5. **CAD software plugins**: Integrate directly into Fusion 360, FreeCAD, or SolidWorks as a "Design for 3D Printing" button that runs the agent loop locally or via API.

## Technical Limitations
1. **B-Rep parsing dependency**: Relies on clean, valid STEP files; corrupted or non-manifold geometry may cause failures. No error recovery or partial repair.

2. **Fixed overhang threshold (45°)**: Doesn't adapt to material (PLA vs. nylon) or printer (different support structures); a parameterized approach is missing.

3. **LLM hallucination risk**: Claude or GPT-4o may propose geometrically invalid modifications that slip past the vision verifier; the visual check is not bulletproof for complex cases (e.g., internal voids, thin walls).

4. **Semantic feature labeling limits**: GraphSAGE trained on MFCAC++ may not generalize to designs with novel feature combinations; no uncertainty quantification on predictions.

5. **No mechanical analysis**: Doesn't verify that proposed changes (e.g., thinner walls, repositioning) preserve structural integrity under load; assumes the designer will check.

6. **Reorientation is conservative**: Simply detecting overhangs and proposing reorient; doesn't explore multi-axis repositioning or support strategy optimization.

7. **Scalability unclear**: End-to-end workflow involves multiple LLM calls and vision checks; latency and cost not reported for large assemblies or high-volume batches.

## Open Questions
1. **How well does the GraphSAGE semantic labeling generalize to out-of-distribution CAD?** What failure modes appear on industrial designs not in MFCAD++?

2. **Can the system handle assemblies, not just single parts?** How do inter-part constraints affect repair recommendations?

3. **What is the success rate (first-pass printability) on a held-out test set of real designs?** The birdhouse is one anecdote; statistical validation is needed.

4. **How sensitive is the LLM reasoning to prompt engineering?** Could a better prompt significantly improve repair quality without retraining?

5. **Can the system learn from failed prints?** A feedback loop (user tells the system "this print failed, here's why") might iteratively improve recommendations.

6. **How does cost scale?** Multiple LLM API calls per design; what is the break-even volume versus manual designer time?

7. **Can the system jointly optimize multiple objectives?** E.g., minimize print time and material cost, not just achieve printability.

## Key Takeaway
AgentsCAD demonstrates that multi-agent LLM systems can automate the tedious geometry-to-repair step in 3D printing by parsing CAD topology, recognizing features with neural networks, and reasoning about design fixes—bridging symbolic geometry and natural language reasoning.
