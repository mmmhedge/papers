# Resolving Asynchronous Distributed Knowledge

**Authors:** Philippe Balbiani, Hans van Ditmarsch, Clara Lerouvillois
**Published:** 2026-06-30 | **arXiv:** [2606.31855v1](http://arxiv.org/abs/2606.31855v1)

## Summary
This paper develops a new modal logic for reasoning about knowledge in distributed systems where agents communicate asynchronously — that is, without a shared global clock or common awareness of when information is being shared. The key innovation is adding "resolution modalities" that track which agents know about knowledge-sharing events, even when some agents are unaware of updates involving other agents. This matters because real distributed systems (computers, networks, organizations) operate asynchronously, so this logic better models how knowledge actually spreads in practice.

## Theoretical Background
The paper builds on epistemic modal logic — a formal system for reasoning about "what agents know" using modal operators (roughly: □ means "it is necessarily true that" and ◇ means "it is possibly true that"). Prior work by Agotnes and Wang introduced *Resolving Distributed Knowledge* (RDK), a synchronous logic where all agents observe a shared global sequence of knowledge-sharing events. Distributed knowledge itself means "the group collectively knows X" (combining what each agent individually knows). The new contribution extends RDK to an asynchronous setting where:
- Each agent only observes resolution events (knowledge-sharing announcements) that involve them directly.
- A history of resolutions must be tracked because agents have partial views of past events.
- Truth is evaluated relative to both a world *and* a history, not just a world.

## Core Contributions
1. **Asynchronous resolution semantics**: A history-based semantic framework where agents have incomplete visibility into which knowledge-sharing events have occurred globally.
2. **Revised axiomatization**: The paper shows that synchronous axioms connecting resolution to distributed knowledge break under asynchrony (e.g., learning that a group shared knowledge no longer implies you now know what they collectively know). New axioms must be derived.
3. **Partial observation model**: Agents are unaware of resolutions for groups that do not include them — a realistic constraint absent in prior synchronous work.
4. **Technical machinery for asynchronous epistemic reasoning**: Handling memory-less agents and avoiding contradictions when different agents have different views of the event timeline.

## Connections to Other Work
This extends [[Resolving Distributed Knowledge]] (Agotnes & Wang) from synchronous to asynchronous settings, analogous to how asynchronous distributed algorithms generalize synchronous ones in computer science. It relates to broader work on dynamic epistemic logic (DEL) — logics that model how knowledge changes after public announcements or information updates. The paper also connects to [[multi-agent systems]] theory and distributed computing, where asynchronous message-passing is the norm. Unlike [[RLHF|reinforcement learning approaches]], this is a symbolic/logical framework rather than learning-based.

## Practical Applications
1. **Distributed consensus protocols**: Reasoning about when agents in a network can agree on facts when they receive updates at different times (e.g., blockchain or Byzantine fault tolerance).
2. **Asynchronous team coordination**: Modeling knowledge-sharing in teams working across time zones or with unreliable communication (e.g., remote software development teams, space missions with communication delays).
3. **Security and privacy in distributed systems**: Reasoning about what information leaks to which agents in asynchronous systems, critical for protocol design.
4. **Formal verification of distributed algorithms**: Checking that distributed algorithms behave correctly even when agents have stale or partial knowledge of global state.
5. **Specification of information-hiding protocols**: Designing systems where agents intentionally obscure knowledge-sharing to preserve privacy.

## Product & Startup Ideas
1. **Formal verification SaaS for distributed systems**: Automated tool to check that a distributed protocol (microservices, consensus, messaging) preserves desired knowledge properties (e.g., "agent A never learns that agent B and C communicated").
2. **Asynchronous team coordination software**: Workplace tool that reasons about what information different team members should have given their time zone and message-send history, suggesting when to re-broadcast or clarify information.
3. **Privacy-preserving audit logging**: System that uses asynchronous distributed knowledge logic to determine what minimal information must be logged so auditors can verify protocol compliance without revealing sensitive cross-agent interactions.
4. **Educational platform for distributed systems**: Interactive tool that lets students design and verify asynchronous distributed algorithms by specifying knowledge constraints.
5. **Smart contract security analyzer**: Blockchain tool that checks smart contracts and inter-contract interactions for unintended information leakage in asynchronous update scenarios.

## Technical Limitations
1. **Computational complexity**: The paper does not discuss the computational cost of model checking or satisfiability checking for this logic — likely exponential, making it impractical for very large agent sets or long histories.
2. **Scalability**: History-based semantics require tracking the full history of resolutions; in practice, systems may need to "forget" old events, but it is unclear how to do this without breaking the logic.
3. **Memory-less agents assumption**: Real agents have memory and learn from past interactions. The paper assumes agents are stateless except for observed resolutions.
4. **No quantitative timing**: The logic captures partial order of events but not real time or timeouts, limiting applicability to systems with strict timing constraints (e.g., real-time control).
5. **Incomplete practical validation**: The paper is purely theoretical; no empirical validation on real distributed systems or comparison with existing verification tools (TLA+, Alloy, etc.).

## Open Questions
1. **Axiom completeness**: Is the proposed axiomatization complete (every valid formula is derivable)? Proof sketches exist but full proofs are absent.
2. **Model checking algorithms**: What is the complexity of deciding truth in a model, and can it be done efficiently for realistic system sizes?
3. **Relationship to temporal logic**: How does asynchronous distributed knowledge relate to Linear Temporal Logic (LTL) or branching-time logic (CTL*), which are standard in distributed systems verification?
4. **Learning and bounded history**: Can agents learn and forget earlier resolutions in a way that is logically sound? How would this affect the axioms?
5. **Multi-resolution scenarios**: What happens when agents resolve (share) knowledge at overlapping times but out of order? Does the order matter axiomatically?
6. **Decidability boundaries**: For which fragments of the full logic are satisfiability and model checking decidable?

## Key Takeaway
Asynchronous distributed systems require a new epistemic logic where agents only observe knowledge-sharing events that directly involve them, invalidating classical synchronous axioms but enabling realistic reasoning about knowledge in real distributed computing scenarios.
