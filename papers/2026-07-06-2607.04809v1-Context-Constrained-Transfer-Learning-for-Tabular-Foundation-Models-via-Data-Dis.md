# Context-Constrained Transfer Learning for Tabular Foundation Models via Data Distillation

**Authors:** Yijun Lin, Sai Li
**Published:** 2026-07-06 | **arXiv:** [2607.04809v1](http://arxiv.org/abs/2607.04809v1)

## Summary
This paper addresses a key limitation of Tabular Foundation Models (TFMs)—neural networks trained on diverse tabular datasets to serve as general-purpose predictors—when applied to transfer learning tasks. The core problem: TFMs struggle when moving to new datasets because they have strict limits on how much context (training data) they can use, and they're sensitive to distribution shifts. The authors propose TL-ANDI, a method that distills a source dataset down to its most informative samples using optimal transport, then fine-tunes the model with those compressed samples plus local label refinement. This matters because it makes TFMs practical for real-world tabular problems where data is heterogeneous and budgets are tight.

## Theoretical Background
Tabular Foundation Models are deep-learning systems trained on large collections of diverse tabular (spreadsheet-like) datasets to learn general patterns, similar to how large language models work on text. They use in-context learning—the ability to adapt predictions based on a small set of examples in the input without explicit retraining.

The paper builds on several prior concepts:

- **In-context learning**: The ability of a model to adapt behavior based on examples provided in its input context, borrowed from the foundation model literature.
- **Optimal transport**: A mathematical framework for measuring the "distance" between distributions and finding efficient ways to transform one distribution into another. Here, it's used to select which source samples best represent the target task.
- **Data distillation / coreset selection**: Techniques for compressing a large dataset into a smaller representative subset without losing predictive power.
- **Negative transfer**: The phenomenon where transfer learning hurts performance because source and target tasks are mismatched.

The authors frame transfer learning as a budget-constrained problem: given a strict limit on context size, which samples from source data should you keep?

## Core Contributions
1. **Formulation of context-constrained transfer as optimal transport**: The paper reframes the sample selection problem as minimizing a cost function that balances two competing objectives—coverage of the target covariate space and compatibility with the target task's posterior (what the model predicts). This is solved via optimal transport with explicit budget constraints.

2. **Posterior-aware distillation framework (TL-ANDI)**: Rather than naive pooling of source data, the method (1) selects anchor samples via the optimal transport solution, (2) computes locally distilled labels for those samples to better align with the target, and (3) applies residual calibration using target data to correct any remaining misalignment.

3. **Empirical validation on heterogeneous domains**: The paper demonstrates that this approach mitigates negative transfer and outperforms naive baselines across diverse tabular tasks, even with severe context constraints.

## Practical Applications
1. **Medical diagnosis systems**: Hospitals train on diverse patient datasets but need to deploy models locally with limited context (e.g., embedded in resource-constrained clinical software). TL-ANDI selects the most relevant historical cases for a new clinic's population without retraining.

2. **Credit risk assessment**: Financial institutions maintain foundation models trained across thousands of portfolios. When deploying to a new market or borrower segment, strict context constraints (e.g., real-time inference on mobile devices) make this approach useful for selecting relevant historical loan outcomes.

3. **IoT sensor anomaly detection**: Device manufacturers train models on diverse sensor telemetry. When deploying to a new customer's equipment, memory and latency budgets are tight. TL-ANDI selects exemplar sensor readings that best represent the new environment.

4. **E-commerce recommendation adaptation**: A recommendation model trained across many product categories needs to adapt to a new seller's catalog with minimal context overhead.

5. **Epidemiological forecasting**: Public health agencies use regional data to forecast disease spread. When deploying to a new region with different demographics, TL-ANDI selects the most similar historical regions' data to bootstrap the model.

## Technical Limitations
1. **Optimal transport computational cost**: The optimal transport solver may be expensive for very large source datasets, limiting scalability to truly massive corpora.

2. **Assumption of covariate stationarity**: The method assumes that target covariates are roughly stable; if the target distribution is highly nonstationary or contains novel feature types, the covariate coverage objective may be insufficient.

3. **Posterior compatibility assumption**: Assumes posterior compatibility is meaningful and can be estimated from limited data; in high-dimensional spaces or with very few target samples, this may be unreliable.

4. **Label distillation requires labeled source data**: The approach assumes the source samples have labels; it may not work well when source data is unlabeled or labels are noisy.

5. **Context-size rigidity**: The method respects hard context constraints but does not adaptively trade off context size vs. accuracy; it may be suboptimal if some slack in context is permissible.

6. **Limited to tabular data**: The approach is specific to tabular settings and does not generalize to images, text, or other modalities.

## Open Questions
1. **How does performance degrade as context budgets shrink to extremely small values (e.g., 5–10 samples)?** Does posterior compatibility remain meaningful at such extreme compression ratios?

2. **Can the optimal transport formulation be extended to handle distribution shifts in the label space (not just covariates)?** Current work focuses on covariate coverage; what if the target task has fundamentally different outcome distributions?

3. **How sensitive is TL-ANDI to the choice of cost function in the optimal transport problem?** Would learned or task-specific cost functions outperform the fixed covariate coverage + posterior compatibility objective?

4. **Can TL-ANDI be combined with active learning** to iteratively request the most informative target samples, rather than assuming a fixed target dataset upfront?

5. **How does this compare to simpler baselines like nearest-neighbor sample selection or random sampling** in the limit of very small context budgets?

6. **Does the residual calibration step remain effective across diverse target domains**, or does it require task-specific tuning?

## Key Takeaway
TL-ANDI solves a practical yet underexplored problem: transferring foundation models to new tasks when you can only afford to carry a tiny context window, using optimal transport to distill the most relevant and compatible source examples.
