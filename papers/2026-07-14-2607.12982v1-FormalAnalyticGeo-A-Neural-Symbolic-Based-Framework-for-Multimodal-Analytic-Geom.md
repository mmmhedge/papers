# FormalAnalyticGeo: A Neural-Symbolic Based Framework for Multimodal Analytic Geometry Problem Generation

**Authors:** Ruoran Xu, Wending Gao, Qiufeng Wang
**Published:** 2026-07-14 | **arXiv:** [2607.12982v1](http://arxiv.org/abs/2607.12982v1)

## Summary
This paper presents FormalAnalyticGeo, a framework that automatically generates thousands of multimodal analytic geometry problems (problems combining text, diagrams, and equations). The system uses large language models coordinated through a formal intermediate language to generate problems and render precise diagrams, then verifies correctness automatically—solving the bottleneck of human annotation that has limited progress in training math-reasoning AI systems on geometry.

## Theoretical Background
The work builds on two main foundations. First, multimodal large language models (MLLMs)—neural networks that process both text and images—have advanced mathematical reasoning, but analytic geometry (the study of geometric shapes using coordinates and equations) has been largely absent from training datasets because creating annotated examples requires human effort at scale. Second, the paper leverages formal languages (structured symbolic representations like programming languages) as intermediaries between free-form text and precise computational outputs. The framework uses Signed Distance Functions (SDFs)—mathematical functions that encode geometric shapes with high precision—to render diagrams correctly, a technique borrowed from computer graphics.

## Core Contributions
1. **CDL (Condition Description Language)**: A formal intermediate representation specifically designed to bridge natural language problem descriptions with precise diagram rendering via SDFs. This is novel—prior work treated text-to-diagram generation as a single end-to-end task, losing precision.
2. **Multi-stage LLM pipeline with feedback loops**: Four specialized components (Generator → Formalizer → Measurer → Quality Verifier) where the verifier's structured feedback automatically triggers retries, eliminating need for human review.
3. **AnalyticGeo7K dataset**: Over 7,000 verified multimodal problems with <0.70% median ground-truth error and 82.3% of answers within 5% of symbolic solutions—a quality benchmark for synthetic math problems.
4. **Vision-based measurement**: Using LLMs with vision capabilities to extract ground-truth answers from rendered diagrams, closing the loop on correctness without symbolic computation.

## Practical Applications
1. **Math tutoring platforms**: Generate personalized geometry problem sets at scale (Khan Academy, Chegg, Coursera competitors).
2. **Standardized test preparation**: Create unlimited geometry problems for SAT, ACT, or university entrance exams with verified correctness.
3. **Curriculum development**: Publishers can auto-generate problem banks for textbooks, worksheets, and homework assignments in real-time.
4. **LLM fine-tuning datasets**: Use AnalyticGeo7K to train specialized math-reasoning models with higher geometric accuracy.
5. **Adaptive learning systems**: Dynamically generate problems matched to student difficulty levels and learning gaps.

## Product & Startup Ideas
1. **GeometryBot Pro**: A subscription SaaS for educators—input curriculum standards and skill levels, auto-generate worksheets, quizzes, and answer keys daily.
2. **MathDataset API**: Sell access to AnalyticGeo7K and the generation pipeline as an API for LLM companies training custom models on domain-specific math problems.

4. **Exam Preparation Platform**: A targeted product (e.g., "SAT Geometry Genius") using auto-generated problems and adaptive difficulty to maximize score improvement.
5. **Corporate Training Content**: Generate geometry/spatial reasoning problems for engineering, architecture, and GIS professional training programs.

## Technical Limitations
1. **Scope limited to analytic geometry**: The framework is not demonstrated on other math domains (algebra, calculus, combinatorics). Generalization to 3D geometry or more complex constraint systems is unclear.
2. **LLM hallucination still present**: Relies on verification to catch errors, but the Formalizer or Generator may produce invalid CDL or unsolvable problems; the feedback loop can only retry, not guarantee success.
3. **SDF rendering constraints**: Analytic geometry's emphasis on conic curves (circles, ellipses, hyperbolas) is well-suited to SDFs, but more complex algebraic curves or implicit surfaces may exceed the method's precision.
4. **Computational cost not disclosed**: No analysis of per-problem generation cost (LLM API calls, SDF rendering time, verification overhead), which matters for production scaling.
5. **Answer extraction via vision**: The Measurer relies on vision-based measurement from rendered diagrams—any rendering artifacts or antialiasing errors propagate to ground-truth labels, potentially biasing downstream models.

## Open Questions
1. How does performance degrade on problems with multiple constraints or rare configuration types? Does the dataset have class imbalance?
2. Can the framework extend to 3D analytic geometry or other mathematical domains without redesigning the formal language?
3. What is the actual computational cost per problem, and how does it compare to human annotation at scale?
4. How well do models trained on AnalyticGeo7K transfer to human-authored geometry problems and to more complex spatial reasoning tasks?
5. Can the CDL and SDF approach be made interactive—allowing students to modify constraints and see diagrams update in real-time?

## Key Takeaway
LLMs and formal languages together can auto-generate thousands of verified geometry problems with diagram precision, scaling math dataset creation without human annotation.
