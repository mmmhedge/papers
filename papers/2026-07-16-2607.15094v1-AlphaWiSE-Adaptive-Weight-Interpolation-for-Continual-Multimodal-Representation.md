# AlphaWiSE: Adaptive Weight Interpolation for Continual Multimodal Representation Learning

**Authors:** Sarthak Jain, Qiran Hu, Zhen Zhu, Yaoyao Liu
**Published:** 2026-07-16 | **arXiv:** [2607.15094v1](http://arxiv.org/abs/2607.15094v1)

## Summary
This paper addresses a fundamental problem in continual learning for multimodal models like CLIP: how to adapt to new data without forgetting earlier cross-modal alignments. AlphaWiSE solves this by interpolating between two frozen model checkpoints using learned scalar weights, achieving better stability-plasticity trade-offs than existing methods while keeping the same model size and inference cost.

## Theoretical Background
Multimodal models like CLIP learn shared embedding spaces by training on image-text pairs, where different modalities (images, audio, text) are projected into a common vector space for cross-modal retrieval. Continual learning addresses the problem of updating models on sequentially arriving data without catastrophic forgetting — the tendency to lose performance on earlier tasks. Prior continual-learning approaches typically return a single checkpoint that represents a fixed compromise between stability (retaining old knowledge) and plasticity (learning new knowledge). Weight interpolation is a post-hoc technique that combines multiple model checkpoints in weight space rather than at inference time, potentially offering flexible trade-off control.

## Core Contributions
1. **Post-hoc weight interpolation for multimodal models**: AlphaWiSE learns per-parameter scalar coefficients that linearly combine two frozen source checkpoints, rather than training a single continuously-updated model.
2. **Efficiency-conscious design**: The interpolated checkpoint maintains the original architecture and parameter count, requiring no additional inference time compared to either source checkpoint.
3. **Cross-modal retrieval robustness**: Demonstrated improvements across multiple retrieval directions (image→text, audio→image, etc.) and evaluation metrics, showing the method generalizes beyond single-modal tasks.
4. **Exemplar-based fitting**: Coefficients are fitted on a small exemplar memory buffer, making the approach practical for memory-constrained scenarios.

## Connections to Other Work
This work relates to:
- [[Continual Learning]] literature, which typically optimizes a single checkpoint; AlphaWiSE differs by composing multiple frozen ones.
- [[Model Merging]] and [[Task Arithmetic]], which also interpolate model weights but often in different contexts (e.g., task-specific fine-tuning rather than continual multimodal adaptation).
- [[CLIP]] and vision-language pretraining, which establish the shared embedding space assumption central to this work.
- [[Catastrophic Forgetting]] — the core problem this addresses in the multimodal setting.

## Practical Applications
1. **Video recommendation systems**: Continually adapt image-text-audio alignment as new video content arrives without retraining from scratch.
2. **E-commerce search**: Update cross-modal product search (image, description, category embeddings) as new inventory and user behavior data stream in.
3. **Medical imaging platforms**: Adapt models to new imaging modalities (CT, MRI, ultrasound) and clinical text without degrading earlier alignment quality.
4. **Accessibility tools**: Improve caption-image-audio synchronization in real-time as new content types and user interactions are observed.
5. **Zero-shot discovery systems**: Maintain robust cross-modal search in dynamic knowledge bases (e.g., scientific papers with figures and abstracts).

## Product & Startup Ideas
1. **Multimodal search API-as-a-service**: A managed service that continuously adapts image-text-audio retrieval models for enterprises (e-commerce, content platforms) without requiring model retraining or downtime.
2. **Continual learning toolkit for vision-language startups**: Pre-built library enabling startups to deploy CLIP-like models that improve with new multimodal data streams without engineering continual-learning expertise.
3. **Incremental knowledge graph builder**: A tool that embeds entities across modalities (images, descriptions, audio) and adapts alignments as new data arrives, useful for search, recommendation, and Q&A.
4. **Dynamic video understanding platform**: Real-time video indexing and retrieval that learns from incoming videos without forgetting prior alignments, targeting broadcasters and social platforms.
5. **Federated multimodal learning service**: Enable organizations to contribute new data to a shared multimodal model checkpoint without full retraining, using weight interpolation to balance global and local objectives.

## Technical Limitations
1. **Two-checkpoint constraint**: Interpolating only between two frozen sources may limit expressiveness if data distribution shifts are large or non-linear; more complex composition strategies may be needed.
2. **Exemplar memory dependency**: Performance depends on selecting representative exemplars; no clear guidance on exemplar selection strategy or memory budget sizing.
3. **Per-parameter coefficient scaling**: Using a single scalar per tensor may be too coarse for parameters with different roles or sensitivities in cross-modal alignment; block-wise or structured sparsity might be more effective.
4. **Evaluation scope**: Tested on audio-image-text but unclear how well this generalizes to higher-order modality combinations or truly out-of-distribution continual scenarios.
5. **Theoretical justification**: Limited analysis of *why* linear interpolation preserves cross-modal alignment — the geometry of multimodal embedding spaces under interpolation is not deeply characterized.

## Open Questions
1. How does AlphaWiSE scale to >2 source checkpoints? Can you combine multiple task-specific models with learned weights?
2. Can you adaptively select which parameters to interpolate (e.g., skip shallow layers) to reduce fitting cost further?
3. How sensitive is performance to exemplar memory size and selection strategy? Is there a principled way to choose exemplars?
4. Does linear interpolation fail for certain types of distribution shifts (e.g., new modality, domain gap)? When would non-linear composition be necessary?
5. Can you extend this to continual learning with label shift or class-incremental scenarios (new classes arriving over time)?
6. How does this compare to other weight-space merging techniques (e.g., task arithmetic, TIES-Merging) in the continual multimodal setting?

## Key Takeaway
Interpolating between frozen model checkpoints adapts multimodal models to new data while preserving old alignments, with no extra inference cost.
