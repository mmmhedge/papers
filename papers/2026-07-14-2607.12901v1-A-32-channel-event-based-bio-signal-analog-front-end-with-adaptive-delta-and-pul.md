# A 32-channel event-based bio-signal analog front-end with adaptive delta and pulse frequency encoding

**Authors:** Narayanan Shyam, Saptarshi Ghosh, Giacomo Indiveri
**Published:** 2026-07-14 | **arXiv:** [2607.12901v1](http://arxiv.org/abs/2607.12901v1)

## Summary
This paper describes a specialized computer chip (ASIC) that reads biological signals from 32 sensors and converts them into sparse, event-based data streams using two encoding methods. The innovation is an adaptive encoder that automatically adjusts compression based on signal properties, enabling ultra-low-power wireless transmission of brain signals for brain-computer interfaces and neuromorphic computing systems.

## Theoretical Background
Event-based signal processing is inspired by how biological neurons communicate — rather than continuously sampling at fixed intervals, the system only transmits information when the signal changes significantly. This paper builds on two encoding approaches: **Pulse Frequency Modulation (PFM)**, where signal intensity maps to spike frequency, and **Asynchronous Delta Modulation (ADM)**, which transmits only deviations from a reference level. The key innovation adds auto-scaling to ADM so the threshold adapts in real-time to the signal's amplitude envelope, improving compression for low-amplitude biosignals. The work targets integration with **Spiking Neural Networks (SNNs)** — neural networks that process information through discrete, temporally-precise events rather than continuous values — making the output format naturally compatible with neuromorphic processors.

## Core Contributions
1. **Dual-mode event-based AFE**: A 32-channel ASIC combining PFM and adaptive delta encoding in a single configurable chip (180 nm CMOS).
2. **Adaptive envelope-tracking mechanism**: The aADM encoder automatically scales its quantization threshold based on the input signal envelope, dramatically improving data compression ratios while maintaining signal fidelity.
3. **Hardware-software co-design**: Direct compatibility with state-of-the-art neuromorphic processors, eliminating conversion overhead between biomedical front-end and SNN backends.
4. **Programmable per-channel control**: Each of 32 channels can independently select encoding mode and parameters, enabling mixed-mode operation.

## Connections to Other Work
This work bridges neuromorphic hardware (event-driven processing inspired by neuroscience) and biomedical signal acquisition. It relates to broader efforts in:
- **Neuromorphic computing**: Matching hardware encoding to downstream SNN processors, avoiding redundant format conversions.
- **Low-power biomedical sensing**: Complementary to wireless neural recording systems that suffer from data bandwidth and power constraints.
- **Adaptive signal processing**: The envelope-tracking concept parallels automatic gain control in classical DSP but applied to event-encoding thresholds.
- **Brain-computer interfaces (BCIs)**: Enabling real-time, wireless neural signal transmission for closed-loop BCI applications.

## Practical Applications
1. **Implantable neural recording systems**: Wireless transmission of brain signals from implanted electrodes with minimal power consumption.
2. **Wearable EEG/EMG devices**: Scalp or muscle sensors streaming to edge neuromorphic processors for local gesture recognition or seizure detection.
3. **Lab-scale electrophysiology**: Replacing traditional data acquisition systems with event-driven streaming to reduce disk I/O and enable real-time artifact rejection.
4. **Rehabilitation monitoring**: Continuous muscle activity encoding in prosthetic or exoskeleton systems with on-device learning.

## Product & Startup Ideas
1. **Neural telemetry module for animal research**: A drop-in 32-channel wireless recorder for freely-behaving neuroscience experiments, reducing tether burden and enabling multi-animal setups.
2. **Edge BCI development kit**: Turnkey platform combining this AFE + SNN processor for researchers prototyping closed-loop brain-machine interfaces without RF/power expertise.
3. **Seizure detection wearable**: Continuous EEG headband streaming to wrist-worn SNN processor for real-time seizure prediction and smartphone alerts.
4. **Sports/fatigue monitoring band**: Multiplex EMG + motion sensors on athletes to predict injury risk or training load via on-device neuromorphic inference.
5. **Biomedical IoT middleware**: Licensing the adaptive encoding IP to medical device makers as a drop-in power-efficient front-end for remote patient monitoring.

## Technical Limitations
- **Fixed 180 nm process**: Limits integration density and power efficiency compared to modern (5–28 nm) nodes; scaling benefits are forfeited.
- **32-channel cap**: Adequate for focal recordings but insufficient for whole-brain or large-scale array applications.
- **Adaptive threshold latency**: Auto-scaling mechanism may lag rapid signal transients; no analysis of steady-state vs. transient encoding fidelity.
- **Mismatch and noise**: No characterization of channel-to-channel variability, thermal drift, or input-referred noise floor across all 32 channels.
- **Single-ended design assumptions**: Likely assumes differential or single-ended inputs; unknown robustness to common-mode interference or impedance mismatches.
- **Limited validation**: No comparison to commercial AFEs or quantitative compression benchmarks on real neural data; feasibility for practical BCIs unclear.

## Open Questions
1. How does adaptive envelope tracking perform on non-stationary signals (e.g., state transitions in sleep, seizures)? Does overshoot or lag degrade event timing?
2. What is the actual power consumption per channel, and how does it scale with input signal statistics across diverse biomedical sources (EEG, EMG, LFP, action potentials)?
3. Can the dual encoding modes be dynamically switched per-channel based on signal content, or does it require offline reconfiguration?
4. How well do encoded spikes preserve temporal fine structure for downstream decoding tasks in BCIs?
5. Is the chip's compatibility with "state-of-the-art" SNNs empirically validated end-to-end on a real application (e.g., movement decoding, seizure detection)?

## Key Takeaway
Event-driven 32-channel biomedical chip with adaptive compression for ultra-low-power neural signal streaming to neuromorphic processors.
