# Cache Merging as a Convergent Replicated State for Multi-Agent Latent Reasoning

**Authors:** Carlos Baquero, Luís Brito
**Published:** 2026-07-01 | **arXiv:** [2607.01308v1](http://arxiv.org/abs/2607.01308v1)

## Summary

This paper solves a fundamental problem in multi-agent reasoning: how to merge the internal memory states (KV-caches) of multiple language model agents into a single coherent context. Previous work concatenated caches in arbitrary order, producing different results depending on input ordering. The authors propose CanonicalMerge, which orders cache fragments by a content-based metric (mean K-norm) to produce byte-identical results regardless of input order—enabling reliable, predictable multi-agent composition for the first time.

## Theoretical Background

Modern transformers (neural networks using attention mechanisms) store key-value pairs in "KV-caches" during decoding—a compressed representation of past context that speeds up inference. Multi-agent systems often need to combine reasoning from multiple agents. Prior work (Agent Primitives) concatenated these caches along the sequence dimension with RoPE re-encoding (a positional encoding scheme), but this operation is order-dependent: the same agents produce different outputs if merged in different sequence.

The paper grounds the solution in distributed systems theory, specifically convergent replicated data types (CvRDTs)—a formal model for achieving consistency across distributed copies without coordination. A CvRDT guarantees that merge operations are commutative (order doesn't matter), associative (grouping doesn't matter), idempotent (duplicates are harmless), and absorbing (information is monotonically added).

## Core Contributions

1. **CanonicalMerge**: A deterministic ordering scheme (by mean K-norm at a middle layer) that makes cache merging commutative—the merged cache is byte-identical regardless of input order.

2. **Formal CvRDT model**: Recasts multi-agent cache merging as set union over content-addressed latent fragments, with CanonicalMerge as the deterministic render function. This separates the abstract replicated state (a set) from its concrete layout (the ordered merged cache).

3. **Empirical validation**: Demonstrates byte-for-bit reproducibility on Qwen 3 models (1.7B and 4B), matches best BagMerge performance across all regime-budget-ordering combinations without requiring oracle knowledge of best orderings, and shows transfer to real multi-document QA (HotpotQA).

4. **Comparison to output-level fusion**: Shows cache-level merging outperforms PackLLM (an output-fusion baseline) by 45 points on matched budget, indicating these are qualitatively different regimes.

This work extends [[Agent Primitives]] (BagMerge) by removing its order-dependency problem. ## Practical Applications

1. **Collaborative multi-agent reasoning**: Teams of specialized LLM agents (e.g., one for math, one for retrieval, one for synthesis) can now merge their reasoning states deterministically for a final aggregation step without trial-and-error ordering.

2. **Distributed document QA**: HotpotQA-style retrieval tasks where multiple agents independently process different documents; their caches merge predictably regardless of which agent finishes first.

3. **Fault-tolerant multi-agent systems**: Duplicate agents or redundant nodes produce identical merged states, enabling straightforward deduplication and recovery without re-computation.

4. **Heterogeneous model composition**: Combining smaller specialist models with different architectures or sizes, where cache-level fusion is more robust than naive concatenation.

5. **Asynchronous agent coordination**: Agents can submit their cache fragments in any order; the final merge is deterministic, eliminating synchronization overhead.

## Product & Startup Ideas

1. **Multi-agent API platform**: A managed service where customers define agent teams (retrieval, reasoning, synthesis) that compose deterministically; charge per cache-merge operation and guarantee latency/reproducibility SLAs.

2. **Specialized reasoning-as-a-service**: Offer "task routers" that deploy multiple fine-tuned models for different domains and merge their caches at inference time; differentiate on deterministic, auditable composition.

3. **Enterprise document processing**: SaaS tool for legal/compliance document analysis where multiple agents (one per document category) merge caches for final synthesis; emphasize reproducibility and auditability.

4. **LLM debugging & testing**: Developer tool that lets users test multi-agent pipelines and verify cache merges produce identical outputs under permutations, with dashboards showing latent reasoning alignment.

5. **Federated reasoning marketplace**: A decentralized network where independent agents (run by different organizations) submit reasoning caches; a coordinator merges them deterministically without central orchestration.

## Technical Limitations

1. **Scaling beyond k=2 agents**: The paper acknowledges that at k>2 agents, the approach "transports and colocates latent traces but does not by itself compose them." Composing three or more caches remains an open problem; current solution merely aligns them without semantic fusion.

2. **Mean K-norm as universal ordering signal**: Assumes mean K-norm at a middle layer is a robust content proxy across all model scales and domains. May not generalize to radically different architectures (MoE, retrieval-augmented models) or fine-tuned variants.

3. **Byte-identical equivalence fragility**: Requires bit-for-bit matching in floating-point operations, which is brittle across hardware (GPUs, TPUs, different precision modes) and library versions.

4. **No theoretical compositionality guarantee**: The CvRDT model ensures merge consistency but does not guarantee that merged reasoning is semantically superior to individual agent outputs—only that it's reproducible.

5. **Budget-agnostic ordering**: The approach works under all latent-step budgets but doesn't optimize which agents should be prioritized when budget is very constrained.

## Open Questions

1. How can we semantically compose k>2 caches to actually fuse reasoning rather than just align latent traces? Is there a learned composition function, or does it require architectural changes?

2. Does mean K-norm remain a stable ordering signal across model families, scales, and domains? How sensitive is byte-identical reproducibility to small changes in ordering?

3. Can we extend the CvRDT model to weighted merges, where some agents' contributions should be prioritized over others?

4. How does cache merging interact with in-context learning or few-shot prompting? Do agent caches that include examples compose differently?

5. Can we design a composable loss function that explicitly encourages agents to produce merging-friendly cache structures during training?

## Key Takeaway

Multi-agent LLM reasoning can be made structurally deterministic and reproducible by ordering cached representations by content rather than concatenating them arbitrarily—turning a regime-dependent heuristic into a convergent replicated state.
