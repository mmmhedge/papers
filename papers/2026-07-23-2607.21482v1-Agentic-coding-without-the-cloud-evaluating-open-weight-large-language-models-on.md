# Agentic coding without the cloud: evaluating open-weight large language models on longitudinal data preparation tasks

**Authors:** Mack Nixon, Liam Wright, Yevgeniya Kovalchuk, Alison Fang-Wei Wu, Martin Danka et al.
**Published:** 2026-07-23 | **arXiv:** [2607.21482v1](http://arxiv.org/abs/2607.21482v1)

## Summary
This paper evaluates open-source large language models running locally on consumer hardware for automated data preparation — a tedious bottleneck in research involving sensitive longitudinal data. The work demonstrates that mid-size open models (31–35B parameters) can handle complex data cleaning tasks like merging survey waves and harmonizing categories with 87.9% success, offering a privacy-safe alternative to cloud-based AI coding assistants.

## Theoretical Background
The paper builds on two converging areas: (1) agentic LLMs — language models augmented with tools and reasoning loops to execute multi-step code generation tasks — and (2) the broader shift toward open-weight models (like Llama, Mistral) as privacy-preserving alternatives to proprietary services. It inherits from prior work on code generation evaluation (benchmarking LLM-produced code correctness) and data preparation automation in research pipelines. The key tension it addresses is governance — many research institutions cannot send personal data to third-party cloud services, creating friction that slows adoption of AI-assisted workflows.

## Core Contributions
1. **RRBench framework**: A curated evaluation suite with ground-truth data preparation scripts from a real British cohort study, task definitions (category harmonization, multi-wave merging, etc.), and automated evaluation routines that check both code correctness and output data validity.
2. **Empirical benchmark**: Systematic evaluation of open-weight models across the consumer-grade spectrum (varying sizes and hardware setups), showing that 31–35B parameter models achieve near-saturation performance (87.9% task completion) on 20 distinct data preparation tasks spanning 102 variable creations.
3. **Practical validation**: Demonstration that locally deployed open models are viable for governance-restricted research, eliminating the need to transmit sensitive longitudinal data externally.

## Connections to Other Work
This work relates to [[Agents|agentic AI systems]] that use code execution as a reasoning mechanism (e.g., ReAct, Toolformer), benchmarking frameworks for code generation (HumanEval, MBPP), and the emerging discourse around [[large-language-models|LLM privacy and data governance]]. It also connects to broader efforts to democratize AI via open-weight models and to address the "last mile" of machine learning — getting models into restricted environments where proprietary cloud services are prohibited. The emphasis on longitudinal data preparation is specific to epidemiology and social science, but the underlying agent + code generation pattern is general.

## Practical Applications
1. **Academic research institutions**: Automating data cleaning for longitudinal studies (NHS biobanks, birth cohorts, panel surveys) without breaching data governance rules.
2. **Clinical research sites**: Preparing Electronic Health Record (EHR) data for analysis while keeping sensitive patient data on-premise.
3. **Government statistical agencies**: Processing census or administrative data on air-gapped systems with AI assistance.
4. **Pharmaceutical and biotech R&D**: Preparing multi-wave clinical trial datasets for analysis under strict data residency requirements.
5. **Patient registries and rare disease networks**: Harmonizing data from decentralized sources without centralized data transfer.

## Product & Startup Ideas
1. **Privacy-first data prep SaaS for research**: A subscription service bundling RRBench-style evaluation frameworks with containerized open-weight models, offered as a drop-in replacement for manual data cleaning — target: academic and biotech labs with governance constraints.
2. **Enterprise AI data governance toolkit**: A managed platform that deploys open-weight models on client infrastructure (on-premise or private cloud), with audit trails and compliance reporting, designed for regulated industries (pharma, finance, healthcare).
3. **Longitudinal study accelerator**: A vertical SaaS for epidemiologists and social scientists that provides templated agentic workflows for common data preparation tasks (harmonization, merging, validation), reducing manual scripting.
4. **GPU time-sharing marketplace for privacy-constrained AI**: A decentralized or federated compute platform where researchers can rent local compute (via edge devices or private data centers) to run agentic LLMs, with payment and provenance tracking.
5. **Data preparation audit and validation service**: A consulting/software hybrid offering automated RRBench-style evaluation of LLM-generated code and data outputs for research institutions, with human review workflows for high-stakes decisions.

## Technical Limitations
1. **Task specificity**: The benchmark is tuned to longitudinal British cohort data; generalization to other data schemas, geographies, or domains (e.g., financial, genomic) remains unclear.
2. **Model size vs. hardware trade-off**: 31–35B models demand significant consumer GPU memory (>24GB VRAM); this excludes smaller research labs with modest hardware.
3. **Code correctness ≠ statistical validity**: The framework evaluates whether generated R code runs and produces the right shape of data, but does not deeply validate statistical semantics (e.g., whether a harmonization rule is scientifically sound).
4. **Agentic loop robustness**: Real-world data is messier than curated benchmarks; the paper does not report failure modes when agents get stuck in infinite loops or produce partially correct code that silently produces wrong outputs.
5. **Reproducibility across model versions**: Open-weight models evolve rapidly; performance on a 34B model today may not match a fine-tuned variant tomorrow.
6. **Evaluation overhead**: The automated evaluation suite requires ground-truth scripts and expected outputs, which is labor-intensive to construct for new data domains.

## Open Questions
1. How do open-weight models perform on *novel* data schemas not seen during the model's pre-training, versus the real-world heterogeneity of research data?
2. Can the framework be extended to handle unstructured or semi-structured data (notes, images, audio) in longitudinal studies?
3. What is the minimum viable model size and hardware spec for acceptable performance? Can smaller, quantized models (7–13B) run on CPU-only machines with acceptable latency?
4. How does performance degrade when data is extremely messy (missing headers, inconsistent formatting, undocumented encoding changes across survey waves)?
5. Can the agentic system provide explanations or uncertainty estimates for its data transformations, useful for researcher validation?
6. How does human-in-the-loop refinement (researcher feedback on partial or incorrect outputs) compare to fully autonomous approaches in terms of total time-to-completion?

## Key Takeaway
Open-weight LLMs running locally can automate data preparation for sensitive research without transmitting data to the cloud, achieving 88% task success on real longitudinal cohort studies.
