# TreeAgent: A Generalizable Multi-Agent Framework for Automated Bias Labeling in Forestry via Compiled Expert Rules and Vision-Language Models

**Authors:** Shiyi Chen, Nicholas Saban, Collin Hargreaves, Huiqi Wang
**Published:** 2026-06-30 | **arXiv:** [2606.31976v1](http://arxiv.org/abs/2606.31976v1)

## Summary
This paper presents TreeAgent, a multi-agent system that combines expert decision trees with vision-language models (VLMs) to automatically label tree bias in forestry remote sensing images. Instead of relying on slow, inconsistent human experts, the system uses VLMs to make semantic judgments at each decision node, with multiple agents voting to reduce errors — achieving better accuracy than traditional supervised learning while cutting annotation costs significantly.

## Theoretical Background
The paper builds on several established ideas: **expert systems** (rule-based AI that encodes human knowledge as decision trees), **vision-language models** (neural networks like CLIP that understand both images and text), and **multi-agent systems** (frameworks where multiple autonomous agents coordinate to solve problems). The key innovation is formalizing how to couple these—using structured expert rules as a scaffold while delegating perception tasks to neural models. This sits at the intersection of neuro-symbolic AI (combining neural networks with symbolic reasoning) and collaborative decision-making, addressing the classic problem of annotation bottlenecks in domain-specific ML tasks.

## Core Contributions
1. **D3 Framework (Decoupled Declarative Decision)**: A formal method to generalize multi-agent VLM orchestration *without modifying code* across different expert-defined decision structures—enabling zero-shot application to new domain rule sets.
2. **Multi-agent voting mechanism**: Demonstrates that ensemble VLM decisions at each decision node reduce stochasticity and improve consistency compared to single VLM calls.
3. **Empirical validation**: Shows the approach outperforms supervised baselines on tree height bias classification while dramatically reducing required expert labels.
4. **Interpretability preservation**: The system maintains explainability because decisions trace back to human-defined rules and localized VLM reasoning, unlike end-to-end deep learning.

## Connections to Other Work
This work bridges [[multi-agent systems]] research with [[large-language-models]] (or more precisely, vision-language models), echoing recent efforts to use LLMs as reasoning engines (e.g., chain-of-thought prompting, tool use). It relates to neuro-symbolic approaches that blend learning with symbolic constraints. The voting mechanism is conceptually similar to ensemble methods in classical ML. The domain application parallels recent work using VLMs for ecological and environmental monitoring tasks. Unlike pure [[reinforcement-learning]] approaches that learn from scratch, TreeAgent leverages human expertise as structural prior—a hybrid paradigm gaining traction in safety-critical domains.

## Practical Applications
- **Forestry monitoring**: Automated assessment of tree height bias in satellite/drone imagery at scale (the paper's testbed).
- **Agricultural yield prediction**: Similar multi-spectral or RGB image analysis with agronomist decision rules.
- **Medical image triage**: Combining radiologist decision logic with VLMs for preliminary screening (e.g., flagging abnormalities).
- **Environmental compliance auditing**: Automated classification of land-use violations or deforestation using regulatory decision trees + visual perception.
- **Quality control in manufacturing**: Inspection of defects using expert rules + visual grading criteria encoded in VLM queries.

## Product & Startup Ideas
1. **ForestAI** — SaaS platform for forestry agencies: Upload satellite imagery, define bias classification rules once, auto-label at scale. Reduces government annotation backlogs by 80%.
2. **BioAssess** — Ecological monitoring for conservation NGOs: Automatically classify species health, habitat degradation, etc., using VLM + conservation biologist decision rules.

## Technical Limitations
- **VLM dependency**: The entire system is only as good as the underlying VLM's ability to perceive domain-specific visual features; if a VLM misses subtle cues (e.g., tree lean at certain angles), voting won't fix it.
- **Rule brittleness**: Requires well-defined, non-overlapping decision rules from experts; messy or ambiguous rules will propagate errors.
- **Domain specificity of training**: While D3 generalizes across rule *structures*, it may not generalize to visually dissimilar domains (trained on forestry imagery, tested on medical images, for example).
- **Voting overhead**: Multiple VLM calls per decision node increases latency and cost; unclear if simple majority voting is optimal or if weighted voting based on VLM confidence would help.
- **Limited expert validation**: No analysis of how the system performs on edge cases or adversarial examples that confound VLMs.

## Open Questions
- How does performance scale with decision tree depth and branching factor? Does voting become less effective in very deep trees?
- Can the framework be extended to **continuous predictions** (e.g., actual tree height) rather than binary bias classification?
- How sensitive is the approach to prompt engineering? Would a more sophisticated prompting strategy (chain-of-thought, few-shot examples) improve per-node accuracy?
- Can uncertainty estimates from the VLM be incorporated into voting (e.g., confidence-weighted voting) rather than simple majority?
- How does this compare to fine-tuning a small VLM on limited expert labels vs. using a large off-the-shelf VLM with no domain training?
- Could the framework handle rules that require temporal sequences of images (e.g., growth bias over seasons)?

## Key Takeaway
By orchestrating vision-language models within expert-defined decision trees and using multi-agent voting, you can automate expert annotation tasks at scale while preserving human interpretability and reducing labeling burden—without rewriting code for each new domain's rules.
