# TREK: Distill to Explore, Reinforce to Refine

**Authors:** Yuanda Xu, Zhengze Zhou, Kayhan Behdin, Jelena Markovic-Voronov, Hejian Sang et al.
**Published:** 2026-07-06 | **arXiv:** [2607.05339v1](http://arxiv.org/abs/2607.05339v1)

## Summary
This paper addresses a critical bottleneck in reinforcement learning for language models: when a model lacks the internal capability to solve hard problems, standard policy optimization (like GRPO) gets stuck because it can't explore beyond what the model already knows. TREK solves this by first using distillation from a teacher to expand the model's solution repertoire, then refining those solutions with standard RL—achieving significant improvements on mathematical reasoning (AIME benchmarks) and agent tasks without requiring access to the teacher's internals.

## Theoretical Background
The paper builds on **Group Relative Policy Optimization (GRPO)**, a recent on-policy reinforcement learning method for LLMs that compares model outputs within groups to improve reasoning. GRPO works well when the model's current sampling distribution already contains useful solution paths, but fails on "hard prompts" where correct solutions lie outside the model's natural output distribution—a problem known as **distributional mismatch** or **support collapse**.

TREK combines two classical ideas: **forward-KL distillation** (pulling the student's probability distribution toward a teacher's high-quality solutions) and **exploration-then-exploitation** (first expand the solution space, then optimize within it). The key insight is treating distillation not as imitation learning (copying the teacher) but as **support expansion**—ensuring the student can at least assign non-negligible probability to correct solution modes before RL refinement begins.

## Core Contributions
1. **Diagnosis of GRPO's failure mode**: Clearly identifies that GRPO stalls on hard prompts because correct solutions have near-zero probability under the current policy, making gradient-based refinement impossible.

2. **TREK procedure**: A simple two-stage algorithm—(a) identify hard prompts via low pass rate, query teacher for verified solutions, rank by student likelihood, apply short forward-KL phase; (b) resume standard on-policy GRPO. Crucially, it only requires verified outputs, not internal teacher access.

3. **Teacher-agnostic design**: Works with external black-box teachers (e.g., DeepSeek-V4), white-box teachers, or "self-context" variants where the model uses additional inference-time information (prompts itself with more reasoning steps).

4. **Selective consolidation**: Identifies which hard-prompt samples are worth consolidating without needing teacher internals—enabling efficient use of limited teacher queries.

5. **Empirical gains on reasoning and agents**: Concrete improvements on AIME 2024/2025 (e.g., Qwen3-8B AIME 2025: 36.9→40.3) and agent tasks (ALFWorld: 75.8→82.8, ScienceWorld: 12.5→26.7).

## Connections to Other Work
- **[[GRPO]]** (Group Relative Policy Optimization): TREK is a direct extension that solves GRPO's distributional support problem.
- **[[RLHF]]** and distillation-based training: TREK reframes distillation as exploration rather than imitation, contrasting with supervised fine-tuning paradigms.
- **Curriculum learning and hard-example mining**: The idea of identifying and strategically targeting hard samples echoes curriculum learning but in an RL context.
- **Mixture-of-experts and ensemble methods**: The external teacher variant resembles leveraging stronger models to guide weaker ones, similar to ensemble distillation.
- **Off-policy RL and importance sampling**: The forward-KL phase ensures on-policy RL can then safely operate on expanded support—related to off-policy correction ideas.

## Practical Applications
1. **Mathematical problem solving**: Improving LLM performance on competition math (AMC, AIME, IMO-level problems) where models naturally lack diverse solution strategies.

2. **Tool-use agents**: Enhancing agents navigating complex environments (ALFWorld, ScienceWorld) where exploration is limited by the model's action distribution.

3. **Code generation and synthesis**: Expanding the model's prior on correct implementations before fine-tuning—useful when the model rarely samples working code on first try.

4. **Autonomous reasoning systems**: Multi-step reasoning tasks (e.g., scientific hypothesis generation, theorem proving) where intermediate steps often fall outside the model's natural distribution.

5. **Personalized tutoring systems**: Using teacher models to guide student models toward pedagogically sound problem-solving strategies, then reinforcing them via environment feedback.

## Product & Startup Ideas
1. **"Math Tutor Pro"**: An app that uses TREK-like staged training to unlock LLMs' reasoning on hard competition math, positioning it as a study aid for AIME/IMO aspirants. Revenue via subscription tiers for harder problem sets.

2. **"Agent OS"**: A platform enabling enterprises to deploy reasoning-heavy agents (legal research, scientific discovery) by intelligently combining weak on-premise models with strong cloud teachers via TREK-style distillation without exposing sensitive data.

3. **"Adaptive Coding Assistant"**: A real-time IDE plugin that identifies coding tasks the user's local/fine-tuned model struggles with, queries an external stronger model for solutions, expands the local model's capability, then refines on user feedback—reducing cloud inference costs.

4. **"Research Assistant SaaS"**: A service for academics and scientists that uses TREK to adaptively improve domain-specific LLMs (e.g., for literature synthesis, experimental design) by distilling from expert systems, then reinforcing on domain feedback.

5. **"Certification Test Prep"**: Automated preparation platform for professional exams (CPA, bar exam, medical boards) that uses TREK to iteratively expand model capabilities on known hard question types, then validates against practice tests.

## Technical Limitations
1. **Dependence on a strong teacher**: While "teacher-agnostic," TREK still requires access to a high-quality proposal source (external teacher or self-context). For domains where no strong teacher exists, the method degrades.

2. **Verification overhead**: The method assumes verified outputs are available (e.g., via formal verification or ground-truth labels). On tasks where verification is expensive or ambiguous (e.g., open-ended creative writing), applicability is limited.

3. **Hard prompt identification heuristic**: Using "low pass rate" as a signal for hard prompts is pragmatic but noisy—some low-pass prompts may be near-solvable with better exploration, while others are genuinely out-of-distribution. The paper doesn't deeply analyze this threshold.

4. **Hyperparameter sensitivity**: The top-$r$ ranking, forward-KL phase duration, and the decision of when to resume GRPO introduce hyperparameters whose tuning may be problem-dependent.

5. **Scalability to very large models**: Experiments use up to Qwen3-72B; unclear how the two-stage approach scales to larger models or how computational costs compare to alternatives.

6. **Generalization across domains**: Results are strongest on math and agent tasks; unclear how well TREK transfers to other hard-exploration domains (e.g., long-context retrieval, code synthesis with rare patterns).

## Open Questions
1. **What is the minimal teacher quality required?** Can TREK work with weaker teachers, and does teacher quality degrade gracefully?

2. **How sensitive is TREK to the forward-KL phase duration and rank-$r$ selection?** Are there principled (non-empirical) ways to set these?

3. **Can TREK be integrated with other RL methods beyond GRPO** (e.g., DPO, IPO, or actor-critic methods)?

4. **How does TREK compare to other hard-exploration strategies**, such as curriculum learning, adversarial prompting, or fine-grained reward shaping?

5. **Why does TREK achieve high success rates early in training?** Is there a theoretical characterization of the sample complexity benefit?

6. **Can the method be extended to multi-agent or competitive settings** where different agents might benefit from different teachers?

## Key Takeaway
TREK solves the exploration bottleneck in RL for LLMs by using distillation to expand the model's solution support before standard policy optimization, enabling dramatic improvements on hard reasoning and agent tasks without requiring teacher internals.
