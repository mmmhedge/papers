# Bad Memory: Evaluating Prompt Injection Risks from Memory in Agentic Systems

**Authors:** Soham Gadgil, David Alexander, Sai Sunku, Franziska Roesner
**Published:** 2026-07-16 | **arXiv:** [2607.14611v1](http://arxiv.org/abs/2607.14611v1)

## Summary
This paper investigates how agentic systems that store persistent memory across sessions become vulnerable to prompt injection attacks — a type of adversarial manipulation where malicious instructions are hidden in files that the agent will read later. The researchers tested two commercial systems (Claude and GPT-based agents) and found that while it's hard to trick agents into corrupting their own memory, pre-planted malicious payloads in memory files can successfully manipulate agent behavior across multiple sessions. This reveals a new security risk as AI agents become more autonomous and adaptive.

## Theoretical Background
Prompt injection attacks exploit how language models (neural networks trained to predict and generate text) respond to instructions embedded in their input. Traditional prompt injection targets the immediate input during a single interaction. This work extends that threat model to memory-augmented agents — systems that maintain persistent files containing behavioral preferences, knowledge bases, and session history. The key innovation is recognizing that agents read from these files as part of their normal operation, creating an attack surface that previous security research hadn't fully characterized. The underlying assumption is that agents will process memory as trusted input rather than untrusted data.

## Core Contributions
1. **First systematic evaluation** of prompt injection risks specific to persistent memory in agentic systems (not just single-turn attacks).
2. **Cross-system comparison** showing that attack success varies significantly across Claude and GPT-based agents, models, and adversarial objectives.
3. **Empirical demonstration** that payload persistence across multiple sessions is achievable even when direct memory corruption is difficult.
4. **Threat model revision**: showing that memory fundamentally changes the attack surface for prompt injection in ways that static, single-session defenses cannot address.

## Connections to Other Work
This paper extends the threat model established in earlier [[prompt injection]] research (which focused on single-turn manipulation) to the more realistic scenario of autonomous agents. It relates closely to [[alignment]] concerns around long-horizon agent behavior and [[mechanistic-interpretability]] questions about how agents use persisted instructions. The work also connects to broader agent robustness literature and complements research on [[RLHF]] and agent training, since memory-based adaptation could be seen as a form of implicit learning that bypasses formal training.

## Practical Applications
- **Enterprise AI assistants**: Systems like GitHub Copilot or enterprise knowledge assistants that maintain per-user preferences and learned behaviors become targets if an attacker can inject malicious instructions into shared or user-controlled memory files.
- **Autonomous customer service agents**: Bots that personalize behavior based on historical interactions could be manipulated by customers planting malicious directives.
- **Robotic process automation (RPA)**: Workflow agents that read persistent task configurations become vulnerable if those configs are accessible to adversaries.
- **Collaborative multi-agent systems**: One compromised agent's memory could poison the behavior of other agents that read shared knowledge bases.

## Product & Startup Ideas
1. **Memory-aware agent security scanner**: A tool that audits persistent memory files in deployed agentic systems to detect planted prompt injection payloads and flag risky patterns.
2. **Trusted memory layer for agents**: A middleware that cryptographically validates memory file integrity and enforces read/write access controls specific to agent operations.
3. **Memory sandbox and replay tester**: A platform that simulates multi-session agent behavior against adversarially polluted memory to test robustness before deployment.
4. **Agent behavior attestation service**: A service that certifies whether an agent's actions across sessions can be traced back to legitimate memory updates vs. injected payloads.
5. **Prompt injection detection as a service**: API that analyzes memory file contents for hidden adversarial instructions before agents consume them, similar to malware scanning for agent systems.

## Technical Limitations
- **Sandboxed evaluation**: Tests occur in a controlled synthetic workspace, not production environments where attack complexity may differ.
- **Limited model coverage**: Only four models tested (Claude Haiku, Opus, GPT-5.2, GPT-5.5); behavior likely varies across older/smaller models and closed-source competitors.
- **Binary success metrics**: The paper doesn't quantify partial or degraded attacks, or measure how subtle injected instructions must be to evade detection.
- **No defense evaluation**: While defenses are motivated, the paper doesn't propose or test concrete mitigation strategies at scale.
- **Single-agent assumption**: Most experiments focus on single agents; multi-agent attack cascades or coordinated memory poisoning isn't deeply explored.
- **Assumption of file-based memory**: Newer systems might use vector databases, encrypted stores, or other mechanisms that could change the threat surface.

## Open Questions
1. Can an attacker inject instructions that *persist stealthily* across sessions while remaining undetectable by the agent itself or external monitors?
2. How do defenses that validate memory integrity or isolate trusted vs. untrusted memory regions perform in practice without breaking agent adaptation?
3. Does fine-tuning agents on poisoned memory during the training phase pose a different (potentially worse) threat than runtime injection?
4. How do multi-session attack strategies evolve? Can an attacker gradually modify memory to inch toward a goal without triggering detection?
5. What memory architectures or formats (vector embeddings, semantic hashing, etc.) are more or less vulnerable to prompt injection?

## Key Takeaway
Persistent agent memory creates a new attack surface where malicious instructions planted in files can manipulate behavior across sessions.
