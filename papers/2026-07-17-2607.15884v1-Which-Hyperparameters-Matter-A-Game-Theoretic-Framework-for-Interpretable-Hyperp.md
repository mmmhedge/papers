# Which Hyperparameters Matter? A Game-Theoretic Framework for Interpretable Hyperparameter Sensitivity Analysis

**Authors:** Nyi Nyi Aung, Heepeom Shin, Abigail Lawlor, Adrian Stein
**Published:** 2026-07-17 | **arXiv:** [2607.15884v1](http://arxiv.org/abs/2607.15884v1)

## Summary
This paper presents a game-theoretic framework for understanding which hyperparameters matter most in neural network training. Instead of proposing a new optimization algorithm, the authors use Shapley Effects (a fairness concept from cooperative game theory) and Pareto front analysis to identify which hyperparameters most influence model performance across multiple competing objectives. The work helps practitioners reduce hyperparameter search spaces and make smarter early-stage model choices.

## Theoretical Background
Hyperparameter tuning is the process of selecting configuration values (learning rate, batch size, etc.) that control how a neural network trains. Traditionally, practitioners use grid search or random search—expensive brute-force methods. Shapley Effects originate from cooperative game theory and measure each "player's" (hyperparameter's) contribution to a collective outcome, accounting for all possible coalitions. A Pareto front is the set of solutions where improving one objective necessarily worsens another—no single configuration dominates all others. Multi-objective optimization acknowledges that real models often compete between accuracy, speed, and memory use rather than optimizing a single metric.

## Core Contributions
1. **Game-theoretic reframing**: Treats hyperparameter tuning as a coalitional game where Shapley Effects quantify each hyperparameter's "fair share" of influence on model performance.
2. **Objective-aware sensitivity**: Identifies which hyperparameters matter most *for different objectives separately*, not just overall performance—revealing that a hyperparameter critical for accuracy may be irrelevant for inference speed.
3. **Pareto-informed search**: Uses Pareto fronts to narrow the effective hyperparameter space and support early-stage model evaluation without full training.
4. **Empirical validation**: Demonstrates the framework across three neural network architectures and multiple problem domains.

## Practical Applications
1. **Neural architecture design**: ML engineers can identify which architectural hyperparameters (dropout rate, layer size) genuinely matter before investing in expensive hyperparameter grids.
2. **Resource-constrained training**: Teams with limited compute can focus tuning effort on high-impact hyperparameters identified by Shapley Effects.
3. **Multi-objective model selection**: In production, practitioners often balance accuracy against latency or memory. This framework reveals which hyperparameters control which trade-offs.
4. **Transfer learning**: Practitioners migrating a model to a new domain can quickly identify which hyperparameters to retune versus which to freeze.
5. **Ablation studies**: Provides a principled, quantitative alternative to manual ablation in papers and internal R&D.

## Product & Startup Ideas
1. **Hyperparameter Analysis SaaS**: A tool that ingests training logs and automatically produces Shapley-based sensitivity reports and Pareto front visualizations, reducing hyperparameter tuning time by 50%.
2. **AutoML interpretability layer**: Wrap existing AutoML systems with this framework to explain which hyperparameters drove the final model choice—building trust in automated decisions.
3. **Multi-objective model marketplace**: A platform where users upload models with Pareto front + Shapley analysis baked in, enabling end-users to pick models matching their specific latency/accuracy/memory trade-off needs.
4. **Embedded hardware optimization tool**: Specialized for edge ML, showing data scientists exactly which hyperparameters to sacrifice for mobile/IoT deployments without extensive retraining.
5. **Enterprise ML governance**: A dashboard for large teams showing Shapley sensitivity across model types and domains, surfacing unintuitive hyperparameter interactions and standardizing tuning best practices.

## Technical Limitations
1. **Computational cost**: Computing Shapley Effects requires evaluating many hyperparameter coalitions; scalability to very high-dimensional hyperparameter spaces (100+) is unclear.
2. **Pareto front approximation**: In complex multi-objective landscapes, the Pareto front is expensive to estimate; the paper doesn't detail how many evaluations are needed for reliable approximation.
3. **Limited to global analysis**: The framework identifies which hyperparameters matter overall but doesn't directly guide the next hyperparameter value to try (unlike Bayesian optimization).
4. **Training variance**: Neural network training is stochastic; the paper doesn't discuss how to handle noise when computing Shapley Effects—a single evaluation may not reflect true hyperparameter influence.
5. **Application scope**: Demonstrated only on neural networks; unclear how well the framework transfers to tree-based or kernel methods where hyperparameter interactions differ fundamentally.
6. **Objective selection**: The choice of which objectives to optimize is user-driven; the framework doesn't help practitioners identify the *right* objectives to care about.

## Open Questions
1. How does computational cost scale with the number of hyperparameters, and what's the practical ceiling for this approach?
2. Can the framework be extended to sequential decision-making (e.g., which hyperparameters should you tune *first* given a budget)?
3. How robust are Shapley Effects to training noise and stochasticity in neural networks? What sample sizes are needed?
4. Can this framework transfer to or provide insights for other domains (reinforcement learning, LLM fine-tuning, AutoML systems)?
5. How do Shapley Effects behave when hyperparameters have complex nonlinear interactions or conditional dependencies?
6. Can practitioners use Shapley-identified low-impact hyperparameters as a regularization signal (e.g., fixing irrelevant hyperparameters to reduce overfitting)?

## Key Takeaway
Game theory reveals which hyperparameters genuinely drive performance, helping teams optimize smarter and faster.
