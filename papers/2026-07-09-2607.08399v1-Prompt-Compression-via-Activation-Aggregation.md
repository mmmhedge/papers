# Prompt Compression via Activation Aggregation

**Authors:** Thibaud Ardoin, Semira Einsele, Evis Bregu, Gerhard Wunder
**Published:** 2026-07-09 | **arXiv:** [2607.08399v1](http://arxiv.org/abs/2607.08399v1)

## Summary
This paper shows that task-relevant information in instruction prompts can be compressed into a single activation vector—a learned weighted sum of hidden representations from a middle layer—and re-injected at an early layer with minimal accuracy loss (under 2%). The technique reduces computational cost for fixed instruction prompts and reveals how information flows across layers in large language models, suggesting that mid-layer representations are compatible with early-layer processing.

## Theoretical Background
Large language models (LLMs) process text as sequences of tokens through dozens of stacked neural layers, with each layer transforming activations—internal numerical representations—based on learned weights. Activation vectors are the hidden states computed at intermediate steps of this process. Prior work on [[mechanistic interpretability]] has shown that LLMs encode semantic and task-relevant information distributed across their internal representations. This paper builds on the assumption that information flow in neural networks is not strictly sequential: if mid-layer activations encode task intent, they might be shiftable earlier in the network. The work also relates to [[prompt engineering]] and efficiency research, where reducing the number of tokens or computational steps per inference is valuable.

## Core Contributions
1. **Empirical validation** that a single learned weighted-sum activation vector can replace a multi-token instruction prompt with <2% relative accuracy drop—a non-obvious result showing information can be substantially compressed.
2. **Evidence of cross-layer compatibility**: mid-layer representations transfer meaningfully to early layers, implying the model's internal encoding is more fluid than a strictly feedforward interpretation suggests.
3. **Quantification of semantic information density**: demonstrates that a single activation vector encodes recoverable semantic information, and weighted sums are robust compressors of this information.
4. **Practical inference optimization**: eliminates token reprocessing for fixed instruction prompts, enabling faster per-query computation.

## Connections to Other Work
This work connects to several research threads:
- **[[Mechanistic interpretability]]**: similar spirit to probing work and activation patching, but applied to compression and efficiency.
- **Prompt caching and prefix tuning**: related efficiency techniques, but this method uses learned aggregation rather than caching full sequences.
- **Knowledge distillation and model compression**: related problem of representing complex information compactly, though applied to activations rather than weights.
- **Sparse activation methods**: related to the observation that information can be concentrated in specific representations.

## Practical Applications
1. **API-based LLM services**: cache compressed system prompts (e.g., "You are a Python expert") once per session and reuse the activation vector, cutting per-query latency and compute for millions of inference calls.
2. **Mobile and edge deployment**: reduce token processing overhead on device-constrained systems by pre-computing compressed instruction representations.
3. **Real-time dialogue systems**: speed up response generation in chatbots by compressing context or system instructions into single vectors.
4. **Document analysis pipelines**: compress lengthy instruction sets for batch document processing without re-tokenizing.
5. **Multi-task inference**: store compressed task descriptions as activation vectors and switch between them without token reprocessing overhead.

## Product & Startup Ideas
1. **LLM Inference Optimizer SaaS**: package this technique as a middleware service that profiles and compresses prompts for cost reduction on cloud LLM APIs (OpenAI, Anthropic), selling to enterprise users paying per-token.
2. **Prompt Caching Layer**: build a cached activation-store for LLMs, selling to companies running repeated queries with fixed instructions (customer support, content moderation, document classification).
3. **Edge LLM Runtime**: create a lightweight inference engine that accepts pre-compressed activation vectors, enabling sub-millisecond system-prompt injection on mobile devices.
4. **Enterprise Prompt Management Platform**: provide tooling to compress, version, and deploy instruction prompts across internal LLM deployments, with analytics on compression quality vs. accuracy.
5. **Custom Model Optimization**: offer a service to fine-tune the activation aggregation weights for a company's specific LLM deployment and instruction set, guaranteeing sub-2% accuracy loss and measurable latency gains.

## Technical Limitations
1. **Task and model specificity**: the learned weights for activation aggregation are likely model- and task-specific; generalization across model families or to novel tasks is unexamined.
2. **No comparison to simpler baselines**: unclear whether the learned weighted sum outperforms simpler methods (e.g., equal weighting, PCA on activations, or extracting a single layer's representation).
3. **Limited scale and diversity**: evaluation on a narrow set of prompts or LLM sizes; robustness to very long instructions or very large models unknown.
4. **Injection layer choice**: the paper doesn't thoroughly explore sensitivity to the early layer chosen for re-injection; this could be a critical hyperparameter.
5. **Dynamic prompt scenarios**: the method is framed for *fixed* instructions; adapting to variable or user-customized prompts remains unclear.
6. **Attribution and interpretability**: while activations are compressed, the paper doesn't offer insight into *which* semantic features are preserved or lost, limiting debugging and improvement.

## Open Questions
1. Can the learned aggregation weights transfer across model sizes or architectures, or do they require per-model retraining?
2. How does compression quality degrade as instruction prompts become longer or more complex?
3. What is the optimal early layer for re-injection, and does it depend on model depth or task type?
4. Can this technique combine with other efficiency methods (e.g., quantization, pruning, distillation) without compounding accuracy loss?
5. How much of the semantic information loss (the ~2% accuracy drop) is recoverable via post-hoc techniques like in-context examples or prompt refinement?
6. Does the activation vector encode sufficient information to support multi-turn conversations, or does the compressed representation degrade with context length?

## Key Takeaway
Task instruction can be distilled from a sequence of tokens into a single learned activation vector with minimal accuracy loss, revealing that LLM information flow is more cross-layer compatible than token processing order suggests, with practical implications for inference speed and cost.
