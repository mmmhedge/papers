# AI Tour Meeting: Group Travel Planning by LLM Agents

**Authors:** Daisuke Kikuta
**Published:** 2026-07-21 | **arXiv:** [2607.18806v1](http://arxiv.org/abs/2607.18806v1)

## Summary
This paper presents AI Tour Meeting, a framework where multiple LLM-based agents with different personas negotiate together to plan a group trip. Each agent has its own constraints and preferences, and they discuss options in natural language until they reach agreement on an itinerary. The work is useful for studying how LLM agents collaborate and make group decisions in realistic scenarios.

## Theoretical Background
The work builds on two main foundations: (1) Large language models (LLMs) — neural networks trained on vast text data that can generate coherent text and follow instructions — and (2) multi-agent systems — frameworks where multiple autonomous agents interact to solve problems together. Prior work has shown LLMs can role-play, follow persona instructions, and engage in dialogue. The paper extends this by creating a structured environment where agents with conflicting goals must negotiate, which is a classic problem in multi-agent coordination and game theory.

## Core Contributions
- A practical framework for orchestrating multi-agent LLM discussions with configurable personas, workflows, and monitoring tools (not just a one-off demonstration)
- System design that makes it easy to set up group negotiation scenarios without heavy engineering
- Empirical analysis of how LLM agents actually behave when forced to compromise and coordinate in travel planning — showing patterns in agreement, negotiation dynamics, and decision-making
- Demonstration that the framework can serve as a simulation sandbox for studying agent behavior

## Connections to Other Work
This relates to [[multi-agent-systems]] research, which traditionally focused on symbolic agents and formal protocols. It also connects to work on [[large-language-models]] as autonomous agents (e.g., agents that can plan and execute tasks). The group decision-making aspect echoes social choice theory and consensus problems. The framework is practical infrastructure similar to how simulation platforms enable RL research — it enables reproducible study of agent interactions.

## Practical Applications
- **Travel agencies and booking platforms:** Automating itinerary generation for group trips by balancing preferences (e.g., budget, activities, pacing)
- **Event planning software:** Multi-stakeholder coordination where organizers, venues, and attendees have conflicting preferences
- **Corporate meeting scheduling:** Negotiating meeting times and agendas across teams with different constraints
- **AI-powered concierge services:** Personal assistants that negotiate on behalf of users in group settings
- **Research and simulation:** Studying how LLM agents behave under resource constraints and conflicting goals

## Product & Startup Ideas
1. **GroupTrip.ai** — SaaS platform where users input their travel preferences, and an AI agent team plans optimized itineraries, presenting recommendations with trade-off explanations
2. **AI Meeting Coordinator** — Enterprise tool that automatically negotiates meeting times, room bookings, and agendas across distributed teams by running agent discussions
3. **Consensus Engine** — B2B API for any multi-stakeholder decision problem (e.g., supply chain routing, resource allocation) where agents represent different departments
4. **LLM Agent Playground** — Developer platform for designing, running, and analyzing multi-agent negotiation simulations; used for game design, policy testing, or organizational modeling
5. **Smart Event Planner** — Consumer app for group events (dinners, trips, parties) where users add constraints and AI agents negotiate the final plan, reducing back-and-forth communication

## Technical Limitations
- **Scalability:** Framework's performance with large groups (10+ agents) is unclear; negotiation may become chaotic or slow
- **Convergence:** No guarantee agents reach agreement; may loop or deadlock without human intervention
- **Preference modeling:** Personas are manually configured; real user preferences are nuanced and change over time
- **Cost:** Each agent requires LLM API calls; group planning becomes expensive at scale
- **Reproducibility:** LLM outputs are stochastic; same scenario may produce different outcomes, making it hard to reliably replicate negotiations
- **Bias in negotiation:** LLMs may have built-in biases that favor certain types of agents or preferences
- **Fairness:** Framework doesn't guarantee fair outcomes or prevent dominant agents from monopolizing decisions

## Open Questions
- How do agent personas affect negotiation dynamics? (e.g., does "budget-conscious" agent always lose?)
- Can the framework detect when agents are in deadlock, and propose automated conflict resolution?
- How does conversation length and depth scale with group size?
- Do agents converge to similar solution quality across multiple runs, or is variance too high for real use?
- How would the framework handle real user preferences, not manually designed personas?
- Can the system explain *why* the final itinerary was chosen (i.e., show the negotiation trace and trade-offs)?
- Does agent behavior generalize across different planning domains (e.g., event catering, project scheduling)?

## Key Takeaway
LLM agents with distinct personas can negotiate together to plan shared trips, creating a sandbox for studying multi-agent coordination.
