# Linguistic Firewall: Geometry as Defense in Multi-Agent Systems Routing

**Authors:** Dvir Alsheich, Adar Peleg, Ben Hagag, Rom Himelstein, Amit Levi et al.
**Published:** 2026-06-29 | **arXiv:** [2606.30555v1](http://arxiv.org/abs/2606.30555v1)

## Summary
This paper introduces ANTAP, a routing architecture for multi-agent systems that selects which agent should handle a task by directly testing their capabilities rather than trusting their self-descriptions. The key innovation is using geometric projections in a non-textual semantic space to make routing decisions, which creates a "linguistic firewall" against adversarial attacks where malicious agents misrepresent their abilities or inject harmful instructions.

## Theoretical Background
Multi-agent systems coordinate multiple specialized agents to solve complex problems. Traditional routing mechanisms rely on textual proxies — agent self-descriptions or learned embeddings — to estimate which agent is best suited for a task. However, this creates a security problem: the paper identifies that agents can lie about their capabilities or embed hidden vulnerabilities that traditional defenses miss. The authors frame this as a gap between *declared* vs. *actual* competence. Their solution draws on dynamic testing (empirically probing agent behavior) and geometric representation learning — the idea that agent capabilities can be encoded as fixed behavioral operators in a shared mathematical space, enabling algebraic routing decisions independent of natural language.

## Core Contributions
1. **Problem Formulation**: Formally identifies the vulnerability in description-based and embedding-based routing in LLM-driven multi-agent systems, distinguishing between metadata injection attacks and adaptive embedding attacks.
2. **ANTAP Architecture**: An evaluation-driven router that actively queries agents on probe tasks to build a non-textual behavioral representation, sidestepping textual vulnerabilities entirely.
3. **Linguistic Firewall**: A routing mechanism using purely algebraic projection in a learned semantic space, making it impossible to express attack intentions through text or metadata.
4. **Empirical Security Results**: Demonstrates near-zero attack success rate (ASR) against description injection (vs. 67.3%+ for baselines) and 20% lower ASR than embedding baselines against adaptive attacks.

## Connections to Other Work
This work sits at the intersection of [[reinforcement-learning]] (agent capability assessment via behavioral probing), adversarial robustness in [[large-language-models]], and security in [[multi-agent-systems]]. Related work likely includes prompt injection defenses, agent capability modeling, and geometric representation learning; ANTAP's novelty is combining active testing with geometric routing to eliminate the textual attack surface entirely. The "linguistic firewall" concept mirrors ideas in mechanistic interpretability about separating semantic meaning from surface form.

## Practical Applications
1. **Enterprise Workflow Orchestration**: Companies using internal LLM agent networks (e.g., document processing, customer service routing) can defend against rogue employees or compromised agents misrepresenting their skills.
2. **Federated AI Marketplaces**: Platforms where external agents offer services (translation, image analysis, summarization) need to verify claims; ANTAP allows safe integration of untrusted agents.
3. **Healthcare AI Coordination**: Routing diagnostic tasks across multiple specialized medical AI agents without trusting their self-reported accuracy, reducing risk of misattribution.
4. **Autonomous Logistics Networks**: Routing tasks across autonomous systems (robots, vehicles) where physical or computational agents might be compromised or misconfigured.

## Product & Startup Ideas
1. **Secure Agent Router-as-a-Service**: A hosted API that manages routing for multi-agent LLM applications, automatically testing and certifying agent capabilities; charge per routing decision or per agent verified.
2. **Agent Auditing & Compliance Platform**: Enterprise tool that continuously re-tests agent capabilities, generates compliance reports, and alerts on capability drift or suspicious behavior changes.
3. **LLM Agent Marketplace with Verified Credentials**: Marketplace (similar to worker platforms) where LLM agents offer services, but buyers can query a shared ANTAP-validated capability profile before hiring.
4. **Agent Honeypots & Security Monitoring**: A monitoring layer that deploys probe tasks as "canaries" to detect when agents are behaving abnormally or attempting to exfiltrate information.
5. **Geometric Agent Embeddings Library**: Open-source or commercial toolkit for converting arbitrary agent behavior into certified geometric representations, compatible with any routing framework.

## Technical Limitations
1. **Probe Task Design**: ANTAP's security depends on the breadth and quality of probe tasks; adversaries with foresight could train to game the specific probes used without revealing general incompetence.
2. **Computational Overhead**: Active testing every agent before each routing decision introduces latency and cost; scalability to thousands of agents or high-frequency routing is unclear.
3. **Semantic Space Assumptions**: The paper assumes agents can be meaningfully embedded in a shared semantic space; highly heterogeneous agent types (symbolic solvers, vision models, language models) may not compress well.
4. **Adaptive Adversaries**: While the paper tests against embedding attacks, a determined adversary with knowledge of the probe set and learning algorithm might learn to pass probes while failing on real tasks.
5. **Cold-Start Problem**: New agents require a full probe suite before routing; no mechanism for incremental trust building or handling novel agent types.

## Open Questions
1. How does probe task selection impact security and performance? Is there a principled way to design probes that are both comprehensive and resilient to adversarial training?
2. Can ANTAP scale to hundreds or thousands of agents with acceptable latency? What is the minimum probe set size needed for security guarantees?
3. How does ANTAP perform on agents that are *genuinely* weak or failing in ways unrelated to adversarial behavior? Does it correctly distinguish incompetence from malice?
4. Can the geometric representation be updated incrementally as agents operate, or must full re-testing occur periodically?
5. What happens when agents are interdependent — when one agent's task quality affects downstream agents' behavior?

## Key Takeaway
By replacing textual self-descriptions with empirically-tested behavioral representations, ANTAP eliminates the attack surface that makes current multi-agent routers vulnerable to adversarial agents misrepresenting their capabilities.
