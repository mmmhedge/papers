# Test-Time Training for Modality Order Consistency in Vision-Language Models

**Authors:** Aditi Gupta, Yossi Gandelsman
**Published:** 2026-07-22 | **arXiv:** [2607.20351v1](http://arxiv.org/abs/2607.20351v1)

## Summary
Vision-language models (like CLIP or LLaVA) perform differently depending on whether you show them the image first or ask the question first — a semantically meaningless change that shouldn't matter. This paper identifies this "modality order" bug, uses activation patching to locate it in mid-network layers, and proposes a simple test-time training fix that closes the performance gap and even improves both orderings.

## Theoretical Background
Vision-language models combine visual encoders (like ViT — Vision Transformers) with language models to understand images and answer questions about them. Prior work has documented various brittleness issues in these models, including sensitivity to prompt formatting and spurious correlations. This paper builds on mechanistic interpretability techniques — methods for understanding where and why neural networks fail by examining internal representations and information flow — particularly activation patching, which isolates causal contributions of specific network regions.

## Core Contributions
1. **Systematic documentation**: Demonstrates across three major VLMs and three benchmarks that image-first prompting consistently outperforms question-first (e.g., 58% vs. 51% on one benchmark).
2. **Circuit localization**: Uses activation patching to pinpoint the failure to a narrow mid-network region (specific transformer layers) where representations diverge sharply.
3. **Test-time training solution**: Proposes an asymmetric adapter that aligns representations between orderings without retraining the base model; this fixes both the weaker ordering *and* improves the stronger one.
4. **Mutual consistency**: Shows that the method bootstraps both orderings toward agreement, suggesting the fix addresses a genuine representational misalignment rather than just fixing one branch.

## Practical Applications
1. **Chatbot robustness**: VLMs powering image-aware assistants (e.g., Claude vision) can be made more reliable regardless of input order.
2. **Autonomous systems**: Robots and embodied agents using VLMs for scene understanding benefit from order-invariant perception.
3. **Medical imaging**: Diagnostic tools using VLMs gain robustness by closing this gap, reducing spurious performance swings.
4. **Content moderation**: Platforms analyzing images + reports together become less sensitive to input ordering artifacts.

## Product & Startup Ideas
1. **VLM robustness-as-a-service**: Wrap any VLM API with test-time training adapters that correct order sensitivity in real-time for higher, more consistent accuracy.
2. **Multimodal model debugger**: SaaS tool that detects and auto-corrects order/format sensitivity bugs in custom VLMs before deployment.
3. **Fine-tuning optimizations**: Sell efficient test-time training adapters (minimal memory, low latency) for edge deployment of robust VLMs.
4. **Benchmark auditing platform**: Help organizations audit their VLMs for modality-order and similar circuit-level failures with interactive visualization of activation misalignment.
5. **Embodied AI middleware**: Standardized adapter layer for robotics platforms that ensures vision-language reasoning is consistent across input orderings.

## Technical Limitations
1. **Test-time compute overhead**: Method requires inference-time adaptation, adding latency and compute cost per query (barrier for real-time systems).
2. **Limited scope**: Only addresses one specific failure mode; other order/format sensitivities may persist.
3. **Generalization unclear**: Not tested on held-out models or architectures beyond the three evaluated; transferability unknown.
4. **Activation patching assumptions**: Relies on the assumption that mid-network misalignment is the *root cause* rather than a symptom of earlier representational issues.
5. **No theoretical guarantee**: Method is empirical; no guarantees that order consistency is achieved across all possible inputs or domains.

## Open Questions
1. **Why does this asymmetry exist?** Does image-first prompting leverage pre-training better? Or is there an architectural bias?
2. **Does order sensitivity correlate with other failures?** Are models with strong order bias also more vulnerable to adversarial attacks or spurious correlations?
3. **Can this be fixed at training time?** Is a simple data augmentation or objective function change sufficient, or does test-time training uniquely solve this?
4. **How does this generalize?** Does the same mid-network localization hold for other modality orders (text-image-text) or other multimodal combinations?
5. **Can we predict the gap?** Can we detect order sensitivity without running both forwards, perhaps via early-layer analysis?

## Key Takeaway
Vision-language models fail at simple input reordering; test-time training fixes it by aligning mid-network representations between prompt orders.
