# From Noisy Traces to Root Causes: Structural Trajectory Analysis and Causal Extraction for Agent Optimization

**Authors:** Ying Chang, Jiahang Xu, Xuan Feng, Chenyuan Yang, Peng Cheng et al.
**Published:** 2026-07-08 | **arXiv:** [2607.07702v1](http://arxiv.org/abs/2607.07702v1)

## Summary
This paper addresses a core challenge in agent optimization: when LLMs reflect on agent failures to improve policies, the execution traces they analyze are noisy, redundant, and contain irrelevant steps that distract from true root causes. The authors propose STRACE, a framework that filters traces at the batch level (removing redundant failures) and within each trace (identifying causally important steps via dependency analysis), enabling more precise optimization signals. On formal verification tasks, STRACE achieves a 1.4× improvement in success rate.

## Theoretical Background
The paper builds on **reflection-based agent optimization**, where LLMs act as optimizers by analyzing execution traces and diagnosing failures. This approach inherits from broader work on [[RLHF]] (reinforcement learning from human feedback) and self-improvement in language models. The core challenge mirrors problems in causal inference — distinguishing causal factors from spurious correlations — and in curriculum learning, where selecting good training examples matters. The paper uses **textual dependency graphs** to represent causal relationships between steps in a trajectory, treating the trace as a directed acyclic graph where edges represent step dependencies.

## Core Contributions
1. **Failure pattern mining at batch level**: Identifies and filters redundant failures across a large trace collection, selecting representative examples that preserve diversity while reducing data size.
2. **Causal localization within traces**: Constructs a dependency graph over trajectory steps and uses graph-based analysis to isolate causally important steps, removing irrelevant actions that naive truncation would preserve or discard incorrectly.
3. **End-to-end framework (STRACE)**: Combines both mechanisms to produce high-signal optimization contexts that lead to more effective policy improvement.
4. **Empirical validation**: Demonstrates substantial gains on VeruSAGE-Bench (formal verification), a challenging domain where agent optimization historically struggles.

## Connections to Other Work
This work sits at the intersection of agent optimization, interpretability, and causal reasoning. It relates to:
- **[[Agents]]** literature on reflection and in-context learning for policy improvement
- **[[Mechanistic-interpretability]]** — identifying which parts of a trace matter most
- **Curriculum learning** and example selection in machine learning
- **Causal inference** approaches to distinguishing root causes from confounders
- Prior work on trajectory compression and context reduction in prompt engineering

The key innovation is applying causal reasoning to trace analysis, whereas prior work often used heuristic truncation or simple filtering.

## Practical Applications
1. **LLM-based code debugging assistants**: Automatically identify root causes in execution logs and suggest fixes, reducing noise in developer-facing error messages.
2. **Automated test case optimization**: Filter and prioritize failing test cases to focus debugging effort on representative failures.
3. **Robotics and embodied AI**: Identify which steps in a failed robot trajectory actually caused failure, enabling more efficient policy learning.
4. **Formal verification tooling**: Optimize theorem-proving agents by extracting relevant proof steps from long, rambling verification attempts.
5. **Business process optimization**: Analyze process logs to identify bottlenecks and root causes, filtering out noise in complex workflows.

## Product & Startup Ideas
1. **Agent Optimization-as-a-Service**: A SaaS platform that takes user-provided agent traces and execution logs, applies STRACE-like causal extraction, and returns optimized prompts or policy updates—targeting autonomous systems companies.
2. **Enterprise Log Forensics Tool**: Sells to DevOps and security teams; automatically mines root causes from application logs by filtering noise and identifying causal dependencies, accelerating incident response.
3. **AI Debugging IDE**: An IDE plugin for prompt engineers and LLM app developers; visualizes trace dependency graphs and highlights causally important steps, making agent failures transparent.
4. **Automated Curriculum Generator for RL**: Applies dependency-based filtering to construct curricula for reinforcement learning agents by prioritizing representative, causally distinct failures.
5. **Smart QA Automation**: A test framework that uses causal trace analysis to automatically prioritize and de-duplicate failing test cases for CI/CD pipelines, reducing flaky test overhead.

## Technical Limitations
1. **Dependency graph construction**: The paper assumes traces can be converted to textual dependency graphs; this may be fragile for unstructured or ambiguously causal domains (e.g., image-based or continuous control tasks).
2. **LLM-as-judge circularity**: If the same LLM both constructs the dependency graph and optimizes the policy, errors in causal reasoning propagate; no validation against ground truth causality.
3. **Scalability to long traces**: As traces grow very long (thousands of steps), graph construction and causal analysis may become computationally expensive.
4. **Domain specificity**: STRACE is evaluated on formal verification; generalization to other domains (robotics, web tasks, etc.) is unclear.
5. **Batch-level filtering**: Mining failure patterns assumes failures are clustered or repeatable; may fail on one-off or highly diverse failure modes.

## Open Questions
1. How sensitive is STRACE to errors in the dependency graph construction? What happens if the LLM misidentifies causal edges?
2. Does STRACE generalize beyond formal verification to domains like robotics, web automation, or code generation?
3. Can the framework handle continuous or probabilistic causality (rather than binary causal/non-causal)?
4. How does STRACE compare to ground-truth causal analysis if human experts manually annotate root causes?
5. Can the dependency graph and causal extraction be learned end-to-end, rather than constructed heuristically?
6. What is the compute cost of graph construction and causal localization relative to naive filtering?

## Key Takeaway
Extracting causal structure from noisy agent traces — identifying which steps actually caused failures — enables dramatically more efficient agent optimization than naive context reduction.
