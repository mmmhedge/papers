# Algebraic Representability as the Limiting Regime of Grokking: An Exactly Solvable Model with Holomorphic Activations

**Authors:** Chon-Fai Kam, Xavier Cadet, Miloud Bessafi, Frederic Cadet
**Published:** 2026-07-15 | **arXiv:** [2607.13749v1](http://arxiv.org/abs/2607.13749v1)

## Summary
This paper studies grokking—the puzzling phenomenon where neural networks suddenly jump from memorizing training data to generalizing—by examining an extreme case where network expressivity is mathematically constrained. Using two-layer networks with special "holomorphic" (complex-analytic) activations trained on modular arithmetic, the authors prove that whether a task can be learned is determined entirely by algebraic geometry: a target is learnable if and only if its Fourier structure aligns with the network's built-in representational constraints. The key insight is that in this bottlenecked regime, grokking disappears—networks either succeed instantly or fail completely—revealing grokking as a capacity-dependent phenomenon.

## Theoretical Background
Grokking was observed in networks trained on modular arithmetic tasks (e.g., learning that (a + b) mod p from examples), where models first memorize training examples perfectly, then suddenly generalize to unseen data. Prior work attributed this to the interplay between model capacity (width/depth) and optimization dynamics.

This paper grounds the analysis in algebraic geometry—the study of solution sets to polynomial equations. The authors use holomorphic activations (activations σ(z) = z^k that are complex-differentiable), which constrain network outputs to lie in a low-dimensional algebraic variety (a geometric shape defined by polynomial constraints). The Fourier analysis of functions on cyclic groups (ℤ_p) and character theory provide the mathematical language for characterizing what tasks can be represented. This connects to classical representation theory: every function on a cyclic group decomposes into irreducible characters, and the network's algebraic confinement restricts which decompositions are achievable.

## Core Contributions
1. **Complete algebraic characterization**: A task on modular arithmetic is representable if and only if its Fourier support satisfies u + v = k (mod p), a discrete geometric criterion. This is proven, not empirically observed.

2. **Representability bounds memorization**: Non-representable targets produce a positive training loss floor independent of network width. This proves that some targets are *fundamentally unlearnable*, not just hard to optimize.

3. **Grokking disappears in the algebraic limit**: Unlike standard networks showing a grokking phase, networks in this constrained regime exhibit a binary outcome: instant success or outright failure. No memorization plateau; no delayed generalization.

4. **Continuous interpolation**: A bottleneck ablation traces the transition from this algebraic regime (instant success/failure) → memorization-only phase (grokking) → standard networks as capacity increases, unifying the phenomena.

5. **99.8% predictive accuracy**: Algebraic theory predicts outcomes across 585 experimental runs with near-perfect precision.

## Connections to Other Work
This work relates to several foundational themes:

- **[[Grokking]]** (Powerlaw et al., 2022): The original observation; this paper explains grokking as a capacity-dependent phenomenon, with the algebraic regime as its limiting case.
- **Neural network approximation theory**: Classical results (Cybenko, Hornik) characterize which functions networks can represent; this work makes that constraint *exact* via algebraic geometry.
- **Loss landscape geometry**: Understanding why networks memorize before generalizing connects to recent mechanistic work on neural network training phases.
- **Modular arithmetic learning**: Prior work has used mod-p tasks as a testbed; this is the first complete algebraic solution.
- **[[Mechanistic interpretability]]**: The algebraic characterization provides interpretable structure—the Fourier criterion is a simple, human-understandable rule.

## Practical Applications
1. **Certifiable learning**: For any task and network architecture, directly compute whether learning is possible before training begins, eliminating wasted computation.
2. **Curriculum design**: Rank tasks by representability; train networks on linearly independent tasks in order of increasing algebraic complexity.
3. **Adversarial robustness**: Identify which perturbations push tasks outside the representable subspace, making them provably unlearnable.
4. **Symbolic regression**: When fitting functions to data, the algebraic criterion identifies when a parametric model class can fit a target function exactly.
5. **Hardware-efficient networks**: Architectures with constrained expressivity (holomorphic activations, low rank, sparse) can be verified to solve specific tasks before deployment.

## Product & Startup Ideas
1. **LearnabilityCheck**: A software tool that takes a supervised learning task and neural architecture and certifies whether it's solvable before training. Target: ML ops, AutoML platforms.
2. **Task-Aware Architecture Design**: Software that, given a task's algebraic properties, recommends a minimal-capacity network guaranteed to learn it. Market: edge computing, robotics.
3. **Debugging ML Pipelines**: When a network fails to generalize, the tool checks if the task is outside the architecture's representable class (hard failure) vs. optimization failure (soft failure), guiding engineers.
4. **Polynomial Learner**: A service for discovering minimal polynomial models for tabular data using algebraic geometry; targets finance and logistics where interpretability is critical.
5. **Adversarial Task Generator**: Given a learner, automatically generate adversarial datasets that lie outside its representable subspace, enabling red-teaming and robustness certification.

## Technical Limitations
1. **Holomorphic activations are exotic**: Networks with z^k activations are not standard; results may not transfer to ReLU, sigmoid, or other practical activations (though the bottleneck ablation suggests they do).
2. **Modular arithmetic is toy territory**: The tasks are synthetic and highly structured. Real-world tasks (images, text, continuous functions) may not admit clean algebraic characterizations.
3. **Two-layer networks only**: The analysis is for shallow networks; extension to deep networks is unclear and likely involves additional algebraic layers of complexity.
4. **Finite-dimensional constraint is extreme**: The algebraic variety is a thin slice of function space (O(k/p²)); most practical networks have much higher capacity. The transition regime (where grokking actually occurs) may behave very differently.
5. **Training dynamics ignored**: The paper characterizes *what is representable*, not *how* optimization finds it. The binary success/failure outcome suggests infinitesimal changes in initialization or learning rate could flip outcomes—practically fragile.
6. **No noise or misspecification**: Analysis assumes noise-free data and that true targets lie in the algebraic variety; real data often doesn't.

## Open Questions
1. **Extension to deeper networks**: Can the algebraic variety characterization be lifted to three or more layers? Does a hierarchy of varieties emerge?
2. **Non-holomorphic activations**: What is the limiting algebraic variety for standard activations (ReLU, tanh, sigmoid)? Is it a semi-algebraic set rather than a variety?
3. **Continuous-domain tasks**: Can algebraic characterization apply to learning functions on ℝ (continuous input/output) rather than finite groups?
4. **Optimization trajectory analysis**: Why do networks in the binary regime avoid the intermediate memorization phase? Does the loss landscape geometry explain it?
5. **High-dimensional modular tasks**: Does the criterion generalize to learning functions on (ℤ_p)^d for d > 2?
6. **Grokking in the transition regime**: What algebraic conditions govern *when* grokking happens (the delayed phase length) as capacity shrinks toward the constrained regime?
7. **Robustness of the binary behavior**: How does label noise, warm-start initialization, or early stopping affect the instant success/failure boundary?

## Key Takeaway
Neural networks with constrained expressivity must either instantly solve a task or fail completely—grokking requires room to memorize before generalizing.
