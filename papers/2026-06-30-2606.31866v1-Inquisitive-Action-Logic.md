# Inquisitive Action Logic

**Authors:** Ivano Ciardelli
**Published:** 2026-06-30 | **arXiv:** [2606.31866v1](http://arxiv.org/abs/2606.31866v1)

## Summary
This paper introduces inquisitive action logic (InqAL), a formal system for reasoning about what agents can do in multi-agent settings. Unlike traditional approaches that ask "what outcomes can an agent force?", InqAL also captures "what aspects of outcomes does an agent determine through their choices?" — treating these questions as modal logic statements. The work provides a complete mathematical framework (axiomatization, decidability proofs) for reasoning about agent capabilities and determination in concurrent settings.

## Theoretical Background
The paper builds on several established logical traditions. **Modal logic** is a formal system for reasoning about possibility and necessity; multi-agent extensions allow talking about different agents' capabilities. **Neighborhood semantics** is a way to interpret modal logic where we associate each agent with a collection of possible outcome sets rather than just possible worlds. The paper extends **inquisitive logic**, which treats questions (interrogatives) as first-class logical objects, not just statements. It also relates to **coalition logic** — a framework from game theory that formalizes which coalitions of agents can force particular outcomes — and grounds itself in **concurrent game structures**, the standard formal model where multiple agents act simultaneously and their joint choices determine outcomes.

## Core Contributions
1. **Inquisitive Action Logic (InqAL)**: A novel multi-agent modal logic that explicitly represents agentive determination — not just what outcomes an agent can force, but what outcome-properties they actively determine.
2. **Technical characterization**: Proves InqAL is expressively equivalent (on single-agent statements) to the individual-agent fragment of socially friendly coalition logic, clarifying the relationship between these frameworks.
3. **Representation theorem for effectivity functions**: Establishes exact conditions under which a neighborhood frame (abstract modal structure) can be realized by a concrete concurrent game structure — i.e., when abstract "agent capabilities" correspond to actual strategic possibilities.
4. **Completeness and decidability**: Provides a full axiomatization of InqAL and proves the logic is decidable via the finite model property, making it practically verifiable.

## Connections to Other Work
InqAL sits at the intersection of several research threads:
- **Coalition logic** and **strategic ability** (what coalitions can achieve): InqAL generalizes these by distinguishing between forcing outcomes and determining outcome-features.
- **Inquisitive semantics**: Prior work treated questions in logic; InqAL applies this to action and agency — a novel direction.
- **Game-theoretic semantics**: Relates to epistemic and strategic logics used in verification, but with a focus on what agents determine rather than just what they know.
- **Temporal and dynamic logics**: Similar to [[ATL]] (Alternating-Time Temporal Logic) but with a different modal structure focused on determination rather than temporal reachability.

## Practical Applications
1. **Autonomous systems verification**: Formally specify and verify what outcome-features (safety, progress, fairness) each agent in a multi-robot or multi-vehicle system is responsible for determining.
2. **Smart contract and protocol design**: Reason about decentralized protocols (voting, mechanism design, blockchain consensus) where we need to guarantee which properties each participant (or coalition) controls.
3. **Responsibility and liability in AI**: Use inquisitive determination to formally assign responsibility — which agent's decisions determine whether a critical property (e.g., privacy, bias) holds?
4. **Game design and strategic interaction**: Analyze multiplayer games or economic systems to clarify what each player's agency actually covers versus what they only influence probabilistically.

## Product & Startup Ideas
1. **Formal verification toolkit for multi-agent systems**: SaaS platform where engineers specify concurrent protocols (autonomous vehicles, robotic swarms, distributed consensus) and automatically verify which safety/liveness properties each agent determines — using InqAL as the backend logic.
2. **Responsibility attribution for AI/autonomous systems**: Tools for insurance, compliance, and liability firms to formally decompose system failures and assign responsibility to different components/agents using inquisitive determination analysis.
3. **Smart contract security auditing**: Plug-in for Ethereum/Solana developers that translates multi-agent contracts into InqAL, then checks which contract participants can force or determine specific attack scenarios.
4. **Game theory analysis platform**: Interactive tool for economists, game designers, and negotiation analysts to visualize and reason about "what each player controls" in complex multi-agent games — more nuanced than standard game-theoretic solution concepts.
5. **Decentralized governance frameworks**: Library for designing DAOs and multi-signature schemes where stakeholders can formally verify which voting coalitions determine which governance outcomes.

## Technical Limitations
1. **Scalability**: Decidability via finite model property guarantees theoretical computability, but the actual complexity (likely PSPACE or worse) may be prohibitive for large concurrent game structures with many agents.
2. **Abstraction mismatch**: Real-world systems have probabilistic actions, incomplete information, and timing constraints; InqAL assumes deterministic concurrent choice and perfect information — mappings to reality require non-trivial approximation.
3. **Question-outcome mapping**: The paper treats determination through questions but doesn't fully address how to specify what questions (outcome-partitions) are relevant in a given domain — this requires domain expertise and isn't automated.
4. **Expressiveness vs. simplicity tradeoff**: While InqAL is expressive enough to distinguish determination from coercion, it may be too abstract for practitioners to specify real protocols without significant formalization effort.
5. **Limited treatment of dynamics**: The logic is essentially static (snapshot of one decision point); it doesn't natively handle long-horizon interactions or learning — for temporal properties, additional frameworks are needed.

## Open Questions
1. **Complexity characterization**: What is the exact computational complexity of model-checking and satisfiability in InqAL? Can it be reduced for restricted fragments?
2. **Learning and approximation**: Can agents learn or approximate which outcome-features they determine through repeated play? How does InqAL interface with reinforcement learning or empirical game theory?
3. **Incomplete information**: How does InqAL extend when agents have private information or epistemic uncertainty about which actions others will take?
4. **Quantitative extensions**: Can one layer probabilities or costs over InqAL to reason about "agent A has a 90% chance to determine outcome X" or "determining Y costs agent A $10"?
5. **Practical specification languages**: What user-friendly syntax or visual language would let practitioners (without logic training) specify multi-agent systems and automatically compile to InqAL models for verification?

## Key Takeaway
Inquisitive action logic rigorously formalizes the difference between what an agent can force and what an agent actively determines, treating determination as a modal-logical phenomenon centered on questions — providing a foundation for principled reasoning about agency and responsibility in multi-agent systems.
