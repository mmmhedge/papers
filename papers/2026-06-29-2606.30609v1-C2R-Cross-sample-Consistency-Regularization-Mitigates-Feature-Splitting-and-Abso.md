# C$^{2}$R: Cross-sample Consistency Regularization Mitigates Feature Splitting and Absorption in Sparse Autoencoders

**Authors:** Haoran Jin, Xiting Wang, Shijie Ren, Hong Xie, Defu Lian
**Published:** 2026-06-29 | **arXiv:** [2606.30609v1](http://arxiv.org/abs/2606.30609v1)

## Summary
This paper addresses critical failures in Sparse Autoencoders (SAEs)—a key interpretability tool for understanding language models—by identifying and fixing "feature splitting" (one concept fragmented across multiple latents) and "feature absorption" (arbitrary exceptions in general features). The authors propose Cross-sample Consistency Regularization (C²R), which enforces that semantic features are represented consistently across a batch, yielding more reliable and human-understandable model decompositions without sacrificing reconstruction performance.

## Theoretical Background
Sparse Autoencoders are neural networks designed to discover interpretable latent features in the high-dimensional activations of large language models. The core challenge: modern SAEs decompose a vector into a sparse linear combination of learned features (dictionary atoms), but scaling to large dictionaries introduces inconsistency — the same semantic concept may activate different latents across different input samples. This paper builds on prior work in mechanistic interpretability (understanding how neural networks compute) and dimensionality reduction. The key insight is that per-sample optimization (fitting each example independently) lacks global constraints, allowing redundant or interfering representations to proliferate. C²R adds a cross-batch penalty term that discourages directionally similar latents from co-activating, enforcing that each concept maps to a single unified latent vector.

## Core Contributions
1. **Systematic diagnosis** of two failure modes in SAEs at scale: feature splitting (coherent concepts fragmented into non-atomic latents) and feature absorption (general features gaining arbitrary exceptions), quantified empirically across models and dictionary sizes.
2. **Cross-sample Consistency Regularization (C²R)**: a novel regularization term that penalizes co-activation of directionally similar latents within a batch, enforcing consistent latent assignment across samples.
3. **Empirical validation** showing C²R mitigates both pathologies while preserving reconstruction fidelity—a non-trivial balance, since naive approaches often degrade model performance.
4. **Analysis of failure origins**: demonstrating that inconsistent per-sample optimization is the root cause, not architectural defects.

## Practical Applications
1. **Better model auditing**: When regulators or safety teams need to understand what a language model is actually computing, more reliable SAE decompositions enable trustworthy feature-level analysis.
2. **Mechanistic debugging**: ML engineers troubleshooting unexpected model behavior can now more confidently trace issues to specific, stable latent features.
3. **Bias detection**: Consistent features make it easier to identify and isolate dataset biases or unwanted correlations learned by the model.
4. **Knowledge distillation**: Cleaner sparse decompositions could improve distillation by providing clearer target feature structures for smaller models.

## Product & Startup Ideas
1. **Interpretability-as-a-Service**: A platform that applies C²R-enhanced SAEs to any proprietary LLM, generating interactive dashboards of stable, human-readable features for compliance and audit purposes.
2. **Feature-level anomaly detection**: Real-time monitoring of LLM behavior by tracking activation patterns of consistent latent features, alerting on unexpected feature firing.
3. **Model governance software**: B2B tools for enterprises to certify LLM behavior by validating that learned features align with organizational values or regulatory requirements.
4. **Interpretable fine-tuning**: Tools that allow non-ML experts to modify or constrain LLM behavior by editing or locking specific consistent features identified by C²R.
5. **Research infrastructure for interpretability**: A open-source/commercial toolkit that bundles C²R with visualization and analysis tools, becoming the standard for SAE-based LLM analysis.

## Technical Limitations
1. **Computational overhead**: Adding cross-batch penalties increases training cost; the paper doesn't fully characterize scalability to very large batch sizes or billion-parameter models.
2. **Batch dependency**: Performance may degrade with very small batches or online (streaming) settings where cross-sample consistency is weaker.
3. **Hyperparameter sensitivity**: The regularization weight (balancing consistency vs. reconstruction fidelity) likely requires tuning per model/dataset; generalization not established.
4. **Definition of "directional similarity"**: The cosine similarity threshold for grouping latents is a design choice; sensitivity to this threshold not fully explored.
5. **Evaluation metrics**: Interpretability is subjective; the paper likely relies on proxy metrics (sparsity, reconstruction loss) rather than ground-truth human validation of feature quality.
6. **Limited scope**: Evaluated on SAEs for transformer-based LLMs; unclear if findings transfer to other architectures or modalities (e.g., vision models).

## Open Questions
1. **How does C²R scale to trillion-parameter models or extremely large dictionaries?** Is there a theoretical limit to consistency achievable?
2. **Can we characterize the exact trade-off between latent consistency and reconstruction fidelity analytically?** When does the trade-off break down?
3. **Does C²R improve downstream tasks** that depend on interpretability (e.g., mechanistic model editing, bias mitigation)? Or does it only help subjective feature quality?
4. **How sensitive is C²R to batch composition?** Does diversity in a batch affect consistency enforcement?
5. **Can we automatically detect and measure feature splitting/absorption** without manual inspection, enabling better automated evaluation?
6. **Do consistent features learned by C²R generalize across model checkpoints or fine-tuning?** Are they stable over the lifetime of a model?

## Key Takeaway
Enforcing cross-sample consistency in sparse autoencoders via targeted regularization solves the fundamental problem of latent feature fragmentation and arbitrary exceptions, enabling more reliable and interpretable decompositions of language models.
