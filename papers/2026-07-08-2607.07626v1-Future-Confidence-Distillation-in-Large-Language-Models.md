# Future Confidence Distillation in Large Language Models

**Authors:** Sahil Kale
**Published:** 2026-07-08 | **arXiv:** [2607.07626v1](http://arxiv.org/abs/2607.07626v1)

## Summary

This paper investigates how LLMs develop confidence about their answers over time, rather than treating confidence as a fixed property of final outputs. The authors show that confidence estimates made *after* generating an answer are much better calibrated than pre-generation estimates, and they propose "future confidence distillation"—a technique that learns to predict post-generation confidence using only the hidden representations available before generation begins, avoiding the computational cost of full answer generation.

## Theoretical Background

The work builds on two cognitive psychology concepts applied to LLMs: Feeling-of-Knowing (FOK)—a pre-commitment judgment about whether you can answer a question—and Judgement-of-Learning (JOL)—a post-answer assessment of whether your response was correct. Traditional confidence calibration in neural networks has focused on either softmax scores or explicit verbal confidence, but this paper treats confidence as dynamic information embedded in hidden layer activations. The approach leverages linear probes—simple linear functions trained to extract information from learned representations—a standard technique in [[mechanistic-interpretability]] for understanding what information neural networks encode internally.

## Core Contributions

1. **Temporal confidence analysis**: Empirically demonstrates that post-solution (JOL) confidence is substantially better calibrated and more discriminative than pre-solution (FOK) confidence across multiple frontier and open-source LLMs.

2. **Hidden representation richness**: Shows that linear probes trained on hidden representations extract far more confidence-related information than models explicitly verbalize, suggesting latent representations contain privileged information.

3. **Future confidence distillation**: A novel distillation framework that trains predictors on pre-solution hidden representations using post-solution correctness labels as teacher signals, achieving much of the calibration benefit of post-solution confidence without requiring full answer generation at inference time.

4. **Transfer efficiency**: Demonstrates that distilled predictors are sample-efficient and transfer across datasets within the same domain.

## Connections to Other Work

This work relates to several research directions:

- **[[Mechanistic interpretability]]**: The use of linear probes to extract confidence information mirrors recent work decoding model capabilities from internal representations.
- **Knowledge distillation**: The core technique borrows from standard distillation approaches but applies them to a latent confidence signal rather than final logits.
- **Uncertainty quantification in NLP**: Complements prior work on confidence estimation in language models, which typically focuses on post-hoc methods or explicit verbalization.
- **Early-exit strategies**: Related to work on adaptive computation—if pre-solution confidence can be reliably predicted, it enables routing decisions before full generation.

## Practical Applications

1. **Retrieval-augmented generation (RAG) systems**: Route low-confidence queries to external knowledge sources before generation, reducing hallucination.

2. **Tool-use in agents**: Adaptive tool selection—if an LLM's pre-answer confidence is low, invoke external tools (calculators, search) before committing to a response.

3. **Inference cost reduction**: Early rejection of low-confidence queries, avoiding expensive full-sequence generation for queries the model will likely get wrong.

4. **Interactive systems with confidence-based escalation**: Automatically escalate to human review or alternative strategies when pre-generation confidence is below a threshold.

5. **Monitoring and auditing**: Confidence scores provide interpretable metrics for detecting model failure modes in production systems.

## Product & Startup Ideas

1. **Confidence-aware API layer**: A middleware service that wraps LLM inference and returns both outputs and distilled confidence scores, enabling downstream systems to make routing decisions automatically—useful for enterprises running multi-model inference pipelines.

2. **Adaptive generation platform**: A system that uses pre-answer confidence to decide between fast approximate generation, full generation, or external tool invocation, trading off latency and reliability per-query.

3. **LLM auditing and risk assessment tool**: SaaS platform that continuously monitors LLM deployments using confidence distillation, flagging high-error-rate segments and identifying fine-tuning opportunities.

4. **Confidence-guided data labeling**: Use distilled confidence to intelligently select which model predictions to send for human annotation, optimizing labeling budgets in active learning workflows.

5. **Hierarchical Q&A system**: Consumer product (e.g., educational platform, customer support) that uses confidence to decide whether to answer immediately, request clarification, or escalate to a human expert.

## Technical Limitations

1. **Domain transfer assumptions**: The paper shows transfer "within the same domain"—unclear how well distilled confidence generalizes across very different task distributions or domains.

2. **Inference-time access to hidden representations**: Requires the ability to extract intermediate activations at inference time, which may not be available in closed-source APIs or production systems without custom implementations.

3. **Calibration on specific models**: Distilled models are trained on specific LLMs and architectures—unclear if a probe trained on GPT-4 hidden states would work on Llama or other families without retraining.

4. **Computational cost not fully addressed**: While avoiding full generation, the approach still requires forward pass to hidden layers, plus linear probe inference—the actual latency savings vs. full generation are not quantified.

5. **Limited to correctness proxies**: The teacher signal is post-solution correctness; other dimensions of confidence (e.g., "how helpful is this response?") are not explored.

6. **Calibration regimes**: Assumes training and deployment operate in similar confidence regimes—miscalibration could occur if the base model is fine-tuned or if deployment queries differ from training distribution.

## Open Questions

1. **Cross-model distillation**: Can confidence estimates from one model family be distilled to another, or does each require its own probe?

2. **Temporal evolution granularity**: Which layers or timesteps contribute most to the FOK-to-JOL improvement? A layer-wise analysis could pinpoint where confidence matures.

3. **Multi-token confidence prediction**: Can the approach predict per-token confidence evolution, enabling early stopping not just at query level but at generation level?

4. **Adversarial robustness**: How does confidence distillation perform on out-of-distribution or adversarial inputs where the model is confidently wrong?

5. **Scaling laws**: Does distillation efficiency hold as models scale, or do larger models develop confidence differently?

6. **Task diversity**: How much variation exists in confidence evolution across reasoning tasks, factual QA, creative writing, code generation, etc.?

## Key Takeaway

Confidence information evolves during answer generation and can be reliably predicted before generation completes, enabling efficient, well-calibrated confidence estimates that don't require full inference.
