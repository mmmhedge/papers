# OrbitQuant: Data-Agnostic Quantization for Image and Video Diffusion Transformers

**Authors:** Donghyun Lee, Jitesh Chavan, Duy Nguyen, Sam Huang, Liming Jiang et al.
**Published:** 2026-07-02 | **arXiv:** [2607.02461v1](http://arxiv.org/abs/2607.02461v1)

## Summary
OrbitQuant is a post-training quantization method that compresses image and video diffusion transformers (DiTs) to low bit-widths without requiring new calibration data for different models or input conditions. The key insight is to quantize activations in a rotated, normalized basis where values stay predictable across all timesteps and prompts, enabling a single quantization codebook to work universally. This achieves state-of-the-art compression ratios (down to W2A4 — 2-bit weights, 4-bit activations) while preserving generation quality.

## Theoretical Background
Diffusion transformers are neural networks that generate images or videos by iteratively denoising random noise over multiple sampling steps. Post-training quantization (PTQ) reduces model size and inference cost by converting weights and activations from 32-bit floating point to lower precision (e.g., 8-bit integers). The core challenge in quantizing DiTs is that activation ranges change dramatically across:
- **Timesteps**: the denoising process starts with high-noise latents and converges to clean outputs
- **Prompts and guidance branches**: different text inputs and control signals produce different activation distributions
- **Layers and input dimensions**: each layer and channel operates at different scales

Prior PTQ methods rely on range estimation calibrated to specific data, forcing expensive re-calibration when switching models or tasks. OrbitQuant sidesteps this by exploiting a mathematical property: if you rotate and normalize activations into a special basis using a randomized permuted block-Hadamard (RPBH) transform, each coordinate's distribution becomes invariant to input and timestep, matching a fixed theoretical distribution (like Gaussian after normalization). This allows a single Lloyd-Max codebook (a classic quantization algorithm from information theory) to serve all contexts.

## Core Contributions
1. **Data-agnostic quantization principle**: The insight that RPBH rotation + normalization stabilizes activation distributions across timesteps, prompts, and guidance conditions, eliminating per-checkpoint calibration.

2. **Unified offline-online scheme**: Weights are quantized offline in the rotated basis, and the rotation is absorbed into weight matrices so it cancels during forward passes—only a single forward rotation on activations occurs at runtime, with negligible overhead.

3. **Cross-modality transfer**: The same quantizer works for both image and video generation (FLUX.1, Z-Image-Turbo, Wan 2.1, CogVideoX) without modality-specific tuning.

4. **Extreme low-bit results**: Achieves W2A4 quantization (2-bit weights, 4-bit activations) on image DiTs with usable quality—a significant step beyond prior PTQ work on generative models.

## Connections to Other Work
OrbitQuant builds on [[Post-Training Quantization]] (PTQ) literature but diverges from calibration-dependent approaches by using principled linear algebra (random rotations, Lloyd-Max quantization) rather than statistics fitting. It relates to:
- **Activation distribution analysis in transformers**: Prior work has noted that [[Transformers]] have shifting activation ranges; OrbitQuant solves this structurally rather than via adaptive normalization.
- **Low-rank and structured rotations**: The use of block-Hadamard transforms echoes fast numerical methods and structured random matrix theory.
- **Generative model compression**: Complements [[Quantization]], [[Knowledge Distillation]], and pruning approaches for diffusion models, but specifically targets the hardest case (weight+activation quantization at extreme low-bit).

## Practical Applications
1. **Edge deployment of generative AI**: W2A4 quantized DiTs can run video generation on mobile devices or embedded systems with strict memory budgets (e.g., on-device TikTok or Instagram filters).

2. **Cloud inference cost reduction**: Data centers serving image/video generation APIs (e.g., Stability AI, Runway) reduce memory footprint and latency per request, enabling higher throughput or lower hardware spend.

3. **Real-time video editing**: Quantized CogVideoX or similar models enable interactive video inpainting or style transfer in consumer applications.

4. **AR/VR content generation**: On-device texture or environment generation for augmented/virtual reality without cloud calls.

5. **Accessible generative AI**: Lower compute requirements democratize access to state-of-the-art image/video models in regions or organizations with limited infrastructure.

## Technical Limitations
1. **Quality loss at extreme compression**: W2A4 likely causes visible artifacts (color shifts, blurriness, or detail loss) compared to full precision—acceptable for some use cases (thumbnails, drafts) but not others (professional content).

2. **Initialization sensitivity**: The method relies on a good rotated basis; if RPBH rotation poorly concentrates activations in practice, quantization performance degrades. The paper doesn't deeply explore robustness to architecture variations.

3. **Calibration cost hidden in Lloyd-Max fitting**: While "data-agnostic," the method still requires fitting a codebook once per input dimension. For very large models, this calibration step's computational cost is unclear.

4. **Limited to DiTs**: Generalizes only to diffusion transformers; unclear if the approach works for other generative architectures (GANs, VAEs, autoregressive models) or discriminative transformers.

5. **Runtime overhead**: The forward RPBH rotation on activations adds latency; the paper claims this is "negligible" but doesn't quantify it against the inference speedup from quantization.

6. **Assumption of fixed marginal distribution**: The core idea assumes that normalization + rotation yields coordinates with a known, fixed marginal (e.g., Gaussian). If real activations deviate significantly, the Lloyd-Max codebook mismatches.

## Open Questions
1. **What is the actual Pareto frontier of latency vs. quality** for various bit-widths on consumer hardware (phones, laptops)? End-to-end measurements would clarify deployment viability.

2. **How does OrbitQuant perform on user-fine-tuned or LoRA-adapted checkpoints**? Real-world models often diverge from base checkpoints; does data-agnosticism still hold?

3. **Can the method be extended to mixed-precision quantization** (different bit-widths for different layers)? A follow-up could dynamically allocate bits based on sensitivity.

4. **Does the approach transfer to text-to-video models with explicit temporal layers** (e.g., temporal attention)? CogVideoX uses temporal modules; are they equally robust to quantization?

5. **What is the theoretical justification for RPBH concentration?** A more rigorous analysis of why RPBH rotation stabilizes activation distributions could guide design of even better rotations or hybrid methods.

6. **How does quantization interact with guidance (CFG) during inference?** Does W2A4 degrade the effect of classifier-free guidance scaling?

## Key Takeaway
OrbitQuant achieves unprecedented low-bit quantization (W2A4) of diffusion transformers by rotating activations into a basis where a single, universal quantization codebook works across all inputs, timesteps, and modalities—eliminating expensive per-model calibration and enabling practical edge deployment of state-of-the-art generative models.
