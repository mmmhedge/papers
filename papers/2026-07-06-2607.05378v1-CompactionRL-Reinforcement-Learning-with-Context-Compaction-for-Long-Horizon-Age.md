# CompactionRL: Reinforcement Learning with Context Compaction for Long-Horizon Agents

**Authors:** Yujiang Li, Zhenyu Hou, Yi Jing, Jie Tang, Yuxiao Dong
**Published:** 2026-07-06 | **arXiv:** [2607.05378v1](http://arxiv.org/abs/2607.05378v1)

## Summary
This paper tackles a fundamental constraint in long-horizon AI agents: language models have finite context windows that can be exceeded during extended task execution. CompactionRL is a reinforcement learning method that trains LLM agents to both solve tasks and generate summaries of past interactions, allowing the model to "compress" its context and continue working on problems that would otherwise exceed memory limits. The method achieves significant improvements on challenging benchmarks like SWE-bench (software engineering tasks), showing that learned compaction outperforms naive truncation.

## Theoretical Background
The work builds on two established areas. First, [[reinforcement-learning]] with language models — using RL to fine-tune LLMs to maximize task success (as opposed to supervised fine-tuning alone). Second, context window limitations in transformers — neural networks that process sequences using attention mechanisms, which scale quadratically with sequence length, making infinitely long contexts impractical. The paper also draws on advantage estimation methods from policy gradient RL, which estimate how much better an action is than baseline behavior. The key insight is that context compaction (summarization) is itself a learnable skill that RL can optimize alongside task execution.

## Core Contributions
1. **Joint optimization framework**: CompactionRL trains agents to simultaneously execute tasks and generate task-relevant summaries using a unified RL objective, rather than treating compaction as a separate pipeline stage.
2. **Token-level loss normalization**: A technical contribution addressing the fact that summaries are typically much shorter than full trajectories, preventing the RL loss on summary generation from being swamped by task execution loss.
3. **Cross-trajectory generalized advantage estimation (GAE)**: An adaptation of standard GAE that works across trajectory boundaries when context has been compacted, enabling the agent to learn from compressed rollouts as if they were contiguous.
4. **Empirical validation on agentic benchmarks**: Demonstrated 5–7 point Pass@1 improvements on SWE-bench Verified and Terminal-Bench 2.0 across multiple model sizes (30B to 750B parameters).

## Connections to Other Work
This relates to [[RLHF|Reinforcement Learning from Human Feedback]] and similar post-training paradigms, but focuses on behavioral constraints (context limits) rather than reward modeling. It complements work on prompt compression and retrieval-augmented generation, but learns compaction end-to-end rather than using heuristics. The paper also connects to research on long-context LLMs (e.g., extended position embeddings, sliding window attention) — CompactionRL offers a complementary approach when those are unavailable or insufficient. It sits at the intersection of agentic LLMs (models that act autonomously) and memory-constrained RL, a relatively underexplored corner of the literature.

## Practical Applications
1. **Software engineering automation**: SWE-bench tasks often require multi-step code exploration and modification across hundreds of files; CompactionRL enables agents to work on these without restarting or losing context.
2. **Long-running autonomous systems**: Robots or deployed agents that operate for hours/days and accumulate large interaction logs can use learned compaction to stay within memory budgets.
3. **Multi-step reasoning tasks**: Question-answering systems that require extensive scratchpad work (math, logic puzzles) can compress intermediate reasoning to continue without truncation artifacts.
4. **Cost reduction in deployed LLMs**: Inference on long sequences is expensive; learned, lossless compaction could reduce token consumption in production agentic systems.

## Product & Startup Ideas
1. **AI code assistant for enterprise repositories**: A GitHub Copilot-style tool that uses CompactionRL to handle week-long refactoring or migration tasks, maintaining coherent understanding across thousands of files without context resets.
2. **Autonomous DevOps agent SaaS**: Sell access to a long-horizon agent trained with CompactionRL that performs infrastructure-as-code tasks, system debugging, and deployment automation across multi-step workflows.
3. **Research paper analysis agent**: A service that reads 100+ papers, compresses its understanding mid-way, and generates high-quality surveys or identifies research gaps — outperforming naive summaries by maintaining semantic cohesion.
4. **Long-horizon game AI**: Train agents for open-world or puzzle games where optimal play requires 1000+ steps; CompactionRL could enable superhuman performance without architectural changes to the LLM.
5. **Medical diagnostic reasoning**: An AI assistant that reviews patient histories, lab results, and imaging over months/years, compacting findings to generate cumulative diagnostic or treatment recommendations.

## Technical Limitations
1. **Summary quality bottleneck**: The agent must learn both good task policies *and* good compaction. If it fails at either, downstream performance suffers; there's no easy way to debug which component is the culprit.
2. **Information loss**: Compression is inherently lossy; some task-relevant details may be discarded. The paper doesn't analyze what information is lost or how often this causes task failures downstream.
3. **Computational overhead during training**: RL training already requires many rollouts; adding compaction decisions and summary evaluation likely increases sample complexity.
4. **Generalization of learned compression**: Unclear whether summaries learned on coding tasks transfer to, say, reasoning or math tasks. The evaluation focuses narrowly on software engineering.
5. **Scalability to very long horizons**: Experiments go to thousands of tokens; unclear if the method scales to hours-long agent sessions or if compaction quality degrades after multiple compression cycles.
6. **Baseline comparisons**: No direct comparison to simpler baselines (e.g., BM25 retrieval, extractive summarization, or rule-based compaction), making it hard to isolate the RL contribution.

## Open Questions
1. **Information-theoretic limits**: What is the theoretical minimum summary length needed to preserve task-solving capability for a given task class? Can this be estimated?
2. **Compaction cascades**: What happens after compacting twice (summary of summary)? Does quality degrade gracefully or fail suddenly?
3. **Interpretability of learned summaries**: What linguistic patterns emerge in learned compaction? Are they human-interpretable or more like latent code?
4. **Multi-task transfer**: Train CompactionRL on coding; does it transfer to math, reasoning, or dialogue tasks with minimal fine-tuning?
5. **Hybrid approaches**: Could CompactionRL be combined with retrieval-augmented generation, allowing the agent to selectively re-fetch compressed summaries rather than re-read them?
6. **Adversarial compaction**: Can an adversary craft tasks where learned compaction fails? What is the robustness profile?

## Key Takeaway
RL can jointly optimize task execution and context summarization, enabling language model agents to work on problems far longer than their context windows would normally allow.
