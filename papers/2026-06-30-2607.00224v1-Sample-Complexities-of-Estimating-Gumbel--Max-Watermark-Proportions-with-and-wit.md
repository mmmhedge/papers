# Sample Complexities of Estimating Gumbel--Max Watermark Proportions with and without Reduction to Pivotal Statistics

**Authors:** Shuwen Chai, Qiaosen Wang
**Published:** 2026-06-30 | **arXiv:** [2607.00224v1](http://arxiv.org/abs/2607.00224v1)

## Summary
This paper solves a statistical problem: given a document that may be partially human-written and partially machine-generated, how can you estimate what fraction was produced by a specific watermarked language model? The authors analyze two methods under the Gumbel-max watermarking scheme—one using full information and one using a simplified "pivot" statistic—and prove how many document tokens you need to see to reliably estimate the watermark proportion. They show that the simpler method requires substantially more samples, suggesting it trades off computational elegance for statistical efficiency.

## Theoretical Background
Watermarking in large language models works by steering token selection toward a pre-specified set, leaving a statistical fingerprint that survives light editing. The Gumbel-max mechanism is a common watermarking approach that assigns scores to candidate tokens using a pseudorandom process.

The paper frames this as a statistical estimation problem where the true next-token distributions are unknown nuisance parameters. This requires understanding **sample complexity theory** — the minimum number of observations needed to estimate an unknown quantity to a given accuracy. The authors employ information-theoretic lower bounds (derived from properties of statistical divergences) and develop estimators (Laguerre-polynomial and event-counting variants) that achieve these bounds, making them **minimax-optimal**. The pivotal reduction they analyze compresses multi-dimensional observation data into a scalar summarizing statistic, which normally simplifies inference but here has a hidden cost.

## Core Contributions
1. **Proportion estimation framework**: Formalizes watermark proportion estimation under Gumbel-max as a statistical problem with unknown token distributions, moving beyond binary detection.
2. **Two estimators with matching bounds**: Develops a Laguerre-polynomial estimator for the pivotal-reduction regime and an event-counting estimator for full observation, each paired with information-theoretic lower bounds proving optimality.
3. **Sample complexity gap**: Proves that pivotal reduction incurs a substantially higher sample complexity than using full observations—a surprising negative result about a widely-used procedural simplification.
4. **Non-asymptotic analysis**: Provides finite-sample guarantees rather than only asymptotic rates, making results actionable for practitioners.

## Connections to Other Work
This paper addresses a gap between practical watermarking (e.g., [[OpenAI watermarking]], [[GPLRA watermarking]]) and theoretical understanding. It complements detection-focused work by asking a finer question: estimation rather than classification. The use of information-theoretic lower bounds and minimax-optimal estimators connects to classical statistical theory (e.g., **Le Cam's method**, **Fano's lemma**). The finding that dimensionality reduction harms sample efficiency echoes observations in high-dimensional statistics and relates to the [[curse of dimensionality]], though here applied to a novel setting.

## Practical Applications
1. **AI-generated content forensics**: Investigators analyzing documents suspected to contain AI-generated passages (e.g., academic submissions, legal discovery) can quantify how much of a document likely came from a specific watermarked model.
2. **Watermark robustness testing**: Developers of watermarking schemes use this to validate that their watermarks survive realistic editing scenarios (paraphrasing, deletion, summarization).
3. **Content authenticity scoring**: Platforms hosting generated content (e.g., AI art, code synthesis) could report confidence that a submission is "X% machine-generated" using this estimator.
4. **Copyright enforcement**: Creators using watermarked generation tools can provide stronger evidence of AI assistance in disputed ownership cases.

## Technical Limitations
1. **Non-degeneracy assumption**: The analysis assumes next-token distributions are "non-degenerate" (token probabilities don't collapse to zero probability sets), which may fail for highly constrained domains (e.g., code generation with specific syntax requirements).
2. **Gumbel-max specificity**: Results apply only to this watermarking mechanism; other schemes (e.g., hash-based, entropy-based) require separate analysis.
3. **Unknown token distributions**: The framework treats true token distributions as adversarial unknowns, but real estimators may benefit from data-driven token distribution estimates—gains not captured here.
4. **Watermark robustness not modeled**: The analysis assumes watermarks survive editing intact; real paraphrasing may partially destroy the watermark signal, breaking the assumptions.
5. **Computational cost not addressed**: The paper proves sample complexity but doesn't analyze runtime of the estimators (e.g., solving for Laguerre-polynomial roots may be expensive).
6. **Single watermark assumption**: Assumes all machine-generated tokens use the same watermarked model; mixed-source documents are not addressed.

## Open Questions
1. **Mixed watermarks**: Can you estimate proportions when a document uses multiple different watermarked models?
2. **Robustness to editing**: How do deletions, insertions, and paraphrasing degrade the watermark signal, and how does sample complexity grow as a function of edit distance?
3. **Dependent token contamination**: What if some tokens are neither purely watermarked nor purely human (e.g., hybrid generation)? How does the proportion estimate degrade?
4. **Optimal observation design**: Can adaptive or sequential sampling (choosing which tokens to examine) improve sample complexity beyond the non-adaptive lower bounds shown here?
5. **Computational-statistical tradeoff**: Is there a fast approximation with slightly higher sample complexity that avoids expensive polynomial computations?
6. **Empirical validation**: How well do these theoretical bounds match real watermarking implementations, especially after realistic editing and under actual LLM token distributions?

## Key Takeaway
Watermark proportion estimation is fundamentally more sample-efficient with full observations than with dimensionality reduction, suggesting practitioners should reconsider the computational convenience of pivotal statistics when estimation accuracy is at stake.
