# Progressive Crystallization: Turning Agent Exploration into Deterministic, Lower-Cost Workflows in Production

**Authors:** Arun Malik
**Published:** 2026-07-08 | **arXiv:** [2607.07052v1](http://arxiv.org/abs/2607.07052v1)

## Summary
This paper solves a critical problem in AI operations: AI agents that handle IT tasks are expensive because every execution requires full language model inference, even when solving the same problem repeatedly. The authors introduce "progressive crystallization," a system that learns from agent behavior over time and automatically converts successful patterns into cheap, deterministic workflows—like turning a thoughtful AI decision into a simple if-then rule. In production testing across tens of thousands of cloud incidents, the approach cut per-incident costs by 70% while tripling the fraction of fully automated (non-LLM-requiring) executions.

## Theoretical Background
This work sits at the intersection of agents (autonomous systems that take actions in an environment) and operational AI. It builds implicitly on reinforcement learning concepts—specifically reward mechanisms and policy extraction—but applies them to a pragmatic domain: turning exploratory agent behavior into locked-in production code.

The paper assumes a lifecycle model for intelligent systems: agents begin as flexible explorers (using LLM reasoning for each task), but over time, successful solutions become static. This echoes ideas in curriculum learning and skill discovery, though the authors frame it more as a DevOps optimization than a learning algorithm per se.

Key prior context: Most deployed AI agent systems treat inference as a constant cost. This paper challenges that assumption by introducing a promotion/demotion mechanism—akin to staged compilation or incremental program synthesis—where validated agent traces become deterministic code.

## Core Contributions
1. **A three-stage execution taxonomy**: fully agent-orchestrated → hybrid (agent + deterministic) → fully deterministic, with formal promotion/demotion criteria based on validation evidence.

2. **An evidence-based promotion algorithm**: uses trace extraction (recording agent decisions) and statistical thresholds to determine when an agent behavior is sufficiently validated to convert into deterministic logic.

3. **Production validation**: demonstrates the approach on a real cloud AIOps system, proving 45% shift to deterministic execution, 70% cost reduction, and improved auditability/safety.

4. **Economic and safety modeling**: articulates the trade-offs between inference cost, conversion overhead, and reproducibility—including automatic demotion when deterministic workflows regress.

## Connections to Other Work
This relates to [[RLHF]] (reinforcement learning from human feedback) in that it hardens successful agent behaviors, but diverges by targeting cost optimization rather than alignment.

It echoes concepts in [[program synthesis]] and [[inductive logic programming]]—learning rules from examples—but applied to operational workflows rather than pure code generation.

The demotion mechanism is novel: most agent systems don't model behavioral decay or rollback. This is closer to adaptive control theory, where a system monitors performance and switches strategies.

Also tangentially relevant: [[mechanistic interpretability]] work asks "what is the agent doing?"—this paper asks the complementary question: "which agent behaviors are worth locking into deterministic form?"

## Practical Applications
1. **Cloud incident response**: Automatically escalate common failure patterns (e.g., "if load spikes on service X, do Y") from agent-discovered to deterministic runbooks, reducing response latency and cost.

2. **Database administration**: Agent explores query optimization heuristics; once validated, convert to static query plans or indexing rules.

3. **Network troubleshooting**: Agent diagnoses routing anomalies; deterministic version becomes a network policy or alerting rule.

4. **Software deployment pipelines**: Agent learns safe deployment sequences; crystallize into immutable CI/CD stages.

5. **Customer support workflows**: Chatbot learns resolution patterns; convert high-confidence patterns into decision trees requiring no LLM calls.

## Product & Startup Ideas
1. **AIOps Crystallization Platform**: A SaaS product that sits on top of any LLM-based agent system (DataDog, New Relic, etc.) and automatically converts agent traces into cost-saving deterministic workflows, with built-in A/B testing and demotion safeguards.

2. **Intelligent Runbook Compiler**: Tools that take raw agent logs and automatically generate, version, and audit deterministic runbooks—targeting DevOps teams that want to reduce LLM costs while maintaining agent flexibility for novel problems.

3. **Behavioral Cost Optimizer for LLM Agents**: A monitoring and optimization layer (similar to FinOps for cloud costs) that tracks which agent behaviors are expensive, suggests crystallization candidates, and recommends rule extraction.

4. **Enterprise Workflow Hardening Service**: A consulting/platform hybrid that partners with enterprises to audit production agents, identify high-value crystallization targets, and produce certified deterministic replacements with compliance documentation.

5. **Incident Pattern Analytics**: A post-incident analytics tool that learns from ops teams' agent behavior (via logs/traces) and proactively suggests deterministic automations to prevent recurrence, with cost/safety trade-off dashboards.

## Technical Limitations
1. **Trace quality assumption**: The method requires rich, accurate logs of agent reasoning and decisions. Real systems may have noisy, incomplete, or biased traces.

2. **Stationarity problem**: The paper assumes problem distributions are relatively stable. In rapidly evolving infrastructure or adversarial environments, deterministic rules may become obsolete faster than agents adapt.

3. **Novel problem blindness**: Once behavior crystallizes, the system loses agent flexibility for truly new failure modes. The demotion mechanism helps, but there's inherent lag and switching cost.

4. **Extraction complexity**: Converting agent traces to deterministic logic is non-trivial. The paper doesn't deeply explore how to handle multi-step reasoning, uncertainty, or probabilistic decisions.

5. **Safety guarantees**: While reproducibility improves, converting agent heuristics to rules can lose important context and introduce false-positive automations (e.g., mis-triggering a runbook).

6. **Generalization**: Results are from one production system (cloud networking). Transferability to other domains (e.g., medical AI, financial trading) is unclear.

## Open Questions
1. **How sensitive is promotion/demotion to threshold tuning?** The paper likely uses heuristic confidence thresholds—what happens at boundary cases, and how do teams set these in practice?

2. **Can demotion keep pace with real-world change?** If a deterministic rule becomes stale, how quickly does monitoring detect it, and how often do rules oscillate between crystallized/demoted states?

3. **How does this scale to multi-step reasoning?** The paper focuses on incident response patterns; what about complex workflows requiring sequential LLM reasoning across many steps?

4. **Explainability of crystallized rules**: Can the system generate human-readable justifications for why a deterministic rule exists (i.e., what agent reasoning led to it)?

5. **Comparative economics**: How does this compare to other cost-reduction methods (e.g., fine-tuned smaller models, caching, distillation)?

6. **Human-in-the-loop promotion**: How do operators validate and approve promotion decisions? Is there a workflow for humans to reject or refine extracted rules?

## Key Takeaway
Progressive crystallization turns expensive, repeated LLM inference into cheap, auditable automation by learning which agent behaviors are stable enough to encode as deterministic rules—achieving 70% cost savings in production while maintaining safety and flexibility for novel problems.
