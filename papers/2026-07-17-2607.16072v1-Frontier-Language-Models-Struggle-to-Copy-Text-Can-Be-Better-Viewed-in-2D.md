# Frontier Language Models Struggle to Copy: Text Can Be Better Viewed in 2D

**Authors:** Haodong Wen, Yiran Zhang, Yingfa Chen, Kaifeng Lyu
**Published:** 2026-07-17 | **arXiv:** [2607.16072v1](http://arxiv.org/abs/2607.16072v1)

## Summary
This paper reveals that even the most advanced language models fail at a seemingly trivial task: exactly copying input text. The authors diagnose the problem in how Transformers encode position information and propose 2D-RoPE, a new positional encoding that organizes text as a 2D grid instead of a 1D sequence. With this simple geometric shift, shallow models achieve near-perfect copying at lengths hundreds of times longer than training, suggesting that how we represent text structure matters deeply.

## Theoretical Background
Transformers rely on positional encodings to tell the model where each token is in the sequence. The standard approach (like RoPE — Rotary Position Embeddings) encodes position as a single 1D coordinate. The paper argues this creates an architectural bias: the model learns to copy by matching local context patterns as shortcuts rather than precisely locating and retrieving tokens from specific positions. The core insight is that copying in 1D requires the model to learn abstract positional logic, whereas in 2D it becomes a simple mechanical lookup: find the token at row X, column Y+offset.

## Core Contributions
1. **Identification of a fundamental failure mode**: Demonstrates that frontier LLMs (GPT-4, Claude, Llama) struggle to copy exact input strings within their context windows — a task they should trivially handle.
2. **Root cause diagnosis**: Attributes this to positional encoding inductive bias that favors local-context matching shortcuts over precise position tracking.
3. **2D-RoPE proposal**: Introduces a novel positional encoding scheme that maps text to a 2D grid with row and column coordinates. Under this view, copying reduces to retrieving tokens at a fixed column offset.
4. **Empirical validation**: Shows shallow Transformers with 2D-RoPE achieve perfect copying at lengths 100–400× longer than training lengths. Validates benefits persist in large-scale pretraining (up to 1.4B parameters on DCLM).

## Connections to Other Work
This relates closely to work on [[Rotary Position Embeddings (RoPE)]] and positional encoding design in Transformers. It echoes concerns in [[mechanistic-interpretability]] about how architectural inductive biases shape model behavior in unexpected ways. The copying failure connects to broader questions about what Transformers struggle with despite having sufficient capacity — similar to findings around in-context learning and length extrapolation. The 2D reorganization is conceptually related to 2D attention patterns explored in vision Transformers and document understanding, though this applies the idea to pure text.

## Practical Applications
1. **Long-context retrieval and summarization**: Models that can precisely locate and extract information from long documents would improve fact-checking, citation generation, and document Q&A systems.
2. **Code generation and synthesis**: Copying and reusing exact code snippets (function signatures, boilerplate) is essential; better copying could improve code LLMs' ability to maintain consistency.
3. **Instruction following with precise references**: When users ask models to "copy this exactly," "repeat after me," or manipulate text strings (e.g., format conversion), 2D encoding could prevent corruption.
4. **Multimodal document understanding**: Organizing text with 2D coordinates could enable better alignment between text and spatial layout in scanned documents or web pages.

## Product & Startup Ideas
1. **LLM-based document recovery and OCR correction**: Train a specialized model with 2D-RoPE to detect and fix OCR errors by precisely matching and copying correct spans from reference documents.
2. **Code review and refactoring assistant**: A tool that uses 2D-aware models to identify duplicate code blocks and suggest refactoring—relying on perfect copying for consistency checking.
3. **Precision knowledge extraction API**: A hosted service for enterprises that needs to reliably extract and reconstruct exact passages from PDFs, contracts, or regulations without hallucination.
4. **Prompt engineering platform**: A debugger/optimizer for prompt engineers that precisely tracks how models handle variable substitution and template rendering by monitoring positional accuracy.
5. **Synthetic data generation for long-context benchmarks**: Sell high-quality datasets of verified long-context copying tasks, extrapolation challenges, and position-sensitive reasoning to train/evaluate proprietary LLMs.

## Technical Limitations
1. **Architectural rigidity**: 2D-RoPE assumes text naturally fits a grid structure. For irregular documents (tables, nested structures, mixed media), the encoding may be suboptimal.
2. **Scalability of grid design**: How to choose grid dimensions (rows vs. columns) for variable-length documents remains unclear; uninformed choices could harm other capabilities.
3. **Tradeoff with other abilities**: The paper doesn't thoroughly evaluate whether 2D-RoPE hurts performance on standard language modeling, reasoning, or generation tasks. Does improved copying come at a cost?
4. **Limited comparison**: No comparison to other positional encoding schemes (ALiBi, T5 bias, etc.). It's unclear if 2D-RoPE is uniquely good or if other changes to position encoding also help.
5. **Training data assumptions**: Experiments use relatively clean, synthetic data. Behavior on noisy, real-world text or highly compressed formats is untested.

## Open Questions
1. Does 2D-RoPE maintain or improve performance on standard GLUE, SuperGLUE, and downstream tasks? What's the full capability profile?
2. How sensitive is the method to grid dimension choice? Can we learn optimal grid structure from data?
3. Can 2D-RoPE help with other "alignment" tasks (e.g., strict format preservation, symbolic manipulation) beyond copying?
4. Does 2D structure help with hierarchical reasoning (paragraphs, sections, documents) compared to 1D?
5. Can we extend this to multimodal or structured data (XML, JSON, tables)? Does the 2D framing generalize?

## Key Takeaway
Standard Transformers fail at copying because their position encoding favors shortcuts; organizing text as a 2D grid makes copying mechanical and dramatically improves length extrapolation.
