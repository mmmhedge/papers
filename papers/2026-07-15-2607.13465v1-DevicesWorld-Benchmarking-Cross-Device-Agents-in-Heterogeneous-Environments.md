# DevicesWorld: Benchmarking Cross-Device Agents in Heterogeneous Environments

**Authors:** Huatao Li, Xinwei Geng, Yuheng Wang, Yutong Li, Runde Yang et al.
**Published:** 2026-07-15 | **arXiv:** [2607.13465v1](http://arxiv.org/abs/2607.13465v1)

## Summary
This paper introduces DevicesWorld, a benchmark for evaluating LLM-based agents that must coordinate actions across multiple heterogeneous devices (phones, desktops, IoT systems) to complete real-world tasks. Current agents struggle dramatically on this challenge—even the best system achieves only 12.5% success—revealing that today's agents cannot reliably acquire information from one device, process it on another, and deliver results to a third, highlighting a critical gap between single-device and real-world multi-device operation.

## Theoretical Background
The work builds on two established research directions. First, the rapid progress in [[large-language-models]] (LLMs) as agents — neural networks trained on vast text data that can reason about tasks and issue commands to digital systems. Second, prior benchmarks for evaluating these agents on **single-device domains**: mobile apps, desktop systems, and smart homes, each studied in isolation. The paper observes that these benchmarks miss a key real-world property: user goals spanning multiple devices. DevicesWorld adapts the benchmark methodology — defining tasks with verifiable outcomes and automatic evaluation — to the cross-device setting.

## Core Contributions
1. **DevicesWorld benchmark**: A large-scale (6,140 tasks) executable evaluation framework integrating mobile, desktop, and IoT environments into a unified interface, with automatic verifiers based on device state and file outputs.
2. **Diagnostic failure analysis**: Detailed categorization of failure modes — agents getting stuck on information acquisition, confusing source vs. output devices, or terminating prematurely — that go beyond simple success/failure metrics.
3. **Empirical findings**: Demonstration that five frontier LLM-agent systems all achieve very low success rates (best 12.5%), with ~28.7% of failures satisfying partial task conditions, indicating agents lack robust cross-device coordination.
4. **Reproducible evaluation infrastructure**: A multi-stage construction and quality-control pipeline that keeps tasks realistic while enabling deterministic verification.

## Connections to Other Work
This extends the lineage of LLM-agent benchmarks (e.g., WebArena, MobileArena for single-device tasks) into the multi-device space. It relates conceptually to [[multi-agent-systems]] research—though here agents coordinate with device state rather than with other agents—and shares evaluation methodology with [[reinforcement-learning]] benchmarks that use automatic verifiers. The work implicitly raises questions about whether standard [[large-language-models]] training and fine-tuning approaches (e.g., [[RLHF]]) are sufficient for sequential multi-device reasoning, hinting at needs for new architectures or training procedures.

## Practical Applications
1. **Smart home automation**: Coordinating actions across thermostats, lighting, security cameras, and phones to achieve composite goals (e.g., "lock the house and notify me on my phone").
2. **Enterprise workflow automation**: Agents that gather data from mobile field apps, process on desktop systems, and push results to cloud dashboards.
3. **Personal assistant services**: Cross-device task completion like "fetch an email attachment from my phone, edit it on my laptop, and save the final version to cloud storage."
4. **Accessibility tools**: Enabling users with mobility constraints to coordinate actions across devices with natural language commands.
5. **IoT orchestration**: Triggering complex device sequences (e.g., turn on smart lights, adjust AC, notify wearable) from a conversational interface.

## Product & Startup Ideas
1. **Cross-Device Task Automation Platform**: A B2B SaaS that uses LLM agents to automate multi-device workflows for enterprises, starting with IT helpdesk tasks (gathering logs from multiple systems, generating reports).
2. **AI Smart Home OS**: A unified operating system for heterogeneous smart home devices that coordinates them via language commands, bundling DevicesWorld-style evaluation into the product roadmap.
3. **Accessibility-First Agent Assistant**: A consumer app enabling people with disabilities to coordinate their phone, laptop, and smart home devices through conversational AI, with a focus on reliability and transparency about failure modes.
4. **Agent Evaluation & Debugging Service**: A tools platform for enterprises and researchers to test and diagnose LLM agents on cross-device tasks, selling both benchmark access and diagnostic dashboards.
5. **Vertical AI Agent for Field Services**: Target plumbers, electricians, contractors — agents that coordinate job scheduling (desktop), customer communication (mobile), and field IoT sensors in a single workflow.

## Technical Limitations
1. **Simulation vs. reality gap**: DevicesWorld uses rule-based verifiers and virtual device environments; real devices have unpredictable latency, crashes, and state inconsistencies that may break agent reasoning.
2. **No learning from failure**: The benchmark evaluates frozen LLM agents without online adaptation; production systems need to recover from partial failures across devices.
3. **Device heterogeneity underexplored**: Only three device classes (mobile, desktop, IoT) are included; real ecosystems (AR, wearables, automotive) are not addressed.
4. **Scalability of task construction**: The multi-stage quality-control pipeline is labor-intensive; unclear if the 6,140 tasks sufficiently cover the combinatorial explosion of cross-device task types.
5. **Agent reasoning transparency**: The paper documents failure modes but does not analyze *why* agents fail at cross-device coordination — whether it's poor planning, information retention, or interface understanding remains unclear.
6. **Synchronization and timing**: Tasks may require agents to understand state dependencies across devices (e.g., "wait for data to sync before processing"); the benchmark's task design and agent capability in this area are not detailed.

## Open Questions
1. **Can new training methods help?** Would agents trained with [[reinforcement-learning]] on cross-device tasks, rather than only on single-device data, achieve significantly higher success rates?
2. **What is the minimal architectural change needed?** Do standard transformers with memory buffers suffice, or do agents need explicit multi-device state representations or hierarchical planning?
3. **How do agents generalize across device types?** If trained on desktop + mobile tasks, do they automatically succeed on unseen IoT or AR tasks?
4. **What about real-time coordination?** Many real-world tasks involve waiting for events on one device before acting on another; how do agents handle asynchronous multi-device workflows?
5. **Can partial task satisfaction be recovered?** When an agent satisfies one condition but not all, can it be prompted or fine-tuned to complete the remaining steps, or does it need to restart?
6. **Scalability of verifiers**: How do automatic verifiers scale to tasks with complex output validation (e.g., "the spreadsheet should look visually correct")?

## Key Takeaway
LLM agents fail dramatically at coordinating actions across multiple devices—even simple cross-device tasks stump today's best systems—exposing a fundamental limitation in how agents reason about and transition between heterogeneous environments.
