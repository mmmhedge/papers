# VAIOM: Continuous-Input, Discrete-Output Decoder-Only Financial Sequence Modeling

**Authors:** Yiming Ma, Xinyu Chen
**Published:** 2026-07-15 | **arXiv:** [2607.13929v1](http://arxiv.org/abs/2607.13929v1)

## Summary
This paper presents VAIOM, a decoder-only Transformer designed to predict next-period returns in foreign-exchange markets by accepting continuous financial data (prices, volumes, events) as input while outputting discrete return predictions. The key innovation is separating input representation—which preserves numerical structure—from output modeling, which uses categorical distributions over return buckets, outperforming traditional machine learning baselines on real FX data across multiple test periods.

## Theoretical Background
The paper builds on two distinct traditions: (1) **decoder-only Transformers** — autoregressive neural networks that predict sequences one token at a time using self-attention mechanisms to weight dependencies; and (2) **quantitative finance**, which typically frames return prediction as classification or regression on engineered features. Prior work in financial ML often discretizes inputs (e.g., tokenizing price levels) or treats returns as continuous values. VAIOM instead bridges these by keeping inputs continuous—preserving their numerical structure and relationships—while treating outputs as ordinal categories (discrete return buckets ranked by magnitude). This aligns with financial domain knowledge: returns naturally belong to ordered buckets (large negative, small negative, flat, small positive, large positive), not arbitrary categories.

## Core Contributions
1. **Continuous-input, categorical-output architecture**: Decouples input representation (multivariate vectors of FX events) from output likelihood (categorical distribution over volatility-normalized return buckets), showing continuous inputs outperform discrete tokenization under the same training setup.

2. **Mixture-of-Market-States return head**: A structured output layer that conditions return probabilities on latent market regime states, improving likelihood over naive categorical outputs.

3. **Full-sequence supervision with auxiliary objectives**: Uses all positions in a sequence for training (not just final tokens) combined with auxiliary losses (Gap, volatility-regime, Ordinal prediction tasks) to shape learned representations.

4. **Empirical validation on real FX data**: Demonstrates consistent gains (0.029–0.043 bits per event) over LightGBM baselines across three seeds and two independent 2025 test periods, with ablations isolating the contribution of each component.

5. **Capacity study**: Shows the smallest evaluated architecture achieves best validation likelihood, suggesting efficient models may be sufficient for this task.

## Connections to Other Work
VAIOM sits at the intersection of [[Transformers|transformer-based sequence modeling]] and financial forecasting. It echoes the success of [[Attention Is All You Need]] in capturing long-range dependencies, but applies it to continuous domain data rather than symbolic tokens. The ordinal-return framing relates to **ordinal regression** in ML, which respects structure between labels. The mixture-of-experts-style return head parallels recent work in [[Multi-Agent Systems|mixture-of-experts]] architectures. Unlike [[RLHF]] or reinforcement learning approaches to trading, VAIOM is purely supervised, predicting returns rather than learning policies. It differs from prior financial deep learning work (RNNs, LSTMs on FX) by using attention mechanisms and by explicitly designing for continuous heterogeneous inputs.

## Practical Applications
1. **Real-time FX trading**: Hourly return predictions feed algorithmic trading strategies, replacing or augmenting rule-based technical indicators.

2. **Risk management**: Banks and hedge funds use return probability estimates to size positions, compute value-at-risk, or set stop losses dynamically.

3. **Portfolio rebalancing**: Systematic funds use predicted return distributions to adjust currency exposures across multi-currency portfolios.

4. **Market-making**: FX market makers use next-hour return predictions to adjust bid-ask spreads and inventory hedging.

5. **Volatility forecasting**: The auxiliary volatility-regime objective can inform vol derivatives pricing and hedging.

## Technical Limitations
1. **Limited to FX, one-hour bars**: Generalization to equities, bonds, or other timeframes untested; model may overfit FX-specific patterns.

2. **No online adaptation**: Models are static after training; they cannot learn from live market drift or regime shifts (2024–2025 distribution may differ from 2026+).

3. **Discrete return buckets**: Binning returns loses information about tail severity; a large -10% and -20% move both map to "large negative," losing granularity.

4. **Causality not addressed**: Model learns correlations in historical data; no guarantee predictions have causal interpretability or will hold out-of-sample under policy changes or market structure shifts.

5. **Baseline is weak**: LightGBM is a strong, simple baseline, but no comparison to other Transformer-based financial models, RNNs, or state-of-the-art time-series forecasters.

6. **Computational cost**: Transformer inference per event is slower than linear models; scalability to microsecond latency trading or many currency pairs unclear.

7. **Data leakage risk**: Validation on 2024H2 and test on 2025 is good practice, but no discussion of look-ahead bias, data cleaning, or survivorship bias in the FX tick data pipeline.

## Open Questions
1. **Does VAIOM generalize to other assets?** How does performance change on equities, commodities, or crypto? Is the FX-specific structure essential?

2. **How to handle regime shifts?** Can the model detect when 2026 market conditions diverge from training data, and does performance degrade gracefully?

3. **Can we improve long-horizon predictions?** The paper focuses on next-hour returns; can the same architecture predict 4-hour or daily returns with similar gains?

4. **What do learned representations encode?** Mechanistic interpretability (e.g., attention patterns, representation similarity) could reveal whether the model learns microstructure, carries, or other economically meaningful signals.

5. **How sensitive is the model to input features?** Ablation study on input event types and feature engineering choices would clarify which data matters most.

6. **Can we incorporate uncertainty quantification?** Beyond categorical probabilities, can the model output confidence intervals or detect when it is out-of-distribution?

7. **Does mixture-of-market-states scale?** How many latent regimes are optimal? Does performance improve with more regimes or richer conditioning?

## Key Takeaway
Decoder-only Transformers with continuous inputs outperform traditional baselines at predicting hourly FX returns when outputs are structured as ordinal categories.
